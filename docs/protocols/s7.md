# S7comm (Siemens, port 102)

Siemens S7comm rides on top of TPKT (RFC 1006) + COTP (ISO 8073).
S7-300 / S7-400 / S7-1200 / S7-1500 all speak variants on port 102.
The protocol does not authenticate by default; newer firmwares add
"access protection" but many production PLCs leave it disabled.

## Probe

1. Open TCP to port 102.
2. Send a TPKT-wrapped COTP Connection Request with SrcTSAP 0x0100
   and DstTSAP 0x0102 (rack 0, slot 2 — the S7-300/400 default).
3. Read the COTP Connection Confirm. A CC response indicates an S7
   family PLC; capability score bumps from 30 → 70.

A COTP Disconnect Request or silent close means the target is
either not S7 or uses a different TSAP pair.

## Proxy policy (default build)

Every TPKT envelope is parsed; the COTP PDU type + S7 function byte
are extracted. CategoryRead functions (Setup Communication 0xF0,
Read Var 0x04) forward; everything else short-circuits to an S7
AckData with error class 0x85 ("Function not allowed") and error
code 0x01. The PDU Ref is echoed for correlation.

## Writes (`-tags offensive`)

`offensive/write/s7` implements:

| Op             | S7 function   | Notes                                  |
|----------------|---------------|----------------------------------------|
| `write_var`    | 0x05          | Byte / Char transport, ≤200-byte cap   |
| `plc_stop`     | 0x29          | Trailing "P_PROGRAM" ident             |
| `plc_restart`  | 0x28          | Cold restart                           |

PLC Stop is functionally a safety event; always triple-confirm
explicitly with the operator.

## Scope

Siemens S7 family — S7-200, S7-300, S7-400, S7-1200, S7-1500, as well
as S7-compatible clones (Vipa, others). The `Setup Communication`
response carries PLC type + firmware identifiers.
