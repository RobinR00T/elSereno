### Design patterns established

- **PBX brand matcher** (priority-ordered substring list):
  `internal/protocols/sip/vendor.go` is the template. Each
  entry is `{lowercase-needle, Vendor}`. More-specific
  strings come first so "cucm" wins before "cisco". Add new
  brands to the matcher list + a `VendorRisk` case in the
  same file.
- **Scoring**: every PBX-confirming probe sets
  `capability=60` (up from the default 30) and uses
  `VendorRisk` for `protocol_risk`. Default 70 for
  unknown-brand SIP responders; 75 for proxies (Kamailio,
  OpenSIPS, FreeSWITCH, SER, Cisco SIP Gateway); 80 for SOHO
  (Yeastar, Grandstream, Fanvil, Yealink); 85 for enterprise
  (Cisco UCM, Avaya, Mitel); 90 for attack-ripe (Asterisk,
  FreePBX, 3CX).
- **Protocol-native refusals** on the default proxy:
    SIP → `SIP/2.0 403 Forbidden` with Server header.
    IAX2 → no reply (UDP; return error from Handle).
  Offensive write-gated variants will ship with
  `offensive/write/sip` + `offensive/write/iax2` in v1.4.
- **Mini-frame / short-frame sanity**: IAX2 mini-frames are
  exactly 4 bytes with byte[0]&0x80==0. Anything else that
  trips ErrMiniFrame is not-IAX2 (HTTP starts with 'H'=0x48).
  See `iax2.go`'s explicit `n == 4` guard.
