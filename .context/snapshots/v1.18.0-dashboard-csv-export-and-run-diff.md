# Snapshot — v1.18.0: dashboard UX (CSV export + run diff)

Two chunks closing long-standing TODO-vNext operator-UX items:
CSV export from the dashboard's Findings panel + a Diff
between runs panel that highlights what changed week-over-week.
Pure dashboard / API additions; no protocol-plugin or audit-
chain changes.

## Chunk 1 — Dashboard CSV export from UI

`cc157d4 feat(v1.18 chunk 1): Dashboard CSV export from UI`

- New `?format=csv` query param on `GET /api/v1/findings`.
  Returns `text/csv; charset=utf-8` with
  `Content-Disposition: attachment;
  filename="findings-<RFC3339>.csv"`.
- Body is RFC-4180. Column order: id, run_id, target_id,
  protocol, severity, score, created_at (RFC3339Nano UTC),
  factors. Factors render as `name=value;name=value` with
  names sorted alphabetically — stable across exports for
  diff-friendly daily snapshots.
- Dashboard Findings panel gains a "Download CSV (top 500)"
  link.
- Backwards compat: no format param → JSON envelope (v1.2
  default). Byte-identical to pre-v1.18 behaviour.
- 3 new tests (CSV format / case-insensitive / no-format
  default to JSON).

## Chunk 2 — Dashboard diff between runs

`1225312 feat(v1.18 chunk 2): Dashboard diff between runs`

- New `GET /api/v1/findings/diff?old=<run>&new=<run>` returns
  a categorised JSON envelope with three buckets: `new` (new
  run only), `resolved` (old run only), `persisting` (in
  both). Match key is (target_id, protocol); the persisting
  bucket carries the new-run row so operators see the
  freshest score / factors.
- Required query params: `old=` + `new=` run IDs. 400 on
  missing or when old==new (operator-typo guard).
- `repo.FindingsQuery` gains a `RunID string` filter; new
  `repo.DiffFindings` library function + pure helper
  `diffFindingsByTargetProtocol` exposed for unit tests.
- Dashboard gains a "Diff between runs" panel with two-input
  form + per-bucket result tables.
- 9 new tests: 6 in `internal/repo/findings_diff_test.go`
  (unit-level bucketiser); 4 in
  `internal/web/handlers/findings_test.go` (HTTP handler).
