# Snapshot — v1.99.0: schedule import endpoint

Single-chunk cycle. Closes the v1.97 carryover. The v1.97
export endpoint had a 1-way story: operators could
download CSV/NDJSON for backup but had to write their own
restore loop. v1.99 adds the server-side import with
explicit conflict-resolution.

## REST

```
POST /api/v1/schedules/import
POST /api/v1/schedules/import?on_conflict=skip|overwrite|rename

Content-Type: application/x-ndjson  → one ScanSchedule per line
Content-Type: application/json      → JSON array of ScanSchedule
(missing/unknown CT auto-detects: starts with `[` → JSON array,
 else NDJSON)
```

`on_conflict` (default `skip`):

  - **skip** — same-name schedule exists → leave it alone;
    outcome="skipped".
  - **overwrite** — Update existing via internal Update path
    (preserves ID + CreatedAt + LastFiredAt); outcome="overwritten".
  - **rename** — append " (imported)" (then " (imported 2)",
    "(imported 3)"… for sequential collisions); Create with
    fresh ID; outcome="renamed".

Server **ignores** the input's `id` field — generates fresh
IDs regardless. Operators importing from another deployment
shouldn't have to coordinate IDs.

Response:

```json
{
  "schema": "api:v1",
  "data": {
    "imported":    N,
    "skipped":     N,
    "overwritten": N,
    "renamed":     N,
    "errors":      N,
    "items": [
      {"index": 0, "name": "alpha", "outcome": "created", "id": "<fresh>"},
      …
    ]
  }
}
```

## Honest scope

  - LastFiredAt + UpdatedAt from input are dropped — Create
    stamps fresh values; Update stamps a fresh
    UpdatedAt. Import isn't a "restore-to-point-in-time".
  - No transactional all-or-nothing — partial-failure path
    leaves earlier rows committed. The aggregate response
    includes per-row outcomes so operators can re-run with
    a filtered subset.
  - No idempotency-key support yet. Re-running the same
    import → duplicate runs through Skip/Overwrite/Rename
    semantics deterministically, but back-to-back rename
    runs would compound the suffix
    ("alpha (imported)" → "alpha (imported 2)" on the next
    import).
  - No dashboard UI for import yet — operators use curl
    with the downloaded NDJSON file. UI deferred.
  - INSTALL.md unchanged. Build sizes unchanged.
