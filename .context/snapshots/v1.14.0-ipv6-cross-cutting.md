# Snapshot — v1.14.0: IPv6 cross-cutting

Closed **2026-04-26**. Four-chunk cycle covering the
operator-requested IPv6 cross-cutting work (request 2026-04-25).
Theme: audit + canonicalise IPv6 paths across the proxy
listen, dry-run, scan input, scope-gate, and target-dedup
layers. Two real bugs fixed; rest is contract-pinning tests
on infrastructure that was already correct.

## Why this cycle matters

ElSereno already used `net/netip` for the Addr-typed paths
(resolve.go, dedupe.go, scope.go, every protocol probe).
The IPv6 hazards lived at the string boundaries: a
substring-based loopback check that missed v6 longform / zone
specifiers, a `--target` byte-for-byte compare that would
fail across equivalent IPv6 forms, a missing dispatcher case
for `internetdb:` that broke the v6 `[::]:port` form, and
zero contract tests for the scope + dedupe v6 paths.

Operator UX before v1.14: type `--target [0:0:0:0:0:0:0:1]:7547`
in dry-run, type `--target [::1]:7547` in `proxy listen` →
byte-for-byte compare fails → confirm-token mismatch. Now
all variants converge to the canonical RFC 5952 form before
the hash is computed.

## Per-chunk delta

### 1 `8824885` — IPv6 foundation: `internal/netutil` package

New helpers:
- `IsLoopbackHostPort(s string) bool` — delegates to
  `netip.ParseAddrPort` + `Addr.IsLoopback()`. Catches every
  spec-conformant variant per RFC 1122 (IPv4 — entire 127/8) +
  RFC 5952 (IPv6 — longform / shortform / zone-scoped).
- `CanonicalHostPort(s string) (string, error)` — normalises
  via `netip.AddrPort.String()`: longform → short, lowercase
  hex, RFC 5952-canonical.
- `ParseAddrPort(s string) (netip.AddrPort, error)` — thin
  wrapper that returns `ErrEmptyHostPort` for empty input.

Replaces fragile substring-based `isLoopbackAddr` in
`cmd_serve.go` (which missed IPv6 longform `[0:0:0:0:0:0:0:1]:port`,
zone-scoped `[::1%lo0]:port`, and IPv4 anywhere-in-127/8 like
`127.0.0.5:8787`). 18 unit tests.

### 2 `0de0923` — Target canonicalisation

Operator UX hazard closed: the canonicaliseTarget helper now
runs at every CLI parse boundary where a target string flows
into a hash, byte-for-byte compare, or YAML emit. Wire-up:
- `runProxyListen` (target / listen / confirmTarget after
  loadAllowFile + validate).
- 6 dry-run RunE handlers (sip / iax2 / pbxhttp / modbus /
  opcua / cwmp).
- `runBACnetDryRun` (in.target).

9 new tests including 6 per-plugin SessionMutation hash-
equivalence regressions (BACnet / OPC UA / SIP / IAX2 /
pbxhttp / CWMP). Backwards compat: IPv4 unchanged; IPv6
already-canonical unchanged.

### 3 `e0cae6f` — `scan --input internetdb:` IPv6 fixes

Two real bugs:

1. **Missing dispatcher case** in `cmd_scan.go` — the CLI
   accepted the `internetdb:` prefix but `readTargets`
   errored as "unknown input kind". Regression from v1.13
   chunk 1 — `readInternetDBTargets` was wired through
   `readTargetsFromProvider` but the dispatcher case was
   forgotten.
2. **IPv6 brackets rejected**: `netip.ParseAddr("[2001:db8::1]")`
   fails because brackets aren't part of the IPv6 literal.
   Operators typing `--input internetdb:[2001:db8::1]`
   (mirroring the host:port bracket convention) always hit
   `ErrInvalidIP`.

Fixes:
- Add `case strings.HasPrefix(opts.inputKind, "internetdb:")`
  to readTargets.
- New `stripIPv6Brackets` helper at the CLI boundary —
  applied to single-IP, file-bulk, and stdin-bulk forms.
- The library `internetdb.Client.Lookup` stays strictly
  RFC-conformant (bare literals only); the bracket
  ergonomics live in the CLI layer.

14 new tests including a httptest-driven IPv6 round-trip and
a dispatcher regression guard.

### 4 `59e7d76` — IPv6 coverage tests for scope + dedupe

Audit-only chunk: confirms the existing `netip.Addr` +
`Unmap()` infrastructure handles IPv6 across the scope-gate
and target-deduplication layers. No code change needed; 9
tests pin the contract:

- 5 scope tests (in-range, host-prefix `::1/128`, out-of-range
  fe80::, IPv4-mapped-IPv6 collapse via `Unmap()`, no
  cross-family leakage).
- 4 dedupe tests (v4-mapped + bare-v4 collapse, longform +
  shortform IPv6 collapse, v6 vs v4 stay distinct, different
  ports preserved).

The canonical safety invariant proven: scope cannot be
bypassed by using the `::ffff:1.2.3.4` v4-mapped form
(pre-existing infrastructure does the right thing; chunk 4
makes it a regression-proof test).

## What's NOT in this cycle

Out-of-scope deferrals:
- IPv6 in `proxy.New` Listen-string parsing — `internal/proxy`
  uses `net.Listen("tcp", addr)` which already accepts
  `[::]:port` literals; no change needed.
- IPv6 in protocol probes — every plugin uses `netip.Addr`
  end-to-end (verified by audit; explicit per-plugin tests
  not added since the underlying `net.Dial` accepts both
  families uniformly).
- DNS A+AAAA resolve — `internal/scanner/resolve.go` already
  resolves both record types; existing tests cover.

These are tracked as "no work needed" in the cycle close
rather than separate chunks.
