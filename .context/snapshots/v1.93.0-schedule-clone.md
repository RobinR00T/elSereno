# Snapshot — v1.93.0: schedule clone endpoint

Single-chunk cycle. Operators previously had to GET +
edit-by-hand + POST to duplicate a schedule. v1.93 adds a
single-call shortcut.

## REST

```
POST /api/v1/schedules/{id}/clone
Content-Type: application/json

{}                          // empty body → full copy
{"name":"weekly-fleet"}     // rename only
{"name":"x", "interval_seconds":604800}  // rename + recadence
```

Body fields are optional overrides on top of the source
schedule's values. Default name is `"<source.name> (copy)"`
when omitted.

Cadence overrides apply atomically — providing
`interval_seconds` OR `cron_expr` replaces the source's
cadence + timezone in one operation (XOR still enforced
by validation).

Returns 201 + the new schedule's full body. 404 if source
unknown. 400 on body parse + validation errors (mirrors
Create's error mapping).

## Copy semantics

  - Clone gets a fresh ID + CreatedAt + UpdatedAt.
  - Enabled = true regardless of source state. Operators
    cloning a disabled schedule almost always want a
    working test.
  - LastFiredAt does NOT carry over (clone is "never-fired"
    until the next scheduler tick).
  - Operator = the cloner (X-Operator header), not the
    source's operator. Provenance per creation; source
    schedule unaffected.

## Dashboard

New "Clone" button on each schedule row. Single-click
POSTs with empty body → renders the new row in the table
on the next poll. Operator can immediately edit the
clone via the Edit button to refine the rename.

## Tests (+3)

  - TestCloneSchedule_DefaultName: empty body → name
    becomes "<source> (copy)"; ID differs; Enabled=true.
  - TestCloneSchedule_OverrideName: body name wins;
    interval_seconds override applies.
  - TestCloneSchedule_NotFound: unknown source ID → 404.

## Honest scope

  - No bulk-clone endpoint. "Clone 10 schedules" is N
    HTTP calls. The use case is "fork this one to tweak
    its cadence", not bulk migration.
  - audit_retention_days is NOT a forced override — 0 in
    body = inherit source's value (which itself may be
    inherit-global). Setting to a non-zero value overrides.
  - No audit event for the clone. The source schedule
    isn't modified; the clone is a fresh Create that
    will pick up future force-overwrite / delete events
    on its own. Consider a `cloned_from` event in a
    future cycle if operators need provenance trails.
  - INSTALL.md unchanged.
