# Snapshot — v1.98.0: OpenAPI strict schemas

Single-chunk cycle. Closes v1.97 carryover. The v1.96 spec
listed every schedule endpoint but referenced the generic
`Envelope` for all bodies. SDK generators couldn't produce
typed clients. v1.98 adds proper component schemas + wires
requestBody refs.

## Operation struct extension

```go
type Operation struct {
    Summary, Description string
    Tags                 []string
    RequestBody          *RequestBody   // NEW
    Responses            map[string]Response
}
type RequestBody struct {
    Description string
    Required    bool
    Ref         string
}
```

`renderOperation` now emits `requestBody.content.application/json.schema.$ref`
for operations that declare a body.

## Component schemas added

  - `SubmitRequest` (input + plugins[] + default_port).
  - `CreateScheduleRequest` (name + template + cadence + tz +
    audit_retention_days; min/max bounds documented).
  - `UpdateScheduleRequest` (same shape minus operator).
  - `CloneScheduleRequest` (all-optional overrides).
  - `ScanSchedule` (full response shape with required
    id/name/template/enabled/created_at/updated_at).
  - `Stats`, `Job` (with triggered_by_schedule_id v1.92+),
    `ScheduleAuditEvent` (with event_type enum mirroring
    the SQL CHECK).

## Operations updated to ref schemas

POST/PUT/clone/preview operations now carry typed
requestBody; 201/200 responses now Ref → `ScanSchedule`
instead of `Envelope`. Audit/runs list responses keep
`Envelope` (because the wrapper layer is what they
return; data inside is array — documented in description).

## Refactor

`specComponents` extended via `scheduleSchemas()` merge.
Per funlen, split into:
  - `scheduleRequestSchemas` (4 body types).
  - `scheduleResponseSchemas` (composes 4 per-schema
    helpers: `scanScheduleSchema`, `statsSchema`,
    `jobSchema`, `scheduleAuditEventSchema`).

## Honest scope

  - Schemas describe shape, not semantic invariants:
    cadence-XOR (interval_seconds vs cron_expr) is
    documented per-field but not enforced via oneOf/
    discriminator (operators relying on SDK generators
    still need the server validation as the source of
    truth).
  - INSTALL.md unchanged. Build sizes unchanged.
