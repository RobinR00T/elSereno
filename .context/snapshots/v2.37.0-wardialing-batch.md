# Snapshot — v2.37.0: wardialing batch orchestrator

Single-chunk cycle. Closes deferred roadmap item #1 (grande).
The v1.1 `dial batch` verb classified a list of numbers
one-by-one with audit. v2.37 adds the full wardialing
orchestrator on top: range expansion + concurrency +
rate-limit + resume-from-checkpoint.

## What changed

### `offensive/dial/range.go` (NEW)

  - `ExpandRange("555-0100..555-0199")` → 100 strings.
  - Handles E.164 prefixes (`+34900-100-100..+34900-100-105`).
  - Preserves leading zeros (`0001..0010`).
  - Errors: `ErrRangeMalformed`, `ErrRangePrefixMismatch`,
    `ErrRangeSuffixLength`, `ErrRangeReversed`,
    `ErrRangeTooLarge` (capped at MaxRangeSize = 10_000).
  - `IsRangeSpec(s)` — quick contains-".." check.

### `offensive/dial/wardial.go` (NEW)

  - `Wardial` struct extends Batch's scope+audit fields with:
    - `Workers int` (clamp [1, 32]).
    - `RatePerSecond float64` (0 = no limit).
    - `CheckpointPath string` (resume marker file).
  - `Wardial.Run(ctx, rangeOrFile, r)` accepts either a range
    spec or an io.Reader of numbers.
  - Workers fan-out classification + audit append. Audit
    Writer concurrency-safe (ADR-013).
  - Dispatcher rate-limits global pace with time.Sleep
    between dispatches.
  - Checkpoint file: append-on-success; skip lines already
    present on resume.

### `cmd/elsereno/cmd_dial_offensive.go` (extended)

New verb `elsereno-offensive dial wardial`:

```
elsereno-offensive dial wardial \
    --range 555-0100..555-0199 \
    --workers 4 \
    --rate 5 \
    --checkpoint /tmp/wardial.ckpt \
    --scope scope.yaml
```

Disposition default = "preview" (audit-only dry-run). Real
hardware delivery still vNext.

### Hard-block invariant

≤3-digit hard block + scope.blocked_numbers checks apply
unchanged via the shared `classify()`. Cannot bypass.

## Refactor

`runConcurrent` split into 4 helpers (clampedParams,
batchFromConfig, worker, dispatch) to clear gocyclo +
funlen. CLI verb body extracted into `runDialWardial` +
`wardialFlags` struct for the same reason.

## Honest scope

  - "Wardialing" here ends at classify+audit. The
    Disposition field records intent but no PSTN/VoIP
    backend dials. Hardware delivery is a separate cycle.
  - Checkpoint file is best-effort append; if disk
    full mid-write, the line is incomplete and resume
    re-processes it. Idempotent classify makes this safe
    (just one extra audit entry).
  - Range size capped at MaxRangeSize=10_000. Larger
    wardials should be split (also gives better
    checkpoint granularity).
  - INSTALL.md unchanged (offensive verb auto-registers
    under existing dial tree).
