# Snapshot — v2.7.0: ETag/If-None-Match on read endpoints

Single-chunk cycle. The dashboard polls schedule endpoints
every 30s. Each poll sends full JSON even when nothing
changed. v2.7 adds HTTP-cache validation to skip
unnecessary body transfers.

## New helper

`writeJSONWithETag(w, r, v)` in `internal/web/handlers/api.go`:

  - MarshalIndent the payload.
  - SHA-256 → first 16 hex chars → ETag header value
    (`"abcdef0123456789"`).
  - `Cache-Control: private, max-age=0, must-revalidate`.
  - If request's `If-None-Match` matches the computed ETag
    → respond 304 Not Modified (no body).
  - Else 200 + body.

64 bits of collision-resistance; sufficient for cache
validation + keeps the header short.

## Endpoints wired

  - `GET /api/v1/schedules/tags` (v2.5) — tag-counts, very
    low churn, dashboard polls every 30s.
  - `GET /api/v1/schedules/{id}/audit` (v1.84) — audit
    events are immutable past records; append invalidates.
  - `GET /api/v1/schedules/{id}/runs` (v1.92) — runs are
    append-only at the head; old pages are stable.

## Endpoints deliberately NOT wired

  - `GET /api/v1/schedules` — computed `next_fire_at` at
    second-resolution means the body changes on every
    call regardless of schedule content. ETag would be
    useless.
  - `GET /api/v1/schedules/{id}/stats` — also second-
    resolution `now` field.

## Tests (+1)

  - TestListScheduleTags_ETag: 3-phase test — initial 200
    + ETag, second call with matching If-None-Match → 304
    + empty body + same ETag header, third call after
    mutation → 200 with different ETag.

## Honest scope

  - ETag is per-response (computed from full body). Not
    a `Weak ETag` — strong validation, matches RFC 7232.
  - No support for `If-Match` write-precondition headers
    (different use case; v1.78 has its own If-Match on
    schedule updates).
  - Dashboard JS doesn't yet send `If-None-Match` —
    operators get the cache headers but the dashboard's
    fetch() honors them automatically only when the
    response is in the browser cache. Explicit
    `If-None-Match` plumbing in JS deferred to vNext.
  - INSTALL.md unchanged. Build sizes unchanged.
