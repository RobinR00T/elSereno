# Snapshot — v1.76.0: named cron shortcuts

Single-chunk cycle. v1.73 shipped 5-field cron syntax; v1.76
layers Vixie-style named shortcuts on top so common cadences
are one token instead of five fields.

## Why

Operator UX. "Run my fleet scan every day at midnight" reads
naturally as `@daily` and matches what `man 5 crontab` shows.
The 5-field form (`0 0 * * *`) is surprisingly easy to
typo — extra space, swapped field order. Shortcuts cover
the common cases without preventing the full grammar.

## Mapping

```
@yearly   / @annually  →  0 0 1 1 *
@monthly                →  0 0 1 * *
@weekly                 →  0 0 * * 0  (Sunday 00:00)
@daily    / @midnight  →  0 0 * * *
@hourly                 →  0 * * * *
```

`@reboot` is intentionally NOT supported — it's a one-shot
at boot that doesn't fit the periodic schedule store
semantics.

## Implementation

`ParseCron` pre-processes:

```go
if strings.HasPrefix(expr, "@") {
    key := strings.ToLower(expr)
    expanded, ok := cronShortcuts[key]
    if !ok {
        return CronExpr{}, fmt.Errorf("%w: unknown shortcut %q",
            ErrCronInvalidField, expr)
    }
    expr = expanded
}
```

Lookup is case-insensitive. The original input string is
preserved on `c.raw`, so `c.Raw()` round-trips what the
operator typed. The dashboard renders `s.cron_expr`
verbatim, so a `@daily` schedule shows `"cron: @daily"` in
the Interval column instead of the expanded form.

Unknown `@`-prefix tokens (`@bogus`, `@reboot`, `@foo`) →
400 `ErrCronInvalidField` (same surface as a malformed
5-field expression).

## Persistence

No migration. The shortcut is stored as-is in the existing
`cron_expr` column. The 8-character DEFAULT '' constraint
from migration 00007 still applies.

## Honest scope

  - No "show me the expansion" toggle in the dashboard.
    Operators who want to see what `@daily` resolves to
    must consult INSTALL.md.
  - No Unicode case-folding — accents/non-ASCII characters
    in the shortcut are rejected.
  - No Quartz-style extras (`@every-N-minutes`).
  - Timezone (v1.75) applies to shortcut-based cron the
    same way as 5-field form. `@daily` + `America/New_York`
    fires at NY 00:00, not UTC 00:00.
