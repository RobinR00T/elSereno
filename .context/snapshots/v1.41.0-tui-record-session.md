# Snapshot — v1.41.0: tui --record session capture

Single-chunk cycle. Adds the symmetric counterpart to
v1.29-chunk-3's `--replay`: `tui --record FILE.ndjson`
tees every event the TUI's model receives onto a file as
the session runs. Operators get a durable record they can
review later, replay through other tooling, or attach to
a bug report.

## Use cases

  - **Screen-recording / training**: capture a live
    incident response session, replay the events later for
    trainees.
  - **Forensics**: persist exactly what the operator saw on
    screen during an incident.
  - **Bug reports**: attach the record file to a github
    issue when the TUI misbehaves; gives maintainers the
    exact event stream that triggered the glitch.
  - **Symmetry**: completes the record/replay loop for the
    TUI (replay was v1.29 chunk 3; capture is v1.41 chunk 1).

## Workflow

```
$ elsereno tui --replay capture.ndjson --record session.ndjson
… (operator reviews / triages — alt screen takes over)
^C
$ jq -c '. | {ts, type, line: .line // empty, score: .finding.Score // empty}' session.ndjson
{"ts":"2026-05-04T16:37:54.961Z","type":"finding","score":85}
{"ts":"2026-05-04T16:37:55.012Z","type":"audit","line":"vault unlocked"}
{"ts":"2026-05-04T16:37:55.123Z","type":"scan_progress"}
{"ts":"2026-05-04T16:37:55.500Z","type":"feed_closed"}
```

## Chunks

### Chunk 1 — recorder + RunOpts + --record flag (`0b1a5df`)

`internal/tui/recorder.go` (new):

  - **recordSchema** = "elsereno-tui-record/v1" — emitted
    on every line so consumers can dispatch.
  - **recordEvent** type — schema + ts + type +
    type-specific fields (`finding`, `line`,
    `completed`/`total`, `mode`/`err`).
  - **recorder** type wraps an `io.WriteCloser`; `Tee(next)`
    returns an emit-shim that records THEN forwards. Mutex
    so concurrent emits stay safe (production has only one
    writer goroutine but tests + future multi-feed
    compositions might overlap).
  - **Stats()** counts successful encodes.
  - **Close()** is idempotent.

`internal/tui/runner.go`:

  - **RunOpts** struct with `Record io.WriteCloser`. Anchor
    for future opts.
  - **RunWithOpts(...)** is the v1.41+ entry point.
  - **Run(...)** preserved as a back-compat shim that
    calls RunWithOpts with empty opts. Existing callers
    (incl. teatest tests) keep working — verified by the
    pre-existing 10-case program_test.go suite passing.
  - In the feed goroutine, `FeedClosedMsg` now flows
    through emit (not directly through `prog.Send`) so the
    close-event lands in the record. Without this change
    the operator's record would always end one event short.

`cmd/elsereno/cmd_tui.go`:

  - `--record FILE` flag.
  - `openRecordSink` helper: empty path → zero-RunOpts
    (recording disabled); non-empty → `os.OpenFile(...,
    O_CREATE|O_WRONLY|O_TRUNC, 0600)`.
  - `RunE` switched from `tui.Run` → `tui.RunWithOpts`.
  - `recordPath` field on `pickFeedArgs`.

### Best-effort recording

If the operator-supplied file fills the disk mid-session
or hits some other write error, the encoder errors are
silenced — the TUI never dies for an unwritable file.
`Stats()` gives the caller a "wrote N events" sanity check
if they care to surface it.
