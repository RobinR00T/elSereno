# Snapshot — v1.5.0: `elsereno proxy listen` goes live

Closed **2026-04-23**. A small-but-operationally-important
cycle: promotes the `proxy` stub (planned since F0) to a real
command that RUNS the v1.4 write-gated handlers against a local
TCP listener. Operators can now use the gates they mint
confirm-tokens for.

## Chunks

### Chunk 1 — `proxy listen` for sip/iax2/pbxhttp (`e8bd030`)
- `cmd/elsereno/cmd_proxy_offensive.go` NEW: `elsereno proxy
  listen --plugin {sip|iax2|pbxhttp}` binds to `--listen`,
  forwards to `--target`, authorises via the ADR-039 triple-
  confirm fences, runs until SIGINT / SIGTERM. Per-plugin
  allowlist flags (`--method`, `--subclass`,
  `--allow METHOD:/path`). Authorise() fires before the
  listener binds so token-mismatch surfaces immediately.
- `cmd/elsereno/cmd_offensive_register.go` MODIFIED:
  replaceProxyStubWithOffensiveCmd walks the root command,
  removes the newStubCmds() auto-registered "proxy" entry,
  and installs the real command tree. Default-build users
  still get the stub's "planned" EX_TEMPFAIL — this wiring is
  `//go:build offensive`-only.
- 7 unit tests covering per-plugin dispatch + error paths
  (unknown plugin, malformed allowlist entry, case-insensitive
  plugin name).

### Chunk 2 — extend to modbus/opcua/bacnet + E2E (`1172eae`)
- `--function`, `--service`, `--service-choice` flags added
  for the three new plugins. Numeric; range-validated (uint8
  for function / service-choice; uint16 for OPC UA service).
- `buildGatedHandler` split into six per-plugin constructors
  (buildSIPHandler, buildIAX2Handler, buildPBXHTTPHandler,
  buildModbusHandler, buildOPCUAHandler, buildBACnetHandler)
  to keep cyclomatic complexity ≤ 15 after gocyclo complained
  about the fat switch.
- **End-to-end test**: spin a fake SIP origin on 127.0.0.1:0;
  construct a sipwrite.WriteGatedHandler + mint the expected
  confirm-token; start proxy.Server.Run; dial the proxy as a
  client; send OPTIONS (origin sees it, 200 back); send INVITE
  (405 back, origin sees nothing). First full-stack integration
  test of the gated proxy — proves framework + handler +
  triple-confirm + per-request filtering all compose correctly.

## Supported `proxy listen` matrix

| Plugin  | Allowlist flag       | Refusal format                        |
|---------|----------------------|---------------------------------------|
| sip     | `--method`           | SIP/2.0 405 Method Not Allowed + Allow|
| iax2    | `--subclass`         | HANGUP full-frame                     |
| pbxhttp | `--allow`            | HTTP 405 / 403 + Allow                |
| modbus  | `--function`         | Modbus ExceptionCode                  |
| opcua   | `--service`          | OPC-UA ServiceFault                   |
| bacnet  | `--service-choice`   | BVLC-wrapped Abort-PDU                |

## Retrospective

### What went well
- The ADR-040 gated-handler pattern established across v1.2
  (opcua) + v1.4 (sip / iax2 / pbxhttp / bacnet) made the CLI
  wiring pure glue — no protocol-specific code in cmd/.
- First E2E test confirms the whole stack composes; confidence
  boost for future `proxy listen` work.

### Pitfalls hit + fixed
- `noctx` linter — `net.Listen` / `net.Dial` need their
  context-aware siblings (ListenConfig.Listen, Dialer.
  DialContext). Fixed in the E2E test.
- `gocyclo` on the fat buildGatedHandler switch. Split into
  per-plugin constructors.
- `funlen` on runProxyListen. Factored out validate…Opts
  helper.

### Carry-overs to v1.6+
- Per-NodeId allowlist for OPC UA (tighter than service-TypeID).
- Per-object allowlist for BACnet (ASN.1 BER parsing).
- SIP To-URI E.164 prefix allowlist for INVITE.
- REGISTER AOR allowlist for SIP.
- `--allow-file` for reading allowlists from YAML/JSON.
- Runtime reload of the allowlist (SIGHUP).
