# Snapshot — v1.9.0: YAML round-trip + input CLI + toll-fraud gate

Closed **2026-04-24**. Five-chunk cycle that closes long-
standing carry-overs, completes the attack-surface input
story, and ships a concrete toll-fraud mitigation for the SIP
gate.

## Chunks

### Chunk 1 — OPC UA NodeID YAML round-trip (`08ec93a`)
Closes v1.6 → v1.9 gap: the v1.6 per-NodeId gate had CLI
support in v1.7 but the YAML `--allow-file` emitter silently
dropped `node_ids:`, breaking the round-trip.

  - `proxyListenOpts.nodeIDs []string` (CLI form `ns=N;i=M`).
  - `--node-id` flag on `proxy listen`.
  - `proxyAllowFile.NodeIDs []proxyNodeID` with
    `yaml:"node_ids,omitempty"`.
  - `buildOPCUAHandler` wires opts.nodeIDs →
    `AllowedNodeIDs` via existing `parseNodeIDFlag`.
  - `buildAllowFileOPCUA` serialises NodeIDs, sorted numerically.
  - 4 tests: emit-load round-trip, empty-list compat, sort
    order, v1.6 tokens remain valid.

### Chunk 2 — Modbus proxy-session dry-run (`942f3fb`)
Closes the write-surface asymmetry: modbus had per-request
dry-run only; sip/iax2/pbxhttp/opcua/bacnet all had proxy-
session dry-run. Now 6/6.

  - NEW subcommand `write modbus proxy-dry-run`.
  - Flags: `--function` (repeatable), `--unit`,
    `--address-from`, `--address-to`.
  - Outputs: Protocol / Operation / Target / Functions /
    Unit / AddressRange / PayloadHash.
  - `--emit-allow-file` safety guard: refuses when
    `--unit`/`--address-*` are combined (YAML schema stores
    `functions:` only; emitting would silently widen the
    gate on round-trip). Error points at two workarounds.
  - RunE body extracted into 4 helpers (buildAllowlist +
    printSummary + maybeEmitAllow) to keep funlen under 60.
  - 6 tests.

### Chunk 3 — Input CLI wire-up (`a509b2e`)
Closes the operator-ergonomics gap: the 4 input clients had
no CLI surface. Now `scan --input fofa:<query>` etc. work
end-to-end.

  - `scan --input` accepts `shodan:<q>`, `censys:<q>`,
    `fofa:<q>`, `zoomeye:<q>` (+ v1.4 `list:` / `nmap:` /
    `stdin`).
  - `--api-creds-file <path.yaml>` — per-provider credentials
    in one YAML. Only the block matching the --input prefix
    is consulted.
  - Security: 0600 permission check at load time (rejects
    0644 / 0640 with clear remediation); KnownFields strict
    (typos fail at parse).
  - Rate-limited to 1 rps per provider (free-tier quota
    safety).
  - 11 tests: loadAPICreds happy + 3 perm variants + KnownFields
    strict + per-provider missing-field tests.

### Chunk 4 — ONYPHE input client (`62f8c6d`)
5th provider — French threat-intel with strong coverage of
exposed ICS equipment. Same shape as FOFA/ZoomEye but OQL
query syntax + bearer auth header.

  - `internal/inputs/onyphe/` new package.
  - `New(apiKey, ratePerSec)`, `Search(ctx, oql, page)`.
  - Query embedded in URL PATH (not query-string); the client
    `url.PathEscape`'s it.
  - `Authorization: bearer <key>` header (RFC 6750).
  - In-body error flag → error-N typed surface.
  - Wired into `scan --input onyphe:<q>` + `--api-creds-file`
    gains `onyphe.key`.
  - 5 client tests + 1 CLI integration test.

### Chunk 5 — SIP INVITE To-URI prefix allowlist (`18e91bf`)
v1.4 INVITE-allowed sessions accept ANY destination. Operators
running ElSereno as a gated carrier trunk want finer control:
allow INVITE but only to whitelisted destinations (e.g. +34 /
+44, blocking +900 premium-rate). Concrete toll-fraud
mitigation.

  - `offensive/write/sip.AllowedToURIPrefix{Prefix string}`.
  - `WriteGatedHandler.AllowedToURIPrefixes` opt-in field.
  - `AllowlistHashWithPrefixes` + `SessionMutationWithPrefixes`
    — mix both dimensions. Empty list = v1.4 hash (backwards
    compatible).
  - `canonicalisePrefix(p)` strips URI schemes (tel:, sip:,
    sips:) + whitespace + lowercases.
  - Per-request check: when INVITE + prefix list set, parse
    the To: header's URI user-part; match against sorted
    prefixes. No match → 403 with `X-Elsereno-Gate-Reason`
    header for traceability.
  - Other methods (REGISTER, MESSAGE, etc.) unaffected —
    prefix list gates INVITE ONLY (the toll-fraud vector).
  - CLI: `write sip dry-run --to-prefix +34 --to-prefix +44`
    + `proxy listen --to-prefix +34` + YAML
    `to_prefixes: ["+34", "+44"]`.
  - `forward()` refactored (per-request logic extracted to
    `forwardOne` helper) to keep gocyclo under 15.
  - 7 tests: 4 hash tests + 3 E2E gate tests (allowed prefix,
    blocked prefix, v1.4 fallback, REGISTER unaffected).

## Retrospective

### What went well
- 4 of 5 chunks follow established patterns (v1.6/v1.7/v1.8
  templates copied verbatim — hash + mutation + backwards-
  compat empty check, CLI + YAML round-trip, tests).
- The SIP prefix gate is opt-in + backwards-compat: operators
  with v1.4 tokens don't need to re-mint anything unless they
  explicitly turn on prefix gating.
- Security-sensitive 0600 perm check on --api-creds-file
  prevents the classic "oh I chmod'd 644 by default" footgun.

### Pitfalls hit
- SIP `forward()` cyc 18 > 15 after adding the prefix check.
  Fixed by extracting `forwardOne(line, br, tp, ...)` helper.
- Modbus `--emit-allow-file` + `--unit`/`--address-*`
  combination would have silently produced a YAML that
  widens the gate on round-trip (unit + address-range not
  yet in proxyAllowFile schema). Fixed with a refuse-at-emit
  guard + clear error message pointing at workarounds.
- Slice-of-[32]byte typecheck error (unaddressable value).
  Fixed by binding hash return to a variable before slicing.
- `TestReadTargetsFromProvider_UnknownProvider` used "onyphe"
  as the "unknown" name — onyphe became known in chunk 4.
  Fixed: renamed test to use "binaryedge" as the unknown.

### Carry-overs to v1.10+
- Per-NodeId per-WriteValue (OPC UA multi-node in one
  WriteRequest) — chunk 1 only checks the first WriteValue.
- OPC UA String/Guid/ByteString NodeID encoding — chunk 1 is
  numeric-only.
- OPC UA CallRequest per-object allowlist.
- BACnet per-object allowlist (ASN.1 BER parsing — different
  shape from OPC UA numeric NodeIDs).
- Modbus `writes:` structured YAML with full (unit, fc, start,
  end) tuple — closes chunk 2's `--unit`/`--address-*` +
  `--emit-allow-file` incompat.
- Input client pagination (chunks 3-4 fetch page 1 only,
  100-hit window).
- SIP REGISTER AOR allowlist (sister to chunk 5's INVITE
  prefix list; blocks registration hijack).
- CWMP offensive proxy (SOAP RPC allowlist).
- SIGHUP reload of proxy listen allowlist (needs
  PayloadHash/token-invalidation redesign).
