# Snapshot — v1.56.0: M-Bus offensive write-gated proxy

Single-chunk cycle. Closes the v1.32+ D2 carryover.
Adds the offensive TCP-aware write-gated proxy for
M-Bus over TCP/10001. M-Bus is the European metering
protocol (EN 13757-3 + EN 13757-4) covering
electricity, gas, water, heat, and heat-cost-allocator
meters. SND_UD writes mutate physical meter state.

## Why

The default mbustcp plugin (v1.21 chunk 2) sends
REQ_UD2 + parses the RSP_UD response and refuses every
offensive byte. v1.56 adds the offensive replacement
that lets operators authorise specific (CI, address)
SND_UD operations against specific meters without
exposing the full attack surface (re-tariff, baud
reconfigure, primary-address reassign, firmware
push, sync action).

## Two-tier gate

```
control field → (CI, address) tuple
   |                  |
   |                  └── refuse if no AllowedSNDUD entry
   |                      matches; wildcards: CI=0 any CI,
   |                      Addr=0 any address
   |
   └── always-safe: SND_NKE, REQ_UD1, REQ_UD2, ACK.
       SND_UD requires per-(CI, addr) allowance.
       Other / unknown control bytes refuse.
```

Refusal mode: **silent drop**. M-Bus has no
"permission denied" frame; the client retransmits a
couple of times and surfaces a clean timeout.

## Chunks

### Chunk 1 — wire control + gate (`c7820ca`)

`internal/protocols/mbustcp/wire/control.go` (new):

  - **Control field constants**: SND_NKE (0x40),
    SND_UD (0x53/0x73), REQ_UD1 (0x5A/0x7A),
    REQ_UD2FCB (0x7B). REQ_UD2 itself (0x5B) was
    already in wire.go.
  - **CI byte constants**: CIAppReset (0x50),
    CIDataSend (0x51), CISelectSlave (0x52),
    CISyncAction (0x55), CISetBaud range
    (0x56..0x5D).
  - **Frame** struct: Raw + classification
    (IsACK/IsShort/IsLong) + Control + Address + CI.
  - **ReadFrame(r)** stream parser: reads one
    complete M-Bus frame off a TCP stream. Handles
    short (5-byte 0x10..0x16) + long (0x68 L L 0x68
    body CS 0x16) + single-byte ACK (0xE5). Bound-
    checks each piece + checksum.
  - **Frame.IsSNDUD()** + **Frame.IsAlwaysSafeControl()**
    classify each parsed frame.
  - 1 new sentinel: ErrFrameUnknownStart (the
    existing ErrShortFrame, ErrBadStart, etc. are
    reused).

`offensive/write/mbustcp/gatedproxy.go` (new, ~270 lines):

  - **AllowedSNDUD{CI, Address}** with `Matches(f)`
    semantics: empty struct matches nothing
    (guards against {0,0}-as-wildcard typos), CI=0
    wildcards any CI, Address=0 wildcards any
    address.
  - **AllowlistHash** with target prefix + sorted
    (CI, Address) tuples. Mirror the modbus/iax2
    template — no new separator since there's only
    one allowlist dimension beyond control.
  - **SessionMutationWithGeneration** for v1.17-style
    cookie.
  - **WriteGatedHandler** with TCP-stream forward
    via mbwire.ReadFrame; refused frames drop
    silently.

## Multi-frame stream behaviour

Each call to ReadFrame consumes exactly one M-Bus
frame off the client stream. The forward loop
processes frames in order; an allowed frame is
written verbatim to upstream, a refused frame is
dropped (the loop continues to the next read). A
malformed frame (any structural error) terminates
the session — the gate doesn't try to recover from
desync.

## Honest scope

  - **Per-CI argument parsing** NOT in this chunk.
    Set-baudrate (CI 0x56..0x5D) leaks the baudrate
    in the CI byte itself, but Data Send (0x51)
    carries parameter ID + value in the UD payload.
    Per-DIB parsing for fine-grained "allow tariff
    write but not address reassign" is a future
    cycle.
  - **Secondary-addressing flow** NOT tracked. CI
    0x52 (Select Slave) + subsequent SND_UD pair —
    each frame is gated independently. A future
    cycle could track "currently selected slave"
    state.
  - **CLI flag plumbing** NOT in this chunk.
    Wire + gate primitives are the foundation;
    flags in cmd_write_offensive.go are a future
    cycle's add (same pattern as v1.52/v1.53/v1.55).
  - **Lab validation against real Kamstrup / Elster
    / Sensus M-Bus over TCP gateway** pending.
    Confidence ~0.85 until validated.
