# Snapshot — v1.40.0: plugins ports reverse-index verb

Single-chunk cycle. Adds `elsereno plugins ports` so
operators can map a port number back to the plugin(s) that
claim it — the inverse of the existing `plugins list`
output.

## Why

Two recurring ops scenarios:

  1. **Discovery-then-fingerprint** — operator runs
     `discover --auto 192.168.1.0/24` (or `--hosts file`
     since v1.39), sees a hit on port 4059, and wants to
     know which `--plugin` value to pass to
     `scan` / `fingerprint validate`. Today they grep
     `plugins list` output. v1.40 → `plugins ports | grep
     4059` is shorter + can be JSON-piped into jq.

  2. **External tooling integration** — operators running
     elsereno alongside nmap want a port→plugin lookup
     table for filtering pipelines. JSON output is the
     ergonomic shape.

## Workflow

```
$ elsereno plugins ports
   80  [pbxhttp]
  102  [mms s7]                ← shared port (v1.25 disambig)
  443  [pbxhttp]
  502  [modbus]
  789  [redlion]
  …
47808  [bacnet]

$ elsereno plugins ports --json | jq '.["102"]'
[
  "mms",
  "s7"
]
```

## Chunks

### Chunk 1 — plugins ports verb (`6cccb2c`)

`cmd/elsereno/cmd_plugins.go`:

  - **newPluginsListCmd** extracted from the inline cobra
    literal (keeps newPluginsCmd tidy as sub-verbs grow).
  - **newPluginsPortsCmd** with `--json` flag.
  - **buildPluginsByPort** helper — testable without
    cobra. Produces a `map[int][]string` keyed by port,
    values sorted alphabetically.

Plugins with `DefaultPort: 0` (atmodem — transport-
agnostic; banner — generic raw-banner probe) are skipped:
they can't go in a port-keyed reverse index.
