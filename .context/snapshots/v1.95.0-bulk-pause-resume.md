# Snapshot — v1.95.0: bulk pause/resume

Single-chunk cycle. Operators doing planned maintenance
(DB migration, network change, etc.) need to pause ALL
schedules in one click. Before v1.95 they iterated per-row
"Disable" → tedious + error-prone (forget one schedule →
unwanted scan during the window).

## REST

```
POST /api/v1/schedules/bulk/disable
POST /api/v1/schedules/bulk/enable
```

No request body required. Returns:

```json
{
  "schema": "api:v1",
  "data": {
    "affected": 47,
    "failed_audits": 0,
    "target_state": false
  }
}
```

`affected` counts only schedules whose state actually
changed (already-disabled schedules are no-ops on
bulk/disable). This keeps the audit log meaningful — only
genuine state transitions get rows.

`X-Schedule-Audit-Warning` header set when one or more
per-schedule audit rows failed to persist (best-effort,
matches v1.84 pattern).

## Audit

One audit row per state transition, event_type =
`set_enabled_false` or `set_enabled_true`. Operator =
the bulk-op invoker (X-Operator header). Compliance
review can grep the audit log to see who did the
pause/resume + when.

## Dashboard

Two buttons above the schedule table: "Pause All" +
"Resume All". Confirm dialog explains the audit
implication. Status line shows affected count + any
audit failures. Re-renders the table after the call.

## Tests (+1, mid-detail)

`TestBulkSetEnabled_DisableAll`: 2 enabled + 1
pre-disabled schedule → bulk-disable affects 2; the
already-disabled one is a no-op (no audit row);
post-conditions verified per-schedule.

## Honest scope

  - No partial-failure recovery. If SetEnabled fails for
    one schedule mid-loop, the bulk-op continues; the
    operator can re-issue.
  - No pause-list-by-ID variant. "Pause these 3 specific
    schedules" is N HTTP calls. The use case is "all" or
    "none"; a labelled filter would need design.
  - Pause/resume is one-way per call (no atomic restore
    of pre-pause state). If some schedules were disabled
    before "Pause All", "Resume All" enables them too.
    Operators tracking the original state should snapshot
    via List + GET before pausing.
  - INSTALL.md unchanged. Build sizes unchanged.

## Build sizes

Unchanged across all 3 variants.

Adds ~140 LoC + 1 test.
