# Snapshot — v1.34.0: tree-wide //nolint:gosec → // #nosec G<NNN> sweep

Hygiene-only single-chunk cycle. Completes the b611f5c
(pre-v1.28) marker migration across the entire tree.

## Why a tiny cycle

PITF-030 + b611f5c established that the standalone gosec
binary in CI's `sec` job only honours same-line `// #nosec
G<NNN>` directives, not line-above `//nolint:gosec` ones.
b611f5c swapped 18 markers; v1.30 chunk 2 swapped 2 more;
v1.32 chunk 1 swapped 10 in cmd/elsereno/. v1.34 finishes
the job: 76 markers across 49 files in internal/**,
offensive/**.

## Chunks

### Chunk 1 — Tree-wide sweep (`75cbcf5`)

Method:

  - **58 markers with explicit G-codes** (G115, G304, G306,
    G204, G306/G703 composite): bulk-swapped via perl regex.
    The pattern preserved the rationale text and only
    changed the directive shape:
    ```
    //nolint:gosec // G<NNN> — text
    →
    // #nosec G<NNN> -- text
    ```
  - **18 markers without explicit G-codes** (test-bounded
    integer truncations): all G115 by content. Bulk-swapped
    to `// #nosec G115 -- <rationale>`.
  - **1 composite case** (`internal/audit/sync_test.go`:
    G306+G703): fixed manually to the gosec multi-code
    syntax `// #nosec G306,G703`.

### Side fix

`offensive/write/enip/write.go` line 148 had a pre-existing
comment-eats-statement bug:

```go
binary.LittleEndian.PutUint16(buf[2:4], uint16(len(body))) //nolint:gosec	buf = append(buf, body...)
```

The `//nolint:gosec` directive was on the same physical line
as a tabbed `buf = append(buf, body...)` statement. Go's `//`
line comment swallowed the `buf = append`, so the length-
rewrite was happening but the body wasn't being appended
back — silently breaking the encapsulation of any frame
where MarshalHeader had already written the header.

Tests `TestBuild_SetAttributeSingle` and `TestBuild_Reset`
had been passing despite this because they only checked for
the presence of the service byte (`0x10` / `0x05`), which
sat in the MR portion that was already in `buf` from
`MarshalHeader`. The body append (containing the SendRRData
encapsulation) was being dropped silently.

Fix: split into two lines + restored the `buf = append`. The
existing two test cases now exercise the intended path. A
larger end-to-end test that decodes the encapsulation would
have caught this; defer to a future cycle.

## PITF-030 enforcement

With this cycle the convention is fully enforced. Any
contribution that reaches for `//nolint:gosec` will fail
`make sec` (the standalone gosec binary doesn't honour
line-above directives), giving the forcing function we
wanted. The `internal/exec/doc.go` reference to the old
form is intentionally untouched — it's prose inside a
package docstring describing the historical convention,
not an active suppression directive.
