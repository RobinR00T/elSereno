# Snapshot — v2.15.0: dashboard If-None-Match plumbing

Single-chunk cycle. Closes v2.7 carryover. v2.7 added
server-side ETag support; the browser fetch() honors
ETags only when the response was already cached, which
doesn't apply to `Cache-Control: must-revalidate`
+ AJAX fetches. v2.15 plumbs `If-None-Match` explicitly.

## Plumbing

  - Module-level `etagCache = {}` map (URL → ETag string).
  - `fetchWithETag(url)` helper:
    - Sets `If-None-Match` header from cache.
    - On 304: returns `{notModified: true, etag, status: 304}`.
    - On 200: stores the new ETag + returns `{notModified:
      false, etag, status: 200, json: <body>}`.
    - On 4xx/5xx: throws with body in error message.
  - Exposed on `window.fetchWithETag` so external handlers
    can reuse.

## Wired

  - `refreshScheduleTagCloud` (polled every 30s by
    renderSchedules) → 304 path skips body re-render but
    calls `repaintTagCloudHighlight` so filter-state
    transitions are still applied.

## Honest scope

  - Only `/schedules/tags` plumbed in v2.15. `/audit` +
    `/runs` open on-demand (not polled), so ETag wins
    are smaller — deferred.
  - Cache is per-page-load (no localStorage). Page
    refresh clears.
  - Don't expect huge bandwidth savings — JSON bodies are
    small. Real win is when many schedules are tagged +
    /tags response gets large.

## Build sizes

Unchanged across all 3 variants.

Adds ~60 LoC (helper + repaint) in dashboard.go.

## Gotcha resolved

JS comment text "Pass `expectJSON: false` for ..."
contained backticks → terminated the surrounding Go
raw-string literal → syntax error. Fixed by rewriting
the comment to avoid backticks. Pitfall worth promoting
(see playbook).
