# Snapshot — v1.13.0: BACnet completion + CWMP polish

Closed **2026-04-26**. Thirteen-chunk cycle that closes every
BACnet mutating service (svc 7/8/9/10/11/15/16/17/20/27) with
wire-level per-target-or-state allowlists, plus CWMP polish +
operator UX improvements (InternetDB bulk lookup, firmware
pre-flight verifier, RPC case-warning, CWMP-over-TLS recipe,
triage utility bucket).

## Why this cycle matters

v1.12 built **depth** for the per-RPC scoping pattern: every
write-gated proxy got per-object / per-path scoping, every
input client paginated. v1.13 finishes the BACnet leg of that
work — the v1.12 chunk-7 design (per-property AllowedObjects
for WriteProperty) generalised to every other BACnet
destructive service:

| svc | Service                       | Chunk | Granularity                |
|-----|-------------------------------|-------|----------------------------|
| 7   | AtomicWriteFile               | 12    | per-File-instance          |
| 8   | AddListElement                | 13    | per-(object, property)     |
| 9   | RemoveListElement             | 13    | per-(object, property) †   |
| 10  | CreateObject                  | 8     | per-type (instance ignored)|
| 11  | DeleteObject                  | 7     | per-(type, instance)       |
| 15  | WriteProperty                 | v1.12 | per-(type, instance, property)|
| 16  | WritePropertyMultiple         | 3     | per-(type, instance, property) — every tuple in batch|
| 17  | DeviceCommunicationControl    | 10    | per-state (enableDisable enum)|
| 20  | ReinitializeDevice            | 9     | per-state (state-of-device enum)|
| 27  | LifeSafetyOperation           | 11    | per-operation (10-value enum)|

† svc 8 + svc 9 share the AllowedListElements list (request
shapes are identical).

The hash ladder convention (each new dimension degrades to the
prior-version hash when the new field is empty) preserves
operator tokens across the upgrade — operators who don't opt
into per-target scoping for a given service keep their v1.4–
v1.12 confirm-tokens valid.

### 1 `781ee50` — InternetDB bulk lookup

v1.12 chunk 9 shipped single-IP InternetDB lookups; chunk 1
adds the bulk forms `internetdb:file:<path>` and
`internetdb:-` (stdin). Rate-limit clamps to upstream's ~10 rps.

### 2 `781ee50` — CWMP firmware pre-flight verifier

`elsereno-offensive write cwmp verify-firmware --firmware
url=…;sha256=…` resolves the URL via HEAD → GET, computes
SHA-256 over the body, compares against the operator-supplied
pin. Output OK / MISMATCH / UNREACHABLE. Catches typos, stale
pins, supply-chain swaps before the operator commits the
firmware allowlist.

### 3 `38dedff` — BACnet WritePropertyMultiple per-object

WPM has nested SEQUENCE-OF-WriteAccessSpecification with
constructed BER values (BACnetWeeklySchedule, …) — the walker
is depth-aware via `skipUntilDepthZero` / `skipOneTagBody`.
ANY single forbidden tuple refuses the WHOLE WPM batch (fail-
closed multi-target gate, analogous to the OPC UA
WriteRequest walker). Uses context tag 0 for inner PropertyId
(vs tag 1 for the v1.12 WriteProperty path).

### 4 + 5 `861aa8d` — CWMP RPC case-warning + over-TLS recipe

- TR-069 §A.4 declares RPC names case-sensitive.
  `emitCWMPRPCCaseWarnings` walks the operator's `--rpc` list
  against the canonical 24-name set and warns on case
  mismatches. Gate stays case-sensitive; warning is UX-only.
- CWMP-over-TLS operator recipe: three front-proxy patterns
  documented (nginx `stream`, HAProxy `mode tcp`, Caddy
  `layer4`). Each terminates TLS on `:7548` and forwards
  plaintext to the gate on `:7547`. Gate stays TLS-agnostic.

### 6 `20f6215` — Triage utility bucket

Fourth priority bucket between `strategic` and `routine`.
Heuristic: severity ∈ {info, low} AND (Protocol ∈ {banner,
atmodem} OR `impact_class < 20`). Catches old SSH banners,
HTTP-HEAD `Server: nginx/1.10`, AT modem chatter — useful
fingerprint info that isn't a direct nail.

### 7 `934c4f7` — BACnet DeleteObject per-target

`AllowedDeleteObjects` is a SEPARATE list from
`AllowedObjects` so the typical BAS pattern "writes ok,
delete forbidden" is naturally expressible. An operator who
allowed `--object type=2;instance=99;property=85` MUST add
`--delete-object type=2;instance=99` to permit deletion.

### 8 `3f570e3` — BACnet CreateObject per-type

Type-only matching (instance ignored even when the [1]
objectIdentifier choice form encodes one) — most CreateObject
calls use the [0] objectType form where the device picks the
instance, so per-instance gating wouldn't be useful in
practice. The typical BAS use-case is "operator may create
new Schedule (type 17) objects on this device" — type-level
allowlist matches naturally.

### 9 `b51f488` — BACnet ReinitializeDevice per-state

Per-state allowlist for the 8-value reinitializedStateOfDevice
enum (0 coldstart, 1 warmstart, 2..6 backup/restore, 7
activate-changes). Typical operator pattern during a
maintenance window: allow only state 7, refuse 0..6.

### 10 `14a7451` — BACnet DeviceCommunicationControl per-state

Per-state allowlist for the 3-value enableDisable enum (0
enable, 1 disable, 2 disableInitiation). Typical operator
pattern: allow only state 0 (recovery direction) and refuse
1/2 to prevent device silencing. Chunk 10 also introduced
the `Allowlists` bundle struct (was `BACnetAllowlists` before
revive caught the package stutter) so chunk-10+ hash +
mutation factories don't grow function parameters every
cycle.

### 11 `6a10a70` — BACnet LifeSafetyOperation per-operation

The most safety-critical BACnet service: silencing a fire-
alarm panel during an active incident can be lethal. Per-
operation allowlist for the 10-value
BACnetLifeSafetyOperation enum: 1/2/3 silence variants
HOSTILE, 4/5/6 reset variants operationally significant,
7/8/9 unsilence variants SAFE. Operators on production
life-safety buses typically allow 7/8/9 freely + 4/5/6
case-by-case + REFUSE 1/2/3 outright. Wire parser introduced
a generic `skipContextPrimitiveField` helper that handles
both inline-length (0..4) and extended-length (low-bits == 5)
forms uniformly.

### 12 `830ce02` — BACnet AtomicWriteFile per-File-instance

The fileIdentifier in AtomicWriteFile MUST be a File object
(ObjectType 10) per ASHRAE 135 §15.8 — gate fails closed on
any other type. Typical pattern: when File#1 is firmware blob
and File#5 is rotating log file, allow `--awf-file 5` to
permit log overwrites + REFUSE firmware overwrites. Note
that fileIdentifier uses APPLICATION tag 12 (`0xC4`) — NOT
the context-tagged `0x0C` form used elsewhere. Refactor
introduced 5 per-plugin flag-registration helpers from
`newProxyListenCmd` (which had grown over funlen).

### 13 `5952c55` — BACnet Add/RemoveListElement per-(object, property)

**Closes every BACnet mutating service.** AddListElement (svc
8) and RemoveListElement (svc 9) share the IDENTICAL request
shape per ASHRAE 135 §15.1 + §15.2 — `[0] objectIdentifier`,
`[1] propertyIdentifier`, ... — which is exactly the
WriteProperty prefix. Gate reuses `wire.ParseWriteProperty`
(no separate parser needed). Same allowlist applies to BOTH
services (operators wanting different policy must omit one
from `--service-choice`). Common targets:
NotificationClass#N.recipient_list (102), Schedule#N.
exception_schedule (38), access-zone occupant lists.
Refactor split `objectListGatesAllow` →
`propertyTupleGatesAllow` (svc 8/9/15/16) +
`objectIdentityGatesAllow` (svc 7/10/11) for gocyclo;
`buildAllowFileBACnet` now takes a struct of inputs (was 9
positional args).
