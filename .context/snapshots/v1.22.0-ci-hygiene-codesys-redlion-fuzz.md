# Snapshot — v1.22.0: CI hygiene + CoDeSys + Red Lion + fuzz

Four chunks: one CI-hygiene fix, two new fingerprint plugins,
and a cross-cutting fuzz-coverage push that surfaced a real
bug in the v1.20 chunk 1 trimASCII implementation. Default-
build plugin count climbs from **23 to 25**; in-tree Fuzz*
target count doubles from **6 to 14**.

## Chunk 1 — CI hygiene: fuzz-flake retry + explicit timeout

`(commit e1e3f25)` Closes a pre-existing intermittent failure
where `xot/wire FuzzParseHeader` and `render FuzzSafeBytes`
would report "context deadline exceeded" at the end of their
fuzztime budget on macOS. Go's fuzz worker scheduling
occasionally races GC and GOMAXPROCS pressure and reports a
deadline even though no real fuzz crash occurred — the 30s
elapsed time matches fuzztime exactly, and the failure
surfaces only when 12 workers are saturating cores under CI
load.

`scripts/run-fuzz.sh` updates: explicit `-timeout` 4× the
fuzz duration (capped at 60s minimum), and per-target retry
loop with MAX_ATTEMPTS=3. Genuine fuzz failures (real
panics / coverage failures) short-circuit the retry loop —
only "context deadline exceeded" without a `--- FAIL:` panic
line is retry-eligible.

## Chunk 2 — CoDeSys V3 TCP fingerprint plugin

`(commit 5aedb05)` TCP/1217. Sends the 4-byte BlockDriver
magic hello (0xCD 0xCD 0xCD 0xCD) and classifies the response
by either the magic echo or canonical banner substring
(CoDeSys / CODESYS / 3S-Smart / 3S-CoDeSys / CmpHostname /
CmpAppBP / CmpRuntime). Reverse-engineered from libcodesys-py
+ codesys-rs + ICS-CERT advisory captures (ICSA-12-242-01 /
19-080-01 / 21-014-04). Fail-closed proxy.

Score factors{protocol_risk:80, exposure:75, auth_state:85,
capability:30→70 on CoDeSys reply, impact_class:75,
**cve_exposure:10**}. **First plugin to set cve_exposure
non-zero by default** — reflects the well-known CVE family
across the CoDeSys runtime.

6 wire tests + 7 plugin tests.

## Chunk 3 — Red Lion Crimson / RLN fingerprint plugin

`(commit 0f474e9)` TCP/789. Banner-substring fingerprint for
Red Lion's G3 / Graphite / FlexEdge / DA-50N HMI families and
the post-2010-acquisition Sixnet RTU line. Plugin connects,
reads any unsolicited banner (Crimson firmware typically
announces itself on connect), falls back to a 3-byte zero
hello if nothing arrives in IOTimeout/2, then classifies by 12
canonical banner substrings ordered most-specific first.
Fail-closed proxy.

Score factors{protocol_risk:75, exposure:75, auth_state:85,
capability:30→70 on Red Lion reply, impact_class:70,
cve_exposure:5}. cve_exposure 5 reflects ICSA-21-103-01
(hardcoded crypto key) + ICSA-22-088-01 (path traversal).

6 wire tests + 7 plugin tests.

## Chunk 4 — Fuzz coverage + finsudp trimASCII bug fix

`(commit 03683a4)` Added Fuzz* targets to all 8 new wire
packages shipped since v1.20: finsudp, slmp, gesrtp, knxip,
mbustcp, dlms, codesys, redlion. Each package gains:
- a parser fuzz (no panic + per-package contract on success
  — trailing-NUL/space absence, hint-prefix conformance,
  wrapper-length consistency)
- a builder fuzz (frame-length stability)

Doubles the in-tree Fuzz* count from 6 to 14.

**Fuzz found a real bug in finsudp's trimASCII**: the
order-dependent two-call form leaves trailing NULs when
padding interleaves NUL and space. Exact duplicate of the
slmp bug fixed in v1.21 chunk 2 but shipped earlier (v1.20
chunk 1) and missed at the time. Fixed with the single-pass
cutset; regression seed preserved at
testdata/fuzz/FuzzParseControllerDataRead/.
