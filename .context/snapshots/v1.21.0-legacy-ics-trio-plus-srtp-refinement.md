# Snapshot — v1.21.0: legacy ICS trio + GE-SRTP refinement

Four chunks continuing the legacy-ICS fingerprint roll-out from
v1.20. Three new read-only plugins (KNXnet/IP, M-Bus over TCP,
DLMS/COSEM) plus a cross-cutting refinement of the GE-SRTP plugin
shipped in v1.20 chunk 3. Default-build plugin count climbs from
**20 to 23**.

## Chunk 1 — KNXnet/IP UDP fingerprint plugin

`(commit 9cd8700)` UDP/3671. Sends a 14-byte DESCRIPTION_REQUEST
(service type 0x0204) with an anonymous control HPAI (0.0.0.0:0)
per KNX Standard 03.08.02 §2.2 and folds the parsed 30-byte ASCII
friendly name + KNX Medium + Individual Address into the finding
hash. Validates header bytes 0x06 0x10, service type 0x0205,
TotalLength consistency, device-info DIB type 0x01 at offset 7;
trims NUL/space padding off the friendly-name field with a
single-pass cutset.

ProxyHandler is fail-closed (UDP). Score factors{protocol_risk:75,
exposure:75, auth_state:90, capability:30→75 on KNX reply,
impact_class:70, cve_exposure:0}. protocol_risk 75 (vs 80 for
industrial PLCs) reflects BAS focus; auth_state 90 (vs 95) because
KNX/IP Secure exists as an optional 2018+ layer.

9 wire tests + 7 plugin tests.

## Chunk 2 — M-Bus over TCP fingerprint plugin

`(commit 1f0f75b)` TCP/10001. Sends the 5-byte REQ_UD2 short frame
to broadcast primary address 0xFE per EN 13757-3 §5.2 and folds
the parsed manufacturer code + medium byte from the RSP_UD long-
frame response into the finding hash. ACK-only responses (single
byte 0xE5) also count as positive identification.

Wire validates 0x68/L/L/0x68 long-frame header (with L1==L2 echo),
total length (= 6 + declaredLen), 0x16 stop, sum-mod-256 checksum,
and CI=0x72 variable-data-response. Decodes the BCD ID (LE) and
the 16-bit-packed manufacturer code (3 ASCII letters per
`M = (c1-'A'+1)*32^2 + (c2-'A'+1)*32 + (c3-'A'+1)`, so 0x0442="ABB",
0x2C2D="KAM" Kamstrup, 0x1593="ELS" Elster, 0x4D2D="SEN" Sensus).

ProxyHandler is wire-layer write-ban: reads the request, replies
with single-byte ACK (0xE5). Matches the protocol's own
link-layer ACK idiom — meter says "got it" but no data returned.

Score factors{protocol_risk:70, exposure:70, auth_state:90,
capability:30→70 on M-Bus reply, impact_class:60, cve_exposure:0}.
impact_class 60 reflects billing accuracy + privacy-of-consumption-
data blast radius (vs PLCs' direct kinetic effects).

11 wire tests + 9 plugin tests.

## Chunk 3 — DLMS/COSEM TCP fingerprint plugin

`(commit 86e1034)` TCP/4059. Sends a 37-byte DLMS-wrapper-framed
AARQ probe (8-byte wrapper + 29-byte canonical minimal AARQ APDU
referencing OID 2.16.756.5.8.1.1 LN-no-ciphering) per IEC 62056-46
§8.4 + IEC 62056-53. Classifies by wrapper version (0x0001) +
AARE tag (0x61). Wrapper-only responses also count as positive
identification — the server speaks DLMS-wrapper but rejected our
AARQ (common with HLS-locked deployments).

ProxyHandler is wire-layer write-ban: reads the wrapper header +
APDU body (capped at 8192 to defuse oversized-length attacks),
replies with a 16-byte wrapper-framed AARE rejected-permanent.

Score factors{protocol_risk:75, exposure:70, auth_state:85,
capability:30→70 on DLMS reply, impact_class:65, cve_exposure:0}.
auth_state 85 (vs 90 for M-Bus): DLMS supports HLS auth but unauth
probes still elicit AARE responses during negotiation.
impact_class 65 reflects the remote-disconnect-breaker dimension
(via ACTION-Request on disconnect-control object).

7 wire tests + 9 plugin tests.

## Chunk 4 — GE-SRTP model-hint extraction refinement

`(commit 3edaad1)` Refines the v1.20 chunk 3 connection-init-only
fingerprint by scanning the 56-byte mailbox response payload for
embedded GE PLC family strings (PACSystems / IC693 / IC695 / IC697
/ IC200 / RX3i / RX7i). When a hint is found, it folds into the
finding note and lifts capability from 70 to 75 — parity with
finsudp / slmp on positive-id capability.

The lift comes from a low-risk decoder pass on the existing
response — no new round-trip, no risky service-0x21 follow-up that
would require test vectors against real PLCs. Service 0x21
probing remains deferred.

`wire.ExtractModelHint`: greedy-matches printable-ASCII runs
(letters / digits / dash / underscore) starting on uppercase
boundaries; rejects runs shorter than 5 chars and runs without a
canonical-prefix match. Plugin's `buildFinding` now takes a
`modelHint string` parameter — non-empty hint → capability 75,
otherwise 70.

7 new wire tests + 1 new plugin test.

## Operator surface at end of v1.21

- 3 new fingerprint plugins on the default build:
  `elsereno scan --plugin knxip` (BAS — KNX gateways, IP-routers),
  `--plugin mbustcp` (smart meters — water/gas/heat/electricity),
  `--plugin dlms` (smart meters — DLMS/COSEM IEC 62056).
- Default build registers **23 protocol plugins** (was 20).
- gesrtp now lifts capability 70→75 when an embedded GE PLC family
  hint is recoverable from the connection-init reply.

## Deferred to v1.22+

- macOS sandbox via `sandbox_init(3)`.
- 6 remaining legacy ICS protocols (PROFINET DCP /
  GOOSE / SV — Layer-2 only, framework requires IP rework;
  CoDeSys, Red Lion — proprietary, sketchy public docs without
  test vectors; IEC 61850 MMS — port-102 share with S7 +
  complex ASN.1 stack; OPC UA HTTPS).
- Offensive plugins for the v1.20 + v1.21 fingerprint trios:
  FINS / SLMP / SRTP / KNX / M-Bus / DLMS write services. Each
  needs the per-target gating pattern + triple-confirm + audit-
  chain emission per ADR-009.
- GE-SRTP service-0x21 (Read PLC Long Status) follow-up probe —
  would yield richer firmware-version + CPU-type info than the
  current connection-init-only hint extractor.
- Big-picture: TUI (bubbletea), Windows support, OIDC + roles,
  record-&-replay proxy sessions.
