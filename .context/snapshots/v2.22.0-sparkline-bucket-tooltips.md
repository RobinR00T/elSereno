# Snapshot — v2.22.0: sparkline bucket tooltips

Single-chunk cycle. v2.13 sparkline only had per-polyline
`<title>` showing the series max. v2.22 adds invisible
hit-target rects per bucket so hovering anywhere in a
bucket's vertical slice reveals the full bucket data.

## What changed

`renderSparklineSVG` now appends one `<rect fill="transparent">`
per bucket, sized to fill the bucket's column. Each rect
carries a `<title>` element with:

```
<bucket-start-local-time> — runs:N failed:M findings:K
```

Polyline `<title>`s (series max) preserved for legacy hover.

## Honest scope

  - No click-to-drill into /runs. Backend would need
    `?since=`/`?until=` filters; deferred to a follow-up
    cycle. The tooltip alone covers the "what happened on
    that day" use case.
  - Tooltips render via browser-native SVG title — no
    custom positioning. Renders as the OS default tooltip
    style (acceptable for v2.22).

## Build sizes

Unchanged across all 3 variants.

Adds ~25 LoC in dashboard.go.

## Carryover

  - Click-to-drill: extend /runs with `?since=&until=`,
    wire a clickable rect to open a pre-filtered runs view.
