# Snapshot — v1.42.0: replay/record round-trip

Single-chunk cycle. Closes the v1.41-chunk-1 carryover by
extending `feeds.Replay` to read the
`elsereno-tui-record/v1` schema that v1.41 emits. The
record/replay loop is now closed end-to-end.

## Workflow (closed loop)

```
$ elsereno scan --output-format ndjson > scan.ndjson    # ndjson:v1
$ elsereno tui --replay scan.ndjson --record session.ndjson
^C                                                         # session ends
# session.ndjson is now elsereno-tui-record/v1 with
# findings + close events the operator saw.

$ elsereno tui --replay session.ndjson    # ← v1.42 makes this work
```

## Chunks

### Chunk 1 — schema dispatcher (`e42cb86`)

`internal/tui/feeds/ndjson_stream.go`:

  - **schemaPeek** struct (just `schema` field) for cheap
    first-pass classification.
  - **tuiRecordRecord** struct mirrors `recordEvent` from
    `internal/tui/recorder.go`. Kept dedicated to the
    feeds package to avoid an import cycle (recorder
    imports tui types, feeds imports them too — going
    through internal/tui/recorder for the decode would
    cycle).
  - **parseRecord** restructured into a 2-step:
    1. Decode `schema` field via schemaPeek
    2. Dispatch on schema → parseScanFinding (legacy) or
       parseTUIRecord (new) or AuditMsg fallback for
       unknown schemas.
  - **parseScanFinding** extracted from the original
    parseRecord — handles `ndjson:v1`.
  - **parseTUIRecord** new — handles
    `elsereno-tui-record/v1` with type-tagged dispatch.

Per-type mapping for elsereno-tui-record/v1:

| `type`         | Resulting `tea.Msg`           |
|----------------|-------------------------------|
| `finding`      | `tui.FindingMsg{Finding: ...}`|
| `audit`        | `tui.AuditMsg{Line: ...}`     |
| `scan_progress`| `tui.ScanProgressMsg{c, t}`   |
| `feed_closed`  | `tui.FeedClosedMsg{Mode,Err}` |
| _other_        | `tui.AuditMsg` with "unknown type" hint (forward-compat) |

Other-schema fallback emits a friendly "unknown schema
%q (want %q or %q)" so operators see exactly what was
skipped + which schemas the replayer accepts.
