# Snapshot — v1.44.0: proxy replay --since/--until time-window

Single-chunk forensics-DX cycle. Adds time-window filtering
to `elsereno proxy replay` so operators investigating an
incident at a known timestamp can isolate the relevant
chunks without piping through grep / jq.

## Workflow

```
$ elsereno proxy replay session.ndjson \
    --since 2026-05-04T12:00:00Z \
    --until 2026-05-04T12:05:00Z \
    --dir client
# capture session.ndjson
# protocol  modbus
# target    10.0.0.1:502
# started   2026-05-04T11:55:00.000000Z
# window    2026-05-04T12:00:00Z — 2026-05-04T12:05:00Z

[12:00:01.123456] c→u  32B  010100 1c 49 …
[12:00:01.456789] c→u  18B  010600 10 BE EF
…
```

## Chunks

### Chunk 1 — --since/--until (`39bf02a`)

`cmd/elsereno/cmd_proxy_replay_offensive.go`:

  - **--since RFC3339Nano** (lower bound, inclusive)
  - **--until RFC3339Nano** (upper bound, inclusive)
  - **timeWindow{since, until}** type + `contains(ts)`
    method:
      - Zero bound disables that side.
      - Zero ts (corrupted-line TS) passes through so the
        operator notices the bad timestamp in the rendered
        output instead of being silently filtered.
      - Both bounds zero → contains returns true
        unconditionally (no allocations, branch-predictor
        friendly; the no-flags path costs nothing).
  - **parseTimeWindow** validates: RFC3339Nano accepted
    (microsecond precision is what the recorder writes);
    `since > until` rejected with a friendly error
    suspecting an operator typo.
  - **`# window` header line** announces the active bounds
    so the operator sees what was applied. Open bounds
    render as "(open)".
