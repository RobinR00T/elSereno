# Snapshot — v2.20.0: ?atomic=tx with Store.WithTx wrapper

Single-chunk cycle. Closes v2.12 carryover (tx atomicity).
v2.12 added `?atomic=true` preflight validation; transient
PG errors at Create time could still leave partial state.
v2.20 adds `?atomic=tx` which wraps the apply pass in
`Store.WithTx` for real rollback on PG.

## REST

```
POST /api/v1/schedules/import?atomic=tx
```

Three modes now supported:
  - **omitted / `?atomic=false`**: v1.99 partial-success.
  - **`?atomic=true`**: v2.12 preflight-only.
  - **`?atomic=tx`**: v2.20 preflight + Store.WithTx-wrapped
    apply.

Unknown value → 400 "atomic must be true|tx (or omitted)".

## Response shape

v2.20 sends BOTH fields for back-compat:
```json
{
  "schema": "api:v1",
  "data": {
    "imported": ...,
    "atomic": true,           // v2.12 wire shape (bool)
    "atomic_mode": "tx",      // v2.20+ wire shape (enum)
    "items": [...]
  }
}
```

## Store

New interface method:

```go
WithTx(ctx context.Context, fn func(ScheduleStore) error) error
```

  - **Memory**: pass-through (no rollback). Documented.
  - **DBScheduleStore**: pass-through pending v2.21 pool
    plumbing — `NewDBScheduleStoreWithPool` ctor will
    BeginTx + bind a tx-scoped Querier. The interface
    contract is in place; the PG path is a follow-up.

## Handler refactor

  - `parseImportParams` now returns
    `(strategy, atomicMode, ok)` instead of bool.
  - Atomic mode constants: `atomicModeOff`,
    `atomicModePreflight`, `atomicModeTx`.
  - `runImportApply(...)` factored out so the
    `WithTx`-wrapping vs direct-call branches share the
    per-row loop.

## Tests (+2)

  - TestImport_AtomicTx_HappyPath: ?atomic=tx all-valid →
    200 + data.atomic_mode="tx" + 2 schedules stored.
  - TestImport_Atomic_UnknownMode: ?atomic=banana → 400.

## Honest scope

  - Memory + current DB store both pass-through WithTx.
    Real rollback semantics deferred to v2.21+ when the
    pool plumbing lands.
  - The promise this cycle: clients can issue ?atomic=tx
    today + get correct contract semantics when v2.21
    PG plumbing ships, without changing their code.
  - `atomic` (bool) + `atomic_mode` (enum) both in the
    response — v2.12 consumers stay green.
  - INSTALL.md unchanged. Build sizes unchanged.
