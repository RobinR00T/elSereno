# Snapshot — v1.48.0: proxy replay --stats summary

Single-chunk DX cycle. Adds a summary mode to `elsereno
proxy replay` that prints per-direction chunk count +
total bytes + time range instead of per-chunk lines.
Composes with v1.44/45/46/47 filters; ideal as a quick
sanity check before running a full replay on a multi-GB
capture.

## Workflow

```
$ elsereno proxy replay session.ndjson --stats \
    --since 2026-05-04T12:00:00Z --until 2026-05-04T12:30:00Z
# stats
c→u  127 chunks, 4892 bytes
u→c  121 chunks, 5104 bytes
tot  248 chunks, 9996 bytes
range  2026-05-04T12:00:01.123456Z — 2026-05-04T12:29:58.987654Z
```

When the filters narrow to nothing the range line shows
"(no matching chunks)" instead of a bogus zero-time
range.

## Chunks

### Chunk 1 — --stats + validateMutexFlags (`1834bdd`)

`cmd/elsereno/cmd_proxy_replay_offensive.go`:

  - **--stats bool** flag (default false).
  - **replayStats** type accumulates chunksClient /
    chunksUpstrm / bytesClient / bytesUpstrm / firstTS /
    lastTS via observe(ev). DirHeader filtered out
    upstream by chunkPassesFilters; only
    DirClient/DirUpstream reach observe().
  - **runProxyReplayStats** walks every matching chunk
    (no --limit / --tail short-circuit; operator asked
    for stats over the full window).
  - **printReplayStats** renders the summary block.
    Direction labels mirror the chunk-line arrows for
    continuity.
  - **validateMutexFlags** consolidates ALL the mutex
    checks in one helper. The pre-existing --limit /
    --tail check moves there. Each error message names
    the specific pair so the operator knows which flag
    to drop.
