# Snapshot — v1.38.0: elsereno fingerprint capture verb

Single-chunk cycle. Adds `elsereno fingerprint capture` —
the natural companion to v1.37's `validate --file`. Closes
a usability gap that the v1.37 carryover noted implicitly:
operators with a lab PLC needed to use Wireshark / tcpdump
/ a Python one-liner to capture bytes, then feed them to
`validate`. v1.38 ships the capture half so the workflow
stays inside the elsereno CLI.

## Workflow

```
(window 1) elsereno fingerprint capture \
               --listen 127.0.0.1:19999 \
               --output cap.bin
           listening on 127.0.0.1:19999; waiting for one connection (timeout 1m0s)

(window 2) plc-vendor-tool ... | nc -q1 lab-host 19999

(window 1) connected from 127.0.0.1:55421; draining bytes…
           wrote 124 bytes to cap.bin

(window 1) elsereno fingerprint validate \
               --plugin proconos --file cap.bin
           protocol: proconos
           …
```

## Chunks

### Chunk 1 — capture sub-verb (`9c44b82`)

`cmd/elsereno/cmd_fingerprint.go` extends:

  - **newFingerprintCaptureCmd** — cobra wiring +
    flag parsing.
  - **runFingerprintCapture** — listen + accept + io.ReadAll
    + os.WriteFile(0600). Refuses to write 0-byte files
    (an empty client write is treated as an error so
    operators don't end up with junk captures they
    silently mistake for "real").
  - **acceptWithCtx** — context-aware Accept. Go's
    `net.Listener.Accept()` isn't context-aware; we close
    the listener on ctx cancel to force the goroutine to
    return + translate "use of closed network connection"
    to ctx.Err(). Avoids the polling-loop / SetDeadline
    anti-pattern.

Flags:

  --listen   bind address (default 127.0.0.1:0; kernel-
             picked port printed on bind)
  --output   required; created 0600
  --timeout  upper bound on Accept + Read (default 60s)

## Test coverage (cmd_fingerprint_test.go, 4 cases)

  - **HappyPath** — fixed-port server, in-process dial,
    end-to-end byte-roundtrip, 0600 perms check
  - **MissingOutput** — required-flag rejection
  - **TimeoutOnIdleListener** — ctx deadline fires when no
    client connects; output file not created on timeout
  - **ClientClosesEmpty** — client opens + closes without
    sending → "without sending any bytes" error (defensive
    against silent-junk-file regressions)

Helpers added to the test file:
  - **waitForListenPort** — polls captured stdout for the
    `listening on 127.0.0.1:NNNN;` line, extracts the port
    via regex. 10ms tick, configurable budget.
  - **dialTimeout** — context-aware net.Dial wrapper.
