# Snapshot ‚Äî v1.47.0: proxy replay --tail N

Single-chunk DX cycle. Symmetric counterpart to v1.46's
`--limit`. Where `--limit` emits the FIRST N matching
chunks (and short-circuits the file walker), `--tail`
emits the LAST N ‚Äî useful when the relevant events are
at end-of-capture (e.g. a write that fired moments before
disconnection).

## Workflow

```
$ elsereno proxy replay session.ndjson \
    --tail 10 \
    --json \
    --dir client \
  | jq -c '{ts, len}'
# The 10 most recent client‚Üíupstream chunks.
```

### Ring buffer math

The ring caps memory at N entries regardless of capture
size ‚Äî operators tailing a multi-GB session don't balloon
RAM. Buffer order is preserved by:

  1. Writing into `ring[count%tail]` for each match.
  2. After the walk: `start = count % tail` when count >
     tail, else 0.
  3. Emit ring[(start+i) % tail] for i in [0, n) where
     n = min(count, tail).

This emits in arrival order regardless of where the wrap
point landed.
