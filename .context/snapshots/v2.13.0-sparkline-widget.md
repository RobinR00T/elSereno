# Snapshot — v2.13.0: dashboard sparkline widget

Single-chunk cycle. Closes v2.11 carryover. v2.11 added
`/stats/timeseries`; v2.13 adds the dashboard UI.

## Dashboard surface

  - New "Sparkline" button on each schedule row (between
    Runs + Clone).
  - Opens an inline SVG widget below the schedules table.
  - Three series rendered as polylines:
    - **Blue (solid)**: total_runs.
    - **Red**: failed.
    - **Green (dashed)**: total_findings.
  - Each series auto-scales independently so low-count
    series (failed) stay visible even when total_runs
    dominates.
  - Bucket selector (hour/day/week) + days input (1-365);
    changing either re-fetches.
  - Summary line: `(N buckets · X runs · Y failed · Z
    findings)`.
  - `<title>` tooltips per polyline give the max value.

## State

Module-level `sparklineSchedID` + `sparklineSchedName`
track the open view so `reloadSparkline()` can re-hit the
endpoint on form changes without re-opening.

## JS API (window-exposed)

  - `openSparklineView(id, name)` — opens + loads.
  - `closeSparklineView()`.
  - `reloadSparkline()` — triggered by select/input
    onchange.

## Honest scope

  - No zoom/pan — operators wanting drill-down change
    bucket/days dropdown.
  - No tooltip-per-bucket hover — `<title>` per polyline
    only.
  - SVG inline (no library); ~80px height; scales to
    container width.
  - INSTALL.md unchanged. Build sizes unchanged.

## Build sizes

Unchanged across all 3 variants.

Adds ~130 LoC (HTML + JS) in dashboard.go.
