# Snapshot — v2.28.0: dashboard rename-tag form

Single-chunk cycle. Consumes the v2.16 bulk-rename
endpoint from the UI so operators don't need curl.

## UX

  - New form above the tag-cloud: `from` + `to` text
    inputs (HTML5 pattern `[a-z0-9_-]{1,32}`) + Rename
    button + inline status span.
  - On submit: POST to
    `/api/v1/schedules/tags/rename` + render
    "renamed N schedule(s)" on success.
  - Client-side guards: empty / `from == to` short-
    circuit with inline error before sending.
  - On success: inputs cleared; active filter set
    migrates `from → to` so the dashboard doesn't show
    "no matches" with a stale chip.

## Honest scope

  - HTML5 pattern attribute is a UX hint; the server
    is the authority on tag-shape validation.
  - Errors surface inline (status span); no toast / modal.
  - INSTALL.md unchanged. Build sizes unchanged.

## Build sizes
