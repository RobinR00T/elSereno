# Snapshot — v1.32.0: cmd/elsereno gosec marker hygiene

Hygiene-only single-chunk cycle. Completes the b611f5c
(pre-v1.28) `//nolint:gosec` → `// #nosec G<NNN>` migration
for the cmd/elsereno/ subtree.

## Why a hygiene cycle

PITF-030 (restated by b611f5c): golangci-lint's bundled gosec
accepts both directive forms, but the standalone gosec binary
that runs in CI's `sec` job only honours `// #nosec G<NNN>`.
b611f5c swapped 18 markers; v1.30 chunk 2 swapped 2 more
(`cmd_tui.go:187` + `feeds/replay.go:50`); 10 stragglers
remained in cmd/elsereno/ that pre-dated the convention.

The cycle does NOT touch the wider tree (~65 more markers
across internal/protocols/**, offensive/write/**,
internal/audit/**, offensive/sandbox/**). Those have
coexisted with the convention since b611f5c and `make sec`
has been exit 0 throughout. Touching them would be its own
~150-line diff with non-trivial review surface (each marker's
G-code needs verification). A v1.33+ sweep is the right
follow-up scope if/when the operator wants full convention
parity.
