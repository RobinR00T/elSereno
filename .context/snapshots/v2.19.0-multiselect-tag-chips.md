# Snapshot — v2.19.0: multi-select Shift+Click on tag chips

Single-chunk cycle. Closes v2.6 dashboard carryover.
v2.6 added single-tag click filter; v2.17 added the
not_in operator; v2.19 wires the dashboard UI to do
multi-select + op switching.

## UX

  - **Plain click** on a chip → replace the active set
    with just that tag (or clear if the chip was the
    sole entry).
  - **Shift+Click** on a chip → toggle that tag's
    membership in the active set.
  - Active chips are highlighted (bg #99c / fg white).
  - New `op:` dropdown (AND / OR / NOT IN; defaults to
    OR) next to the cloud lets the operator switch the
    boolean composition without rebuilding the set.

## State

  - `scheduleTagFilterSet` (array) — active tags.
  - `scheduleTagFilterOp` (string) — current op.
  - `scheduleTagFilter` (string) — legacy single-tag
    mirror, kept for back-compat with v2.13/v2.14 code
    paths.

## URL

```
/api/v1/schedules?tag=a&tag=b&op=or
```

Plumbed through renderSchedules. Single-tag + op=AND
still hits the v2.4 fast path server-side.

## Honest scope

  - No drag-select. Shift+Click + click are sufficient
    for the operator who's checking 3-5 tags at most.
  - No keyboard shortcut (Esc to clear). Click "Clear
    filter" button.
  - INSTALL.md unchanged. Build sizes unchanged.

## Build sizes

Unchanged across all 3 variants.

Adds ~50 LoC (state + new helper) in dashboard.go.

## Carryover

  - **vNext**: right-click on a chip → add to not_in
    filter (faster "exclude this tag" workflow).
