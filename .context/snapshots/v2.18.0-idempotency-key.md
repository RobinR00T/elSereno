# Snapshot — v2.18.0: Idempotency-Key on /import

Single-chunk cycle. Closes v2.12 carryover (idempotency
on /import). Operators with flaky networks retry imports;
without idempotency the retry creates duplicates.

## Wire protocol

```
POST /api/v1/schedules/import?on_conflict=skip
Idempotency-Key: <opaque string>
Content-Type: application/x-ndjson

{"name":"alpha","template":...}
```

  - **First request**: processed normally. Response stored
    in the in-memory cache under the key (with body hash).
  - **Retry with same key + same body**: 200 + replays
    the cached response. Header `Idempotency-Replay: true`
    tells the client it was a replay.
  - **Retry with same key + DIFFERENT body**: 409 Conflict
    (operator likely confused two requests; surfaces the
    error rather than silently committing).
  - **Cache miss / no key**: identical to v2.17 behaviour.

## Cache

`internal/web/handlers/idempotency.go`:

  - `idempotencyCache` struct: TTL (default 1h) + maxSize
    (default 256). LRU eviction by storedAt timestamp.
  - SHA-256 body fingerprint per entry.
  - `Lookup(key, body)` returns
    {miss, hit, conflict} + the cached entry.
  - `Store(key, body, status, response)` caches the
    rendered response bytes.
  - Module-level `defaultIdempotencyCache` shared across
    requests in the process.

## Helpers (funlen refactor)

importSchedules grew past the 60-line cap once
idempotency was wired. Split into:

  - `parseImportParams(w, r)` — query validation.
  - `tryReplayIdempotency(w, key, body)` — cache check.
  - The main handler keeps just the orchestration.

Also dropped the now-unused `decodeImportBody(r)` (lint
caught it); only `decodeImportBodyBytes` is exposed.

## Tests (+2)

  - TestImport_IdempotencyKey_Replays: same key+body twice
    → store has 1 schedule (not 2); replay carries
    Idempotency-Replay header; body bytes are identical.
  - TestImport_IdempotencyKey_BodyMismatchConflict: same
    key + different body → 409.

## Honest scope

  - Per-process cache. Multi-process serve doesn't share;
    failover may double-commit. Future: back with the v1.90
    advisory-lock pattern for cross-process consistency.
  - TTL = 1h. Client retrying after 1h must use a new
    key OR retry beyond the cache window (cache miss →
    re-process).
  - Only /import wired. Other mutating endpoints (clone,
    bulk/enable, bulk/disable) are deferred — they're
    cheap to retry by design.
  - INSTALL.md unchanged. Build sizes unchanged.
