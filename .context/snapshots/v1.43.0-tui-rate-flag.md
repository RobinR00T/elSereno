# Snapshot — v1.43.0: tui --rate N

Single-chunk DX cycle. Exposes the long-existing
`feeds.Replay.Rate` + `feeds.Stdin.Rate` field via a
`tui --rate N` CLI flag.

## Why now

The `Rate` field has been in place since v1.29 chunk 3
(replay) + chunk 4 (stdin) for completeness — but never
had a CLI knob. Operators wanting slow-motion demos had
to build their own producer that spaced events out.

`paceFromRate(eventsPerSec) time.Duration` already exists.
This chunk just plumbs the operator-facing flag through to
the feed constructor.

## Chunks

### Chunk 1 — flag plumb (`b53d7cc`)

`cmd/elsereno/cmd_tui.go`:

  - `cmd.Flags().Float64Var(&args.rate, "rate", 0, …)`
  - `rate float64` field on `pickFeedArgs`
  - In pickFeed:
    - `feeds.Replay{Path, Rate: a.rate}`
    - `feeds.Stdin{In: os.Stdin, Rate: a.rate}`
  - --watch / --input / no-flag default ignore the rate
    (those are live; flag-doc says so explicitly).

## Operator workflow

```
$ elsereno scan --output-format ndjson > big-scan.ndjson
$ elsereno tui --replay big-scan.ndjson --rate 5
# Plays at 5 findings/sec for a calm audience-paced demo.
```
