# Snapshot — v2.1.0: cloned_from audit event

Single-chunk cycle. Closes v1.93 carryover. v1.93 added
schedule cloning but no audit row was written, so
compliance reviews couldn't trace clone provenance.

## Schema (migration 00015)

```sql
ALTER TABLE scan_schedule_audit
    DROP CONSTRAINT scan_schedule_audit_event_type_check;
ALTER TABLE scan_schedule_audit
    ADD CONSTRAINT scan_schedule_audit_event_type_check
    CHECK (event_type IN (
        'force_overwrite', 'delete',
        'set_enabled_true', 'set_enabled_false',
        'cloned_from'  -- NEW
    ));
```

PITF-030 pattern: SQL DDL is source of truth, mirrored
Go-side by `ValidScheduleAuditEventTypes` constant.

## Go enum

New constant + slice entry:

```go
ScheduleAuditEventClonedFrom ScheduleAuditEventType = "cloned_from"
```

## Handler

`cloneSchedule` signature change: now takes `audit
scanorch.ScheduleAuditStore` (was: store only). The
factory wires the audit store from the existing
`Schedules(store, audit, scanStore)` params.

After successful Create:
  - audit_row.schedule_id = CLONE's ID (this is the clone's
    origin event, not the source's).
  - payload_before = full source snapshot at clone time.
  - payload_after = full clone snapshot.

Best-effort: persistence failure → `X-Schedule-Audit-Warning`
header (v1.84 pattern). Clone itself always succeeds.

## Dashboard

Audit history view: `event_type='cloned_from'` now renders
a blue "CLONED FROM" badge + `source: <name> (id=<id>)`
line (parallel to the v1.89 "DELETED" badge pattern).
Refactored the inline `parsePayload` helper out of the IIFE
so both delete + cloned_from branches share it.

## OpenAPI

`ScheduleAuditEvent` schema's `event_type` enum extended
to include `cloned_from`. `docs/openapi.yaml` regenerated.

## Tests (+2)

  - TestCloneSchedule_WritesAudit: with audit store wired,
    a clone produces 1 event keyed on clone.ID with
    EventType=cloned_from, operator from X-Operator, and
    payload_before decoding to the original source's ID.
  - TestCloneSchedule_NoAuditStore_StillSucceeds: nil
    audit → clone still 201 (best-effort).

## Honest scope

  - The audit row is keyed by clone.ID rather than the
    source. An operator wanting "show me everything cloned
    FROM schedule X" must grep
    `event_type='cloned_from' + payload_before.id=X`
    (not a single index lookup). A `source_id` column
    could speed this up but adds schema for a rare query
    pattern; deferred.
  - Same v1.85 diff renderer would show "every field new
    on the clone" — unhelpful. The badge avoids that.
  - INSTALL.md unchanged. Build sizes unchanged.
