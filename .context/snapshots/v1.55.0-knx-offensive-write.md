# Snapshot — v1.55.0: KNX offensive write-gated proxy

Single-chunk cycle. Closes the v1.32+ D1 carryover.
Adds the offensive UDP-aware write-gated proxy for
KNXnet/IP on UDP/3671. KNX is the dominant European
BAS: HVAC, lighting, blinds, access control, metering,
life-safety. Mutating a KNX bus = changing physical
state in real buildings.

Plus a buried correctness fix for v1.21 chunk 1's KNX
service-type constants — DESCRIPTION_REQUEST/RESPONSE
were off by one from the KNX standard, masked by self-
consistent test fixtures.

## Why

The default knxip plugin (v1.21 chunk 1) only sends
DESCRIPTION_REQUEST and refuses every offensive byte —
the read-only build can't relay UDP at all. v1.55 adds
the offensive replacement that gates KNXnet/IP at three
nested tiers (service-type, APCI, group-address)
without either over-refusing legitimate traffic or
under-gating the dangerous one.

## Three-tier gate

```
service-type → APCI → group-address
   |              |          |
   |              |          └── refuse if outside
   |              |              (GroupAddr & GroupMask)
   |              |
   |              └── inside TUNNELLING / ROUTING:
   |                  refuse APCIs not on allowlist;
   |                  Read+Response always pass
   |
   └── allowed services: caller-defined.
       always-safe: SEARCH, DESCRIPTION,
       CONNECTIONSTATE, DISCONNECT, TUNNELLING_ACK,
       routing-diag
```

Refusal mode: **silent drop**. KNXnet/IP has no
"permission denied" service-type; fabricating a
DISCONNECT_REQUEST risks tearing down the wrong
session. The standard 10s heartbeat covers stuck
sessions.

## Chunks

### Chunk 1 — wire + gate (`9d688ac`)

`internal/protocols/knxip/wire/services.go` (new):

  - **ServiceType(frame)** extracts the 16-bit BE
    service-type from any KNXnet/IP frame.
  - **15 service-type constants** (Core 0x02xx,
    Device Management 0x03xx, Tunnelling 0x04xx,
    Routing 0x05xx).
  - **6 APCI constants** (GroupValue_Read/Response/
    Write, IndividualAddress_Write, Memory_Write,
    Restart) covering the operationally important
    write APCIs.
  - **6 cEMI Message Codes** (L_Data req/ind/con,
    M_PropRead/Write, M_Reset).
  - **CEMILData** struct: MsgCode, SourceIA,
    DestAddr, DestIsGroup, APCI.
  - **ParseTunnellingCEMI** strips the 4-byte
    TUNNELLING connection-header and parses the
    inner cEMI L_Data.
  - **ParseCEMILData** for ROUTING_INDICATION (no
    connection-header).
  - **FormatGroupAddress** renders 16-bit GA as
    canonical "main/middle/sub" 5/3/8-bit pack.
  - 4 sentinels: ErrServiceTypeMissing,
    ErrCEMITooShort, ErrCEMINotLData,
    ErrTunnellingBodyTooShort.

`offensive/write/knxip/gatedproxy.go` (new, ~360 lines):

  - **AllowedService**, **AllowedAPCI**, **AllowedGroup**
    — 3-dimensional allowlist.
  - **AllowedGroup.Matches(dest)** = (dest & mask)
    == (addr & mask). Operator picks granularity:
    0xFFFF = single GA, 0xFF00 = sub-group, 0xF800 =
    main group.
  - **AllowlistHash** with separators 0xE1 (APCI
    dimension) and 0xE2 (group dimension). Empty
    sub-dimensions yield service-only hash.
  - **SessionMutationWithGeneration** for v1.17-style
    token-generation cookie.
  - **WriteGatedHandler** with UDP per-datagram
    routing (mirrors the iax2 ADR-040 template).
  - **Always-safe service-types** map: 13 entries
    covering discovery + keep-alive + ack + diag
    flow-control.
  - **Read-only APCIs** map (GroupValue_Read,
    GroupValue_Response) always-pass inside
    allowlisted TUNNELLING.

`internal/protocols/knxip/wire/wire.go` (corrected):

  - DESCRIPTION_REQUEST: 0x0204 → 0x0203.
  - DESCRIPTION_RESPONSE: 0x0205 → 0x0204.
  - Per KNX Standard 03.08.02 §4.1. v1.21 chunk 1
    had these mis-set; real KNX hardware would have
    refused the request.
  - Existing tests + fingerprint plugin updated to
    use the corrected values.

## Honest scope

  - **CLI flag plumbing** is NOT in this chunk. Wire
    + gate primitives are the foundation; flags in
    cmd_write_offensive.go are a future cycle's add
    (same pattern as v1.52/v1.53 close).
  - **M_PropWrite / M_Reset** (DEVICE_CONFIGURATION)
    is gated only at the service-type level. Per-
    object ID + PID granularity needs a Device Mgmt
    cEMI parser — future cycle.
  - **KNXnet/IP Secure** (0x09xx encrypted wrappers)
    NOT supported. Encrypted frames can't be gated
    until session-key handshake is parsed.
  - **Per-IndividualAddress source filtering**
    (refuse writes from unauthorised source IAs)
    NOT in this chunk. Operator-driven; not yet
    asked for in field.
  - **Lab validation against real KNX hardware**
    pending. Confidence ~0.85 until validated
    against a real Gira/MDT/JUNG IP-interface.

## Hash separator allocation

| Sep   | Cycle  | Plugin | Purpose                   |
|-------|--------|--------|---------------------------|
| 0xF1  | v1.52  | s7     | per-(area, db, addr)      |
| 0xF2  | v1.53  | enip   | per-(class, inst, attr)   |
| 0xE1  | v1.55  | knxip  | per-APCI dimension        |
| 0xE2  | v1.55  | knxip  | per-(GA, mask) dimension  |

knxip uses two separators (0xE1, 0xE2) because the
allowlist is genuinely 3-dimensional: service +
APCI + group. Other plugins so far have only had
one extra dimension beyond the FC/cmd base.
