# Snapshot — v1.53.0: enip per-(class, instance, attribute) gating

Single-chunk cycle. Closes the v1.32+ B carryover.
Tightens the enip write-gate from command-only (v1.27)
to per-(class, instance, attribute) granularity for
SendRRData / SendUnitData. Sister to v1.52's s7 per-
address gating; together they close the wire-level
allowlist tightening for the two most common ICS
write-protocols.

## Why

CIP/ENIP-based ICS deployments routinely allow
SendRRData for legitimate engineering ops (firmware
queries, parameter reads, controlled writes). Command-
only gating (v1.27) means an operator who authorises
SendRRData has authorised access to ALL CIP objects on
the device — including safety-critical controllers and
identity-modification services. Per-attribute gating
closes that gap.

## Chunks

### Chunk 1 — wire parser + gate (`c08edfb`)

`internal/protocols/enip/wire/epath.go` (new):

  - **EPathTarget**{Class, Instance, Attribute, Has*}
    parsed from the MR request EPATH.
  - **readSegmentValue** centralises bounds-checked
    decode of 8/16/32-bit logical-segment values. Pulls
    ParseMRPath's cyclomatic complexity from 20 to 6
    (table-driven dispatcher).
  - **ParseMRPath** walks the segment list. Supports
    Class/Instance/Attribute logical segments in all
    three width forms. Unknown segment types
    (port-segment, symbolic, network-segment) refuse —
    the gate doesn't allow what it can't fully classify.
  - **ExtractMRTarget** locates the MR request inside a
    SendRRData/SendUnitData encapsulation body
    (skipping InterfaceHandle + Timeout, walking the
    CPF item list, finding the Unconnected Data Item
    TypeID 0x00B2 OR Connected Data Item TypeID 0x00A2,
    parsing the path).

`offensive/write/enip/gatedproxy.go`:

  - **AllowedAttribute**{Class, Instance, Attribute,
    MatchType} with three match strictnesses:
      - **MatchExact**         — all three must match.
      - **MatchClassInstance** — Class + Instance match;
                                 Attribute wildcarded.
      - **MatchClassOnly**     — only Class matches;
                                 Instance + Attribute
                                 wildcarded.
  - **AllowlistHash** signature gains attrs dimension.
    Hash separator 0xF2 (high range). Empty attrs list
    yields the v1.27 hash — pre-v1.53 confirm-tokens
    keep validating.
  - **SessionMutationLegacy(target, allowed)** preserved
    for any 2-arg caller.
  - **WriteGatedHandler.AllowedAttributes** field;
    empty list = v1.27 behaviour, non-empty activates
    per-attribute gate for SendRRData/SendUnitData
    only (other write commands keep cmd-level gating).

## Per-attribute match strictness — when to use which

  - **MatchClassOnly**: operator allows ALL writes to a
    specific vendor object — typical for "any
    parameter on the Drive Profile object".
  - **MatchClassInstance**: operator allows any
    attribute of one specific instance — typical for
    "the controller's primary recipe table" where
    individual fields aren't known up-front.
  - **MatchExact**: operator allows one specific
    (class, instance, attribute) — tightest grain;
    typical for "the Identity Object's Status attribute
    only".

The three modes compose: an allowlist with a mix of
strictnesses works fine; the gate evaluates each entry
independently.

## Honest scope

  - **CLI flag plumbing** (`proxy listen --attr
    class=1,instance=2,attr=7,match=exact`) is NOT in
    this chunk. Wire + gate primitives are the
    foundation; flag plumbing in cmd_proxy_offensive.go
    is a future cycle's add.
  - **Other write commands** (RegisterSession,
    UnregisterSession) are NOT constrained by
    AllowedAttributes — those don't carry an MR
    request. They keep command-level gating.
  - **Symbolic segments** (Rockwell tag-based
    addressing like "MyTag") refused via
    ErrEPathUnknownSegment. Future cycle could add
    symbolic match if asked.
