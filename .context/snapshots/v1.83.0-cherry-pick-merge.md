# Snapshot — v1.83.0: cherry-pick merge view

Single-chunk cycle. Dashboard-only JS+HTML. v1.81's
merge view forced an all-or-nothing choice (Take server
/ Force overwrite). v1.83 adds per-field radio buttons
so the operator can keep some of their edits + accept
some of the server's changes in one merged submit.

## UI

Each diff row now renders as:

```
field-name: (•) mine=<value>   ( ) server=<value>
```

Default = mine. The merge view gains a new button:

```html
<button id="schedule-apply-selected-button"
        onclick="applySelectedMerge()">
  Apply selected (per-field)
</button>
```

v1.81's Take server + Force overwrite remain
(Take server = quick "all server"; Force overwrite =
unconditional last-write-wins).

## applySelectedMerge

1. Walks `<li data-field=…>` rows.
2. For rows where "server" is checked, applyServerField
   copies the field's server value into a merged
   payload (starting from pendingMergePayload).
3. Cadence XOR preserved automatically:
   - server interval picked → cron_expr cleared.
   - server cron picked → interval_seconds = 0.
4. PUT **with** If-Match. A third concurrent edit
   re-opens the merge view (412 → enterMergeView).

## applyServerField

Switch-by-field-name helper. Six fields handled:
name, template.input, template.plugins,
interval_seconds (clears cron), cron_expr (clears
interval), timezone.

## Tests
