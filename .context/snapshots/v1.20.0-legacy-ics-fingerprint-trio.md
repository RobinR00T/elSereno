# Snapshot — v1.20.0: legacy ICS fingerprint trio

Three chunks shipping read-only fingerprint plugins for legacy
ICS protocols that were on the v1.19 deferred list. Each plugin
is from-scratch wire (per `internal/protocols/<name>/wire/`),
defence-in-depth fail-closed proxy, and per-protocol scoring
factors. The plugin count in the default build climbs from 17
to **20**.

## Chunk 1 — Omron FINS UDP fingerprint plugin

`(commit 96c5453)` UDP/9600. Sends a 13-byte CONTROLLER DATA READ
datagram (MRC=0x05, SRC=0x01, area=0x00) per OMRON CPU manual
W421 §5.1/§5.4. Validates ICF response bit, SID echo (fresh
non-zero SID via `crypto/rand` per probe), MRC/SRC match, end-
code zero. Trims NUL/space padding off the 20-byte ASCII Model.

Sentinel errors → operator-facing notes via classifyParseError:
- ErrShortFrame → "short FINS frame (N bytes)"
- ErrNotResponse → "FINS frame not a response or wrong MRC/SRC"
- ErrServiceMismatch → "FINS SID echo mismatch"
- ErrEndCodeNonZero → "FINS end-code non-zero (refusal)"

ProxyHandler is fail-closed: TCP framework can't relay UDP. A
dedicated UDP relay arrives with the future offensive write
plugin (memory-area write MRC=0x01 SRC=0x02, RUN/STOP transitions
MRC=0x04, etc.) — analogous to BACnet's pattern.

Score factors{protocol_risk:80, exposure:80, auth_state:95,
capability:30→75 on FINS reply, impact_class:75, cve_exposure:0}.

12 wire tests + 11 plugin tests.

## Chunk 2 — MELSEC SLMP TCP fingerprint plugin

`(commit fcf5931)` TCP/5007. Sends a 15-byte READ CPU MODEL NAME
3E-frame request (command 0x0101, subcommand 0x0000) per
Mitsubishi Electric SLMP Reference Manual SH(NA)-080956ENG.
Validates subheader 0xD000, declared-length consistency, end-code
zero. Caps the declared-length field at MaxResponseDataLength
(8192) to defuse oversized-length attacks. Trims NUL/space
padding off the 16-byte ASCII Model with a single-pass cutset
(the order-dependent two-call form leaks embedded NULs when
padding interleaves spaces and zeros).

Plugin reads the 9-byte header first to learn the body length,
then reads the body with growth fallback if the initial 256-byte
buffer is too small.

ProxyHandler is wire-layer write-ban: drains the request and
replies with a 13-byte error frame carrying SLMP end code 0xC059
("command unsupported" per §6.6 end-code table); does NOT
forward to upstream.

Score factors{protocol_risk:80, exposure:75, auth_state:95,
capability:30→75 on SLMP reply, impact_class:75, cve_exposure:0}.

9 wire tests + 11 plugin tests.

## Chunk 3 — GE-SRTP TCP fingerprint plugin

`(commit 8a89baa)` TCP/18245. Sends a 56-byte CONNECTION INIT
mailbox (byte 0 = 0x02, rest zero) reverse-engineered from
Rapid7's nmap NSE script gesrtp-info and Conpot's GE simulator
fixtures. Reads exactly 56 bytes, classifies by SRTP type byte
(0x03 = response).

Public protocol documentation is sparse, so deeper service-code
probing (CPU model identification via service 0x21) is deferred
to a future cycle that can carry test vectors against real PLCs.
The connection-init classifier is the safest reliable fingerprint.

ProxyHandler is wire-layer write-ban: reads the client's 56-byte
mailbox and replies with a 56-byte mailbox carrying byte 0 = 0x03
(response) + byte 42 = 0x01 (non-zero status / minor error
indicator from the public reverse-engineering notes); does NOT
forward to upstream.

Score factors{protocol_risk:80, exposure:75, auth_state:95,
capability:30→70 on SRTP reply (slightly lower because the
connection-init classifier doesn't decode the response payload —
operators have less actionable detail on controller model),
impact_class:75, cve_exposure:0}.

6 wire tests + 7 plugin tests.
