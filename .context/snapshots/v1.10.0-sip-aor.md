# Snapshot — v1.10.0: SIP REGISTER AOR allowlist (anti-registration-hijack)

Closed **2026-04-24**. Single-chunk cycle shipping the twin of
v1.9 chunk 5: where the INVITE prefix gate controls **WHERE**
calls can go (toll-fraud mitigation), this AOR gate controls
**WHO** can register a binding (registration-hijack mitigation).
Both together block the two most common PBX abuse vectors.

## Chunk

### Chunk 1 — SIP REGISTER AOR allowlist (`fd9e493`)

Operator use-case: attacker captures SIP creds for one account
(phishing, WiFi sniff, compromised endpoint). Without AoR
gating, the attacker can register a binding for ANY AoR the
upstream accepts — e.g. use alice's creds to register
`admin@pbx` pointing to the attacker's IP, and hijack calls
addressed to admin. With AoR gating, REGISTER is scoped to the
exact AoR(s) the operator pre-authorised at session open.

Library (`offensive/write/sip/gatedproxy.go`):
  - `AllowedAOR{AOR string}` opt-in type.
  - `WriteGatedHandler.AllowedAORs []AllowedAOR` field.
  - `canonicaliseAOR` strips angle brackets, URI parameters,
    scheme (sip:/sips:/tel:), trims, lowercases host (user
    part preserved per RFC 3261 §19.1.1 — case-sensitive by
    default).
  - `AllowlistHashWithAORs(target, methods, prefixes, aors)`
    triple-dimensional hash. Backwards-compat: empty aors →
    v1.9 `AllowlistHashWithPrefixes`; empty aors AND empty
    prefixes → v1.4 `AllowlistHash`. Uses 0xFE separator
    (can't collide with 0xFF prefix separator nor ASCII
    method bytes).
  - `SessionMutationWithAORs` factory — `Authorise()` now
    calls this variant; degrades cleanly for operators not
    opting into AOR gating.
  - `registerAORAllowed` performs canonical compare against
    each allowlist entry; fails closed on empty/malformed
    To: header.
  - `extractToURIFull` is the sibling of `extractToURIUser`
    (v1.9): keeps user@host form (AOR gating is identity-
    level, not prefix-level).
  - `writeRegisterForbidden` emits SIP/2.0 403 Forbidden with
    echoed Via/From/To/Call-ID/CSeq + `X-Elsereno-Gate-Reason:
    AOR not in session allowlist (REGISTER hijack guard)`.
  - `forwardOne` adds a parallel branch to the INVITE prefix
    check: `REGISTER + len(AllowedAORs) > 0` → canonical
    compare.

CLI (`cmd/elsereno/cmd_write_gates_offensive.go`):
  - `--aor <AoR>` flag on `write sip dry-run`, repeatable.
  - Output includes `AORs:` line (or "(none — REGISTER AoR
    not constrained)").
  - `canonAORs` helper dedupes + sorts for display.

YAML (`cmd/elsereno/cmd_proxy_allowfile_offensive.go`):
  - `proxyAllowFile.AORs []string` with `yaml:"aors,omitempty"`.
  - `loadAllowFile` translates `af.AORs` → `opts.aors` for the
    SIP plugin case.

Emitter (`cmd/elsereno/cmd_write_emitfile_offensive.go`):
  - `buildAllowFileSIP` signature: `(target, methods,
    toPrefixes, aors)`. Dedupes, sorts, omits `aors:` when
    empty.
  - All 3 existing test callers updated to 4-arg.

Proxy listen (`cmd/elsereno/cmd_proxy_offensive.go`):
  - `proxyListenOpts.aors []string` field.
  - `--aor` flag on `proxy listen` + YAML loader populates it.
  - `buildSIPHandler` wires `opts.aors` →
    `handler.AllowedAORs`.

Tests:

## Retrospective

### What went well
- Pattern copy from v1.9 chunk 5 was near-verbatim. Most of
  the work was renaming `Prefix` → `AOR` + adjusting the
  semantics (exact match vs. prefix match; user@host vs.
  user-only).
- Triple backwards-compat ladder (empty aors → v1.9; empty
  aors + prefixes → v1.4) keeps all existing tokens valid.
  The `AllowlistHashWithAORs(t, m, nil, nil) ==
  AllowlistHash(t, m)` test guarantees this.
- Fail-closed contract on malformed To: is the right default:
  if a gate is active, refuse what you can't verify.

### Pitfalls hit + fixed
- `buildAllowFileSIP` signature change (3-arg → 4-arg) broke
  3 existing test callers. Fixed with a targeted sed pass +
  double-checking each call site.
- Chunk started on CWD `/Users/…/AI projects` instead of the
  project root — a leftover from the Claude Code restart. A
  single `cd elsereno` at the start corrects it; no other
  impact.

### Carry-overs to v1.11+
- CWMP offensive proxy (SOAP RPC allowlist) — TR-069 v1.4
  landed the fingerprint; offensive gate is the matching
  piece.
- BACnet per-object allowlist (ASN.1 BER parsing) — mirrors
  OPC UA per-NodeId but BACnet's wire format is very
  different.
- OPC UA String/Guid/ByteString NodeID encodings — v1.6 gate
  is numeric-only.
- OPC UA multi-node per WriteRequest — first-WriteValue only
  today.
- OPC UA CallRequest per-object.
- Modbus structured `writes:` YAML — unit + FC + start/end
  tuple vs. just FC list.
- Input client pagination (chunks 3-4 of v1.9 fetch page 1
  only).
- SIGHUP reload (still needs PayloadHash/token-invalidation
  redesign).
