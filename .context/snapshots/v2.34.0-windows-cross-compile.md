# Snapshot — v2.34.0: Windows cross-compile target

Single-chunk cycle. Closes the long-deferred "Windows
cross-compile" vNext entry. Default + mini builds now
cross-compile cleanly to `windows/amd64` and
`windows/arm64`; offensive stays unix-only.

## What ships

  - **default** build for Windows (amd64 + arm64) — ~34 MB.
  - **mini** build for Windows (amd64 + arm64) — ~31 MB.
  - **offensive** stays linux/darwin (SIGUSR1 + libsandbox
    are unsupportable on Windows without a CGO/wireless
    porting effort that's out of scope).

## Code changes

### `cmd_doctor.go` — split `checkDisk`

The previous in-file `syscall.Statfs_t` is Unix-only.
v2.34 hoists it into platform-specific files:

  - `cmd_doctor_disk_unix.go` (`!windows`): retains the
    existing `syscall.Statfs` path.
  - `cmd_doctor_disk_windows.go` (`windows`): calls
    `kernel32.GetDiskFreeSpaceExW` via the `syscall`
    standard-library LazyDLL trampoline (no
    `golang.org/x/sys/windows` dependency).

### `cmd_doctor.go` `syscall.Geteuid()` call (Darwin path)

Kept as-is — `syscall.Geteuid` is stubbed on Windows
(returns -1), so the runtime `case "darwin"` branch
never executes there but the symbol still exists at
compile time.

### `.goreleaser.yml` matrix

Default + mini build entries: `goos: [linux, darwin]`
→ `goos: [linux, darwin, windows]`. Offensive
unchanged.

## Honest scope

  - Windows builds reuse the Unix scoring/scanner code
    paths. The 25 default-build protocol plugins
    cross-compile but real-world testing on Windows is
    operator-led.
  - No Windows-native UAC checks; the privileged-scan
    `doctor` row reports `skip` ("only linux/darwin in
    v1") on Windows. Future cycle can add
    `OpenProcessToken` UAC detection.
  - No installer (MSI/Chocolatey/winget). Operators run
    the binary directly; v2.34 ships the binary, not the
    packaging.
  - INSTALL.md still says "Linux + macOS only"; v2.34
    DOES change the platform matrix at the binary level
    but the operator UX (systemd unit, .deb, etc.)
    remains unix-only. Update doc deferred to v2.35+.

## Build sizes

| build       | os/arch          | size  |
|-------------|------------------|-------|
| default     | linux/amd64      | 23 MB |
| default     | darwin/arm64     | 23 MB |
| **default** | **windows/amd64**| **34 MB** |
| **default** | **windows/arm64**| **31 MB** |
| mini        | linux/amd64      | 21 MB |
| **mini**    | **windows/amd64**| **31 MB** |
| **mini**    | **windows/arm64**| **29 MB** |
| offensive   | linux/amd64      | 24 MB |

Windows binaries are larger because the Go runtime ships
the Windows-specific symbol tables + the kernel32
trampoline glue.
