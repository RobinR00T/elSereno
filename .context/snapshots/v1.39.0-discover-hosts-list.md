# Snapshot — v1.39.0: discover --hosts <file>

Single-chunk cycle. Adds the natural counterpart to v1.15-
chunk-2's `--auto <CIDR>`. When operators already have a
curated host inventory (CMDB / asset-management export,
nmap host-list extract, hand-maintained list), expanding
a sparse CIDR to walk it wastes time + RAM. v1.39 lets
them feed the list directly.

## Workflow

```
$ cat scada-hosts.txt
# production scada subnet — exported from CMDB
10.10.20.5
10.10.20.6:502    # already-known modbus port — port half stripped
2001:db8::5
fe80::1

$ elsereno discover --hosts scada-hosts.txt --max-hosts 100
{"address":"10.10.20.5","port":502,"protocol_hints":["modbus"], …}
…
```

## Chunks

### Chunk 1 — --hosts flag + loader (`657242d`)

`cmd/elsereno/cmd_discover.go`:

  - **--hosts <file>** flag, mutex with --auto (the
    `(auto == "") == (hostsFile == "")` XOR check returns
    EX_USAGE for both-empty and both-set).
  - **loadDiscoverHostsFile** helper:
    - One IP per line; `#` for full-line and inline
      comments; blanks skipped.
    - Tolerates `host:port` lines (port half stripped
      with an IPv6 `::` skip-heuristic — the discover
      verb supplies its own port list, the operator's
      port suffix is informational only).
    - --max-hosts caps the walk identically to --auto.
    - Line-numbered error context on bad IP / scanner
      error.
  - **runDiscover** extracted from `newDiscoverCmd`'s RunE
    so the parent function stays under the funlen
    ceiling as new flags accrete.

## Test coverage (7 cases)

  - HappyPath          — file → 3 IPs (comments + blanks
                         skipped, inline comment preserved)
  - HostPortStrip      — host:port line yields just the host
  - IPv6Preserved      — `::` lines kept verbatim through
                         the port-strip heuristic
  - MaxHostsCap        — 4-line file with cap 2 → 2 entries
  - EmptyFile          — comments-only file → "no hosts
                         parsed" typed error
  - BadIP              — netip.ParseAddr error wrapped with
                         "line N" context
  - MissingFile        — os.Open error wrapped with the
                         file path

All pass under -race; lint clean.
