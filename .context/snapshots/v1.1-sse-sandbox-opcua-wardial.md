# Snapshot — v1.1: SSE + seccomp + OPC UA + wardial batch

Closed **2026-04-21**. Ships the eight feature chunks and the
close-and-tag step. The DB-backed half of chunk 4 (findings /
triage / runs tables + dashboard panels reading from DB) is the
single carry-over: it lands alongside the DB-backed audit Writer
in v1.2 so the two cross-cutting DB migrations ship as one
review unit.

## Chunks

### Chunk 1 — per-plugin offensive WriteGatedHandler (`f5d8902`)
- `offensive/write/<proto>/gatedproxy.go` for modbus / s7 / enip
  (full wire-level Handle with protocol-native refusal frames)
  + bacnet / dnp3 / iec104 / hartip / atg / fox (session
  primitives only — AllowlistHash + SessionMutation). Closes
  ADR-040 on the offensive side; the default-build wire-layer
  write-ban stays intact.
- Allowlist hash is order-insensitive (sort then SHA-256 over
  target + entries). Per-frame decisions consult `Allowed`; no
  per-frame Authorize.

### Chunk 2 — file-backed audit writer (`cd58f40`)
- `internal/audit/writer.go`: `FileWriter` at
  `~/.elsereno/audit.jsonl` (mode 0600, parent dir 0700).
  Chain-resumable across process restart (scans last line).
  One JSON-encoded `audit.Entry` per line; `audit.VerifyFile`
  walks the chain and returns `ErrChainBroken` on mismatch.
- `offensive/confirm/adapter.go`: `NewAuditor(w, actor)` maps
  `confirm.AuditEvent` → `audit.Entry` without schema
  migration (category → `EventOff*` enum; disposition /
  operation / target / payload_hash / reason embedded in the
  JSONB payload).

### Chunk 3 — offensive CLI network delivery (`4df28e4`)
- `cmd/elsereno/offensive_runtime.go`: `newOffensiveRuntime`
  opens the vault + audit writer + wraps into a
  `confirm.Auditor`. `rt.Close()` releases the file handle.
- `write modbus send` (Execute → wire), `exploit run` (dial +
  send + read reply), `audit verify-file` (walk JSONL chain
  from CLI).

### Chunk 4a — SSE /api/v1/stream (`fc7c4fe`)
- `internal/web/stream`: channel-per-subscriber Broadcaster
  with slow-subscriber-dropped policy. Event kinds: `finding`
  / `run_start` / `run_end` / `audit`.
- `internal/web/handlers/stream.go`: SSE wire handler
  (event/id/data framing + 3s retry hint + 15s keepalive).
- `internal/audit/FileWriter.SetObserver` post-append hook.
- `stream.TailAudit` cross-process file tailer so offensive
  verbs in separate processes light up the dashboard.
- Dashboard inline template: live-feed panel (EventSource,
  CSP-nonce script). `internal/web/httpctx` threads the nonce
  through request context.
- E2E: audit.jsonl append from a separate process → SSE event
  framed correctly in under 500 ms.

### Chunk 4b — DB tables + panels (CARRY-OVER to v1.2)
Findings / triage / runs DB tables + dashboard panels reading
from DB land with v1.2 when the DB-backed audit Writer ships.
The live feed above already tails everything the process
publishes, so dashboards aren't blind in the interim.

### Chunk 5 — GHCR docker image (`2fa03d3`)
- `.goreleaser.yml` `dockers_v2` block: multi-arch
  (linux/amd64 + linux/arm64) → `ghcr.io/robinr00t/elsereno:
  <tag>` + `:latest`. `sbom: true` (CycloneDX attestation) +
  `flags: [--pull, --attest=type=provenance,mode=max]`.
  `docker_signs` cosigns the manifest keyless.
- `.github/workflows/release.yml` adds
  `docker/setup-qemu-action@v3` + `docker/setup-buildx-action@v3`
  before the ghcr login step so buildx is ready for
  multi-platform builds + attestations.
- `Dockerfile` pinned to `golang:1.25.4-alpine3.22`;
  `Dockerfile.sqlite` to `golang:1.25.4-bookworm` for the
  CGO/SQLCipher variant.
- README + RELEASING document pull + cosign verify recipes.

### Chunk 6 — seccomp-bpf sandbox
- `offensive/sandbox/bpf_linux.go`: compiles denylist BPF
  programs per profile (exploit / harvest / dial) — prologue
  (LD arch / JEQ auditArch / RET KILL / LD nr) + one JEQ per
  blocked syscall + tail RET ALLOW / RET ERRNO|EPERM.
- `installFilter` registers via
  `seccomp(SECCOMP_SET_MODE_FILTER, TSYNC)` so the filter
  covers every thread in the thread-group.
- `syscalls_linux.go` tables for x86_64 + aarch64 (zero
  entries dropped so unsupported syscalls don't accidentally
  match `read` at nr=0).
- New `audit.EventOffSandbox` + goose migration 00002.
  `offensiveRuntime.ApplySandbox(ctx, profile)` records
  profile + availability in the audit chain.
- Wired into `write modbus send`, `exploit run`, `harvest *`,
  `dial batch`. Integration tests (build tag
  `sandbox_integration`) fork a child + verify ptrace +
  socket return EPERM on native Linux.
- Legacy top-level `migrations/` dir removed —
  `internal/db/migrations/` is the single source of truth.

### Chunk 7 — OPC UA plugin (`bd90591`)
- `internal/protocols/opcua/wire/`: UA-TCP Part 6 §7.1
  Hello / Acknowledge / Error parser + encoder. 1 MiB message
  ceiling.
- `internal/protocols/opcua/opcua.go`: port 4840 probe. Sends
  HEL, classifies ACK / ERR / non-UA bytes. UA-confirmed
  findings score severity=high (66). Default ProxyHandler
  refuses with a UA-native ERR frame.
- `simulators/opcua/`: minimal CI responder (ack | err modes).
- Write gating (SecureChannel + Session + Write service)
  deferred to v1.2 — full UA binary encoding surface is too
  large for v1.1's remaining scope.

### Chunk 8 — wardialing batch (`8895148`)
- `offensive/dial/batch.go`: `Batch.Run` classifies a list of
  numbers against the ADR-041 dial guard (Normalise → ≤3-digit
  hard block → scope.blocked_numbers) and appends one
  `offensive_dial` audit entry per decision.
- `elsereno dial batch --numbers-file <path> --scope <yaml>`
  (stdin if `--numbers-file` omitted). Seccomp `dial` profile
  installed before classification. Default disposition =
  "preview" (audit-only dry-run). Actual modem / VoIP delivery
  is a v1.2 carry-over.
- Former `elsereno dial` single-number check preserved as
  `elsereno dial validate`.

## Retrospective

### What went well
- **SSE framing**: the Broadcaster / cross-process TailAudit /
  dashboard EventSource pipeline was end-to-end testable in one
  afternoon because the three layers (broadcast, transport,
  consumer) are genuinely decoupled. The `Observer` hook on
  `FileWriter` means in-process publishers don't need to know
  about web/stream, and the tailer handles multi-process.
- **Docker image**: `dockers_v2` with `sbom: true` is a cleaner
  path than the old `dockers:` + manual sbom hooks. The
  `labels:` map + `flags:` array for buildx `--attest` flags
  fell into place once the v2 schema was clear.
- **Seccomp denylist**: picking denylist over allowlist kept
  the Go runtime happy (GC needs ~50 syscalls) while still
  blocking the escape paths ADR-042 calls out. TSYNC is a
  one-line flag that quietly solves the goroutine-hops-onto-
  unfiltered-thread concern.
- **OPC UA scope discipline**: punting the write-gating to
  v1.2 kept chunk 7 small (~1000 lines including simulator +
  tests) and let us ship the fingerprint without opening the
  whole UA binary encoding rabbit hole.

### Pitfalls hit
- **Docker Desktop's amd64 emulation** rejects seccomp
  filters with EINVAL even under `--security-opt
  seccomp=unconfined`. The integration tests live behind a
  build tag (`sandbox_integration`) and run on native arm64
  Linux in containers; they're green there. On CI with a
  real x86_64 runner this should Just Work — the filter
  install was verified on native arm64 locally.
- **Legacy top-level `migrations/`** was shadowing the
  authoritative `internal/db/migrations/` and the
  events-vs-SQL sync test was reading the wrong copy. Removed
  the top-level dir; the test now walks every migration in
  order and takes the last CHECK block.
- **Goreleaser v2 deprecations**: `dockers:` +
  `docker_manifests:` print a "phase-out" warning. The
  `dockers_v2:` schema differs (labels map instead of
  --label templates; `flags:` instead of
  `build_flag_templates:`). One round of `goreleaser check`
  + schema dump was enough to converge.
- **`log.Fatal` vs `defer cancel()`**: gocritic's
  `exitAfterDefer` rule caught the simulator's main. Split
  main/run so defers fire before the outer `os.Exit`.

### Carry-overs
- **v1.2 chunk 4b**: findings / triage / runs DB tables +
  panels. Lands with the DB-backed audit Writer so both
  migrations ship as one review unit.
- **v1.2 OPC UA write gating**: full SecureChannel + Session +
  Write service handling. Will live under
  `offensive/write/opcua/gatedproxy.go` following the
  chunk 1 template.
- **v1.2 PSTN / VoIP dial backend**: the batch currently
  records intent; a real modem or VoIP integration ships with
  the backend that actually dials.
- **v1.2 bacnet / dnp3 / iec104 / hartip / atg / fox full
  Handle**: session primitives only in v1.1; full wire-level
  forwarding loops arrive with the ASDU / APDU / HART-command
  parsers.
- **SLSA `.intoto.jsonl`**: generator v2.1.0 still hits the
  upstream `final` step exit-27. v1.1 release wraps the
  outcome in a non-blocking gate, same as v1.0.1. v1.2
  tracks dropping the reusable workflow.
