# Snapshot — v1.29.0: TUI verb + mini build variant

Six-chunk cycle that adds the operator-facing TUI alongside the
existing batch verbs, and introduces the **mini** build variant —
a stripped binary suitable for resource-constrained device
deployments (jump hosts, hardened workstations, embedded
inspection rigs).

The TUI is **purely additive**: every existing batch verb (scan,
write, exploit, harvest, dial, audit, proxy listen, …) keeps its
current behaviour. Operators opt into the TUI via the `tui` verb;
the cycle introduces zero changes to the offensive runtime,
scoring, audit, or web layers.

## Chunks

### Chunk 1 — Mini build scaffolding (`f70bbd5`)

3-variant build matrix:

| Variant   | Tags        | TUI | Web | Use case                |
|-----------|-------------|-----|-----|-------------------------|
| default   | (none)      | ✓   | ✓   | Workstation / SOC       |
| offensive | `offensive` | ✓   | ✓   | Authorised pen-testing  |
| mini      | `mini`      | ✗   | ✗   | Device / embedded       |

Mini excludes the dashboard (`serve`, `api`) + the TUI verb. Stub
verbs print descriptive errors + exit `EX_UNAVAILABLE` (69)
instead of cobra's "unknown command", so an operator who runs the
wrong tarball gets pointed at the default binary.

Goreleaser ships 3 archives per OS/arch (linux, darwin × amd64,
arm64); Makefile gains `build-mini`; CI gates `build-mini` to
catch tag bitrot (PITF-031 enforced).

Stripped binary sizes (`-s -w`):

  * default     ~ 23 MB
  * offensive   ~ 24 MB
  * mini        ~ 21 MB

### Chunk 2 — TUI core (`dc15e69`)

`internal/tui/` (build tag `!mini`) — bubbletea Model/View/Update
scaffolding. Pure-data Model with rolling windows (200 findings,
100 audit events), severity buckets mirroring ADR-006 bands,
focus-aware key routing (Tab cycles 4 panes; j/k/g/G drive the
findings cursor only when that pane is focused).

`internal/tui/feeds/` defines the `Feed` interface; chunk 2 ships
`Empty` (no events; lets operators verify the program starts
clean).

Side fix in `cmd/elsereno/main.go`: `cliError`-wrapped errors
were exiting silently because cobra's `SilenceErrors:true`
suppressed them and most call sites of `fail()` don't print
before returning. The fix prints `cliError.Error()` to stderr in
the entrypoint's typed-exit branch — surfaces every failed
operator command without changing any subcommand.

16 tests (`model_test.go` + `update_test.go`).

### Chunk 3 — TUI replay mode (`ac10f03`)

`feeds.Replay` reads `ndjson:v1` capture files (the same shape
emitted by `elsereno scan --output-format ndjson`) and streams
each line as a `tui.FindingMsg`. Optional `Rate` (lines/s) for
demo-paced playback. Per-line parse failures become AuditMsgs
("ndjson: skipped malformed line N: …") rather than aborting the
feed — a single corrupt line shouldn't kill a long capture.
Pre-flight `os.Stat` in `cmd_tui.go` so missing-file typos exit
fast (EX_USAGE 64) before the alt screen takes over.

8 tests.

### Chunk 4 — TUI feed mode (`7332de1`)

`feeds.Stdin` — live producer pipe. Pairs with the same
ndjson:v1 emitter, streamed:

```
elsereno scan ... --output-format ndjson | elsereno tui --feed -
```

Refactor: chunk 3's per-line scanner extracted to a shared
`streamNDJSON(ctx, src, emit, pace)` helper. Both Replay (file)
and Stdin (live pipe) funnel through it; the Watch feed (chunk 5)
parses SSE and routes through the same parseRecord on the
decoded data: payload.

CLI rejects `--feed FILE` with a clear pointer to `--replay` —
keeps the two flags' intent distinct. 7 tests.

### Chunk 5 — TUI watch mode (`33260c6`)

`feeds.Watch` — read-only SSE consumer for the dashboard's
`/api/v1/stream`. Decodes the SSE wire format (event/id/data with
multi-line data: concat) and dispatches by kind:

  * `finding`   → tui.FindingMsg
  * `audit`     → tui.AuditMsg ("audit <kind> @ ts by actor")
  * `run_start` → tui.AuditMsg ("run X started by Y @ ts")
  * `run_end`   → tui.AuditMsg ("run X ended (status); counts {…}")
  * unknown     → tui.AuditMsg ("watch: unknown event …") — no
                  silent drops on schema bumps

Auto-reconnect with 3s default interval (matches the server's
`retry:` hint). 401/403 wrapped in `authError` short-circuits the
retry loop — looping on a bad token is just noise. ADR-005
isolation: payload structs are duplicated locally rather than
imported from `internal/web/stream` so the TUI compiles in builds
without the web tree.

13 tests covering happy path, every event kind, auth, retry
exhaustion, content-type guard, keepalive comments, multi-line
data, malformed JSON, context cancel.

### Chunk 6 — Docs (`f147cde`)

`.context/tui.md` — operator-facing reference (architecture +
key bindings + wire formats + build-tag matrix). INDEX.md +
STATE.md + CHANGELOG.md mirrored.

## Operator usage

After this cycle:

```
# Replay a previous scan
elsereno tui --replay shift-2026-04-29.ndjson

# Live triage during a long scan
elsereno scan --input targets.txt --output-format ndjson | \
  elsereno tui --feed -

# Watch a remote dashboard
elsereno tui --watch https://jumphost:8443/api/v1/stream \
            --bearer "$(pass elsereno/dashboard-token)"
```

Every batch verb keeps its prior behaviour; the TUI is opt-in.

## Known limitations

- **Interactive mode** (no flag) currently uses `feeds.Empty`. The
  in-TUI scan launcher is a future enrichment — for now drive the
  TUI from `--feed -` against a live `elsereno scan` pipe.
- **Audit-pane filtering** (e.g. show only `audit` events) is not
  yet wired; operators see the full stream chronologically.
- **`teatest` integration tests** deferred; per-component coverage
  is exhaustive.
