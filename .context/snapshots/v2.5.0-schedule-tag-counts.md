# Snapshot — v2.5.0: schedule tag-counts aggregate

Single-chunk cycle. Closes v2.4 carryover #2: operators
want a fleet-wide tag distribution view ("how many
schedules are tagged 'prod'?") for autocomplete + tag
clouds without listing every schedule.

## REST

```
GET /api/v1/schedules/tags
```

Returns:

```json
{
  "schema": "api:v1",
  "data": [
    {"tag": "prod",     "count": 12},
    {"tag": "critical", "count":  3},
    {"tag": "dev",      "count":  2}
  ]
}
```

Sorted by count DESC, tag ASC. Empty store → `data: []`
(never `null`).

## Store

`Store.TagCounts(ctx) ([]TagCount, error)` (new) + new
`TagCount` struct `{Tag, Count}`.

  - **Memory**: walk all schedules, accumulate
    `map[string]int`, sort via insertion sort
    (tag space is small, < 50 typical).
  - **PG**: single SQL with `UNNEST(tags)` + `GROUP BY` +
    `ORDER BY count DESC, tag ASC`. Cheap thanks to the
    v2.4 GIN index — though the query plan does a seq
    scan + sort on the unnest result (acceptable for
    schedules table size).

## Refactor side-quest

`Schedules()` factory exceeded the 40-statement funlen
cap after the new route. Split into:
  - `schedulesUnavailableMux` (503-on-everything; iterates
    a paths slice instead of 18 explicit HandleFunc
    calls).
  - `schedulesActiveMux` (active handlers).

Schedules() is now 4 statements; future routes don't
re-trigger funlen.

## Tests (+2)

  - TestListScheduleTags: 4 schedules → 3 distinct tags;
    asserts count + sort order (prod=3, critical=2,
    dev=1).
  - TestListScheduleTags_Empty: empty store → empty
    array (decode + len==0, robust against pretty-print
    whitespace).

## Honest scope

  - Server-side sort is by count then tag. Operators
    wanting alphabetical-only or filtered-by-min-count
    can sort client-side.
  - No HEAD request support / no caching headers. The
    dashboard fetches on-demand; a future cycle could
    add ETag for dashboards polling frequently.
  - PG impl doesn't enforce an explicit transaction
    isolation level — UNNEST + COUNT happens on a
    snapshot anyway thanks to PG's default read-committed.
  - INSTALL.md unchanged. Build sizes unchanged.

## Build sizes

Unchanged across all 3 variants.

Adds ~120 LoC + 2 tests + Schedules() refactor.
