# Snapshot — v2.24.0: localStorage ETag cache

Single-chunk cycle. v2.15 introduced an in-memory ETag cache;
page refresh cleared it. v2.24 persists the cache in
`localStorage` so the 304 win survives refresh.

## Impl

  - Storage key: `elsereno:etag-cache:v1` (versioned so a
    future schema bump can reset cleanly).
  - On script load: `loadEtagCacheFromStorage()` hydrates
    the in-memory map from JSON in localStorage. Parse
    failures fall back to empty (cache is optimisation,
    not correctness — silent recovery is safe).
  - On each 200 response: in-memory map updated + persisted
    via `persistEtagCache()`. Quota / disabled-storage
    errors swallowed.
  - New escape hatch: `window.elserenoResetEtagCache()`
    clears both in-memory + localStorage. Useful when
    debugging perceived staleness after a server upgrade.

## Honest scope

  - No size-cap on the persisted cache; a fleet with
    thousands of unique URLs could exceed the typical
    5 MB localStorage quota. In practice the dashboard
    polls ~5 URLs; quota is far from a risk.
  - Cache doesn't include request method or vary
    semantics — `fetchWithETag` is GET-only by design,
    so this is fine.

## Build sizes

Unchanged across all 3 variants.

Adds ~30 LoC in dashboard.go.

## Carryover

  - **vNext**: TTL on persisted entries (current logic
    relies on the server returning a different ETag when
    content changes; stale entries are harmless but waste
    storage indefinitely).
  - **vNext**: settings page button to call
    `elserenoResetEtagCache()` so operators don't need
    devtools.
