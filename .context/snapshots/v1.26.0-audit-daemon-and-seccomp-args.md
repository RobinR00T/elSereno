# Snapshot — v1.26.0: audit daemon (UDS) + seccomp arg-filter primitives

Two-chunk cycle that ships two long-running TODO items: a
centralised cross-process audit daemon (replacing the v1.15-
chunk-4 flock at SOC scale) and seccomp BPF arg-level filtering
primitives (the ADR-042 follow-up the v1.1-chunk-6 syscall-level
denylist explicitly carried forward).

No new fingerprint plugins; default-build plugin count stays at
**27**. cve_exposure non-zero coverage unchanged at 24/27.

## Chunks

### Chunk 1 — `elsereno audit serve` daemon (UDS) (`c870858`)

A centralised single-writer audit daemon that emitter processes
fan-in to via a Unix domain socket. Cleaner alternative to the
v1.15-chunk-4 flock at SOC scale: instead of N tail-reads per N
appends (every emitter takes the lock + reads the tail to resume
the chain on every Append), the daemon holds the FileWriter
once + computes prev_hash in memory + writes once.

**Architecture**:

```
Server                              Client
──────                              ──────
(single FileWriter)                 (one per emitter process)
Listen("unix", path, mode 0600)     lazy-dial on first Append
Accept loop, one goroutine/conn     Append → JSON line over UDS
Read JSON Request                   Read JSON Response
→ Writer.Append (mutex serialises)  → resume chain or fail
Write JSON Response
```

Wire protocol: line-delimited JSON. `Request` carries an `Entry`
with operator-supplied `{OccurredAt, Actor, EventType, Payload}`;
the daemon fills `{ID, PrevHash, EntryHash}` and returns the
canonical persisted form. Easy to drive with `nc -U` for
debugging.

Public API:

  - `audit.NewServer(writer, socketPath)` — listen + chmod 0600.
    Removes stale sockets so crashes don't block restart.
    Rejects relative paths.
  - `audit.Server.Serve(ctx)` — accept loop, ctx-cancel triggers
    `Close` which removes the socket file.
  - `audit.DialClient(socketPath)` — lazy-dial. First `Append`
    establishes the socket. Re-dials transparently after a
    daemon bounce.
  - `Client.Append(ctx, entry) → (Entry, error)` — implements the
    `audit.Writer` contract. Drop-in replacement for FileWriter.

CLI: `elsereno audit serve --socket PATH --output PATH`. Both
flags required + must be absolute. SIGINT/SIGTERM trigger
graceful shutdown (close listener + remove socket file).

**8 new tests**. Highlights: TestNewServer rejects
relative-path / nil-writer / recreates on stale socket / chmod
0600. TestClient_AppendRoundTrip verifies non-zero ID + 32-byte
PrevHash/EntryHash. TestClient_ChainOrderUnderConcurrentClients
spins up 8 goroutines × 10 Appends and asserts the IDs form a
contiguous 1..80 range (no duplicates) — proves the daemon's
serialised Writer keeps chain order under concurrent fan-in.
TestClient_LazyDial confirms a missing daemon doesn't error
until first Append. TestClient_SurvivesDaemonReconnect proves
the explicit-Close + re-dial path works.

**Cross-platform**: Unix-only (server uses `net.Listen "unix"`).
Windows builds keep the v1.15-chunk-4 flock fallback.

### Chunk 2 — seccomp-bpf arg-level filtering primitives (`aa78f60`)

The ADR-042 follow-up that the v1.1-chunk-6 syscall-level
denylist explicitly carried forward. Adds the BPF building
blocks for refusing a syscall based on its arguments:

  - **Equal**: deny when `args[N]` equals one of a list of
    values. Use case: deny `socket(family, …)` when family is
    `AF_PACKET` or `AF_NETLINK`.
  - **MaskAny**: deny when `(args[N] & maskBits) != 0`. Use
    case: deny `openat(…, flags, …)` when flags has any
    write/create/truncate bit set — effective "openat read-only
    or fail".

Public API:

  - `ArgDenyRule` struct + constructors `NewArgDenyEqual` /
    `NewArgDenyMaskAny`.
  - `ArgFilterPresets(syscallNums)` returns the two canonical
    rules ADR-042 calls out (openat-no-write + socket-deny-
    AF_PACKET-and-AF_NETLINK).
  - `CompileArgFilter([]ArgDenyRule) []unix.SockFilter` — emits
    a self-contained BPF program that ORs all rules together.
  - `CompileFilterWithArgs(profile, rules)` — composes the
    existing syscall-level denylist (`FilterProgram`) with the
    arg-level rules. Arg rules run FIRST so a matching arg-
    deny returns `ERRNO|EPERM` before the syscall denylist
    gets a chance to ALLOW.
  - Added `Openat` field to `syscallNums` struct + populated
    in syscallsAMD64 (NR=257) and syscallsARM64 (NR=56). Was
    missing despite being an obvious arg-filter target.

**11 new tests**. Constructor helpers, preset rule composition,
seccomp_data offset arithmetic (16 + idx*8), instruction count
on mask + equal modes, CompileFilterWithArgs composition,
ruleBodyLen vs emitRuleBody parity (the jump-offset invariant).

Wiring into specific profiles (ProfileExploit / ProfileHarvest /
ProfileDial) is intentionally NOT in this chunk — that's a
v1.27 operator-preference call. The primitives + presets ship
now; profile integration follows once operators decide which
arg-filters they want enforced where.

**Cross-platform**: file is `//go:build offensive && linux`. On
non-Linux builds the arg-filter primitives are silently absent
(the syscall-level denylist + `PR_SET_NO_NEW_PRIVS` still apply).
Cross-compiles cleanly to both linux/amd64 and linux/arm64.

## Carry-overs to v1.27+

- **Wire seccomp arg-filter into specific profiles**.
  Operator-preference call:
    - ProfileHarvest → openat-no-write (already emits findings
      to stdout, doesn't need to write files in the harvest
      sandbox).
    - ProfileDial → socket-deny-AF_PACKET / AF_NETLINK (the dial
      subprocess only needs IPv4/IPv6 sockets).
    - ProfileExploit → likely no preset; exploits sometimes
      legitimately need `openat(O_CREAT)` for state files.

  ProConOS (both blocked on real-PLC test vectors), offensive
