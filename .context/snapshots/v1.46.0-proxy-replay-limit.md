# Snapshot — v1.46.0: proxy replay --limit N

Single-chunk DX cycle. Caps `elsereno proxy replay` output
at N matching chunks. Composes with --dir / --since /
--until / --json filters; the cap fires AFTER those so an
operator picking the first N events in a window gets
exactly N.

## Workflow

```
$ elsereno proxy replay session.ndjson \
    --since 2026-05-04T12:00:00Z \
    --dir client \
    --limit 10 \
    --json | jq '.len'
# 10 lengths, then EOF.
```

The cap fires inside the file walker — the replay
package returns from `Replay()` as soon as the sentinel
`errReplayLimitReached` is observed — so multi-GB
captures don't keep being read after the operator has
seen enough.

## Chunks

### Chunk 1 — --limit + chunkPassesFilters (`95deea4`)

`cmd/elsereno/cmd_proxy_replay_offensive.go`:

  - **--limit int** flag (default 0 = no cap; preserves
    pre-v1.46 streaming).
  - **errReplayLimitReached** sentinel returned from the
    replay.Replay callback when emitted >= limit. The
    dispatcher recognises it via errors.Is and returns
    nil so the operator sees no error on the way out.
  - **chunkPassesFilters** helper extracted from the
    callback. Encapsulates DirHeader skip + dir filter +
    timeWindow.contains. Drops runProxyReplay's
    cyclomatic complexity from 16 (over the ceiling) back
    to a comfortable single digit.
