# Snapshot — v1.82.0: AbortController on /preview

Single-chunk cycle. Dashboard-only JS. v1.80 debouncing
delayed dispatch but didn't cancel in-flight requests;
fast-typing operators could see stale previews flash
between the cancelled and new responses. v1.82 fixes the
race with `AbortController`.

## Implementation

```js
var previewAbortController = null;
// Inside previewNextFire, just before fetch():
if (previewAbortController) {
  try { previewAbortController.abort(); } catch (_) {}
}
previewAbortController = (typeof AbortController !== "undefined")
  ? new AbortController() : null;
var signal = previewAbortController ? previewAbortController.signal : undefined;
fetch(url, { ..., signal: signal })
// In catch:
if (e && (e.name === "AbortError" || (e.message || "").indexOf("abort") !== -1)) return;
```

The `typeof AbortController` guard keeps the dashboard
functional on browsers that pre-date the API (IE11, Edge
<16). There the signal is undefined and fetch ignores it,
falling back to v1.80 behaviour (stale-flash possible).

- Pre-2018 browsers run without cancellation.
- No "preview cancelled" UI.
