# Snapshot — v1.6.0: --allow-file + OPC UA per-NodeId allowlist

Closed **2026-04-23**. Two operator-value chunks:
allowlist-from-YAML ergonomics + the first per-object tightening
of a write-gate.

## Chunks

### Chunk 1 — `elsereno proxy listen --allow-file` (`a5ee374`)
- `cmd/elsereno/cmd_proxy_allowfile_offensive.go` NEW. YAML
  schema per plugin:

      plugin: sip                    # or iax2 / pbxhttp / modbus / opcua / bacnet
      target: pbx.example.com:5060
      methods: [INVITE, REGISTER]   # per-plugin field

- `loadAllowFile(path, &opts)` uses `yaml.NewDecoder.KnownFields(
  true)` so typos (`method:` vs `methods:`) fail noisily rather
  than silently dropping the operator's allowlist.
- `runProxyListen` calls `loadAllowFile` FIRST if `--allow-file`
  is set, then the existing `validateProxyListenOpts`. Command-
  line flags act as defaults when the file doesn't override them.
- Plugin-name constants (`pluginNameSIP`, `pluginNameIAX2`, …)
  promoted to shared constants so goconst stays clean across
  both `cmd_proxy_offensive.go` and the loader.
- 10 tests: one happy-path per plugin + missing plugin +
  missing target + unsupported plugin + unknown field (typo) +
  nonexistent file.

### Chunk 2 — OPC UA per-NodeId allowlist (`f76b3f2`)
Tightens the v1.2 OPC UA write-gate from "any WriteRequest"
to "WriteRequest to specific NodeIds".

- `internal/protocols/opcua/wire/writerequest.go` NEW.
  - `NodeID{Namespace, Identifier}` — parsed form for numeric
    encodings.
  - `WriteRequestFirstNode(msgBody)` walks past the 16-byte
    MSG prefix, the 4-byte TypeId, the variable-length
    RequestHeader (AuthToken NodeId + Timestamp + RequestHandle
    + ReturnDiagnostics + AuditEntryId + TimeoutHint +
    AdditionalHeader ExtensionObject), then reads the
    NodesToWrite array length + the first WriteValue's NodeId.
  - Supports TwoByte + FourByte + Numeric encodings. Rarer
    encodings (String / Guid / ByteString) are structurally
    consumed but return ok=false so the caller fails closed.
  - `parseRequestHeader` split into `skipAuditEntryAndTimeout`
    + `skipAdditionalHeader` to keep funlen under 40 stmts.

- `offensive/write/opcua/gatedproxy.go` (extended).
  - `AllowedNodeID{Namespace uint16, Identifier uint32}` new
    type. Optional `AllowedNodeIDs []AllowedNodeID` field on
    `WriteGatedHandler`.
  - `AllowlistHashWithNodeIDs(target, services, nodeIDs)` new
    hash. **Backwards-compatible**: when nodeIDs is nil/empty,
    hash equals the v1.2 `AllowlistHash` so existing operator
    tokens remain valid.
  - `SessionMutationWithNodeIDs(...)` factory. `Authorise()`
    now calls this variant which degrades cleanly for empty
    NodeIDs.
  - `routeFrame` does a second-stage NodeId check when
    `AllowedNodeIDs` is set AND the frame is a WriteRequest.
    `writeRequestNodeAllowed` parses the body's first NodeId
    and matches against the allowlist.

- `offensive/write/opcua/nodeid_test.go` NEW — 8 tests.
  - AllowlistHashWithNodeIDs: empty-nodes-equals-v1.2, nodes-
    change-hash, order-insensitive.
  - WriteRequestFirstNode: FourByte numeric happy path +
    short-frame fail.
  - Gate E2E: NodeID allowed forwards, NodeID blocked returns
    ServiceFault with upstream seeing nothing, empty
    AllowedNodeIDs falls back to v1.2 service-level gate.
  - `safeBuffer` wrapper fixes a race between the recorder
    goroutine and test assertions on the forwarded bytes.

## Behaviour matrix (OPC UA gate)

  TypeID in Allowed?   AllowedNodeIDs?   NodeId matches?   Result
  ----------------------------------------------------------------
  no                   (any)             (any)             refuse
  yes                  nil / empty       (any)             forward (v1.2 fallback)
  yes                  set               yes               forward
  yes                  set               no                refuse
  yes                  set               unparseable       refuse (strict)

## Retrospective

### What went well
- Backwards compatibility on the OPC UA hash: operators who
  already minted v1.2 tokens don't need to re-mint unless
  they opt into per-NodeId gating.
- The YAML loader + `KnownFields(true)` caught typos early in
  testing — one test case intentionally mistypes `method:` vs
  `methods:` to prove the rejection path.
- Both chunks land with full test coverage and 0 lint issues.

### Pitfalls hit + fixed
- Race on the upstream buffer in the OPC UA E2E test. Fixed
  with a `safeBuffer` mutex-wrapped `bytes.Buffer`.
- gosec G115 on narrowing uint16→byte / uint32→uint16 in the
  test helper `buildMSGBody`. Fixed with explicit `& 0xFF` /
  `& 0xFFFF` masks + a rationale comment.
- funlen on `parseRequestHeader` (41 > 40). Split into two
  helpers, each under 30 stmts.
- goconst flagged repeated "sip" / "iax2" string literals.
  Promoted to package-level constants.
- `unparam` flagged `buildMSGBody`'s typeID parameter always
  being WriteRequest. Dropped the parameter; the helper is
  now WriteRequest-only (and documented that way).

### Carry-overs to v1.7+
- Match against String / Guid / ByteString NodeId encodings
  for OPC UA per-NodeId gate.
- Multi-node-per-WriteRequest allowlist: today only the first
  WriteValue's NodeId is checked. A 5-node WriteRequest where
  some nodes are denied still passes if the first node is
  allowed.
- CallRequest per-object allowlist (sibling of WriteRequest).
- BACnet per-object allowlist (same shape, ASN.1 BER parsing).
- SIP To-URI E.164 prefix allowlist for INVITE.
- REGISTER AOR allowlist.
- CWMP offensive proxy (SOAP RPC allowlist).
- SIGHUP reload of the proxy allowlist (non-trivial because
  changing the allowlist changes the PayloadHash which
  invalidates the existing confirm-token).
- `elsereno write <plugin> dry-run --emit-allow-file` to
  generate the YAML file from the dry-run output.
