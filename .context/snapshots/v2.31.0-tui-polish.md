# Snapshot — v2.31.0: TUI findings-pane polish

Single-chunk cycle. Two small UX additions to the
findings pane during incident triage.

## What changed

### Header carries count + cursor position

Old: `Findings:  (cursor → ▸)`.
New: `Findings (N total · cursor on M):  (▸ = active row)`.

Operators triaging 100+ findings know where they are
in the scroll buffer at a glance — previously they had
to count rows.

### Severity column colour-coded

The `severityColor()` helper existed but was unused in
this render path. v2.31 wires the label so:
  - critical → bright red (196)
  - high     → orange (208)
  - medium   → yellow (220)
  - low      → blue (39)
  - info     → grey (245)

At-a-glance triage scan; especially useful in mono-
chrome SSH sessions where ANSI colour is the only
signal.

## Honest scope

  - Truncated-to-10-char severity label still applied;
    color is the new value.
  - No filter shortcut (`/`) on findings pane yet —
    deferred.
