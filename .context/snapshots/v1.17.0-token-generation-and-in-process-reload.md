# Snapshot — v1.17.0: token-generation parity + in-process allow-file reload

Five chunks delivering operator-facing in-process allow-file
reload across every offensive write-gated proxy. Builds on the
v1.16 chunk-4 BACnet token-generation foundation: chunks 1-3
roll out the cookie pattern to the remaining 6 plugins; chunks
4-5 wire the SIGUSR1 signal handler + atomic swap + audit
event. Operators no longer need supervisor-restart for
allowlist edits — `kill -USR1 $pid` swaps in-place with no
connection drop.

## Chunk 1 — CWMP token-generation cookie

`ed868af feat(v1.17 chunk 1): CWMP token-generation cookie +
shared --token-generation flag`

Extends the v1.16 chunk-4 BACnet token-generation pattern to
CWMP. New `AllowlistHashWithGeneration` /
`SessionMutationWithGeneration` at the top of the CWMP hash
ladder; separator `0xFC` (below 0xFD firmware, 0xFE paths). New
`TokenGeneration uint32` field on `cwmp.WriteGatedHandler`.

CLI: the v1.16-chunk-4 BACnet `--token-generation N` flag was
promoted from the BACnet-specific flag registrar to the shared
session-flag registrar so both `proxy listen --plugin <name>`
invocations pick it up. `write cwmp dry-run --token-generation
N` flag added.

Refactor: shared `parseBACnetTypeInstance` helper unifies the
common `type=N;instance=M` parser surface across BACnet flag
families.

7 new tests (`tokengeneration_test.go`) covering hash-ladder
degradation, distinctness, determinism, and the E2E Authorise
matrix.

## Chunk 4 — SIGUSR1 in-process allow-file reload

`(commit hash to follow at cycle-close) feat(v1.17 chunk 4):
SIGUSR1 in-process allow-file reload + atomic swap`

Delivers the operator-facing in-process reload that the
chunks-1-3 token-generation parity made possible.

Architecture:
- New `reloadableHandler` (cmd/elsereno/cmd_proxy_reload_
  offensive.go) wraps a `gatedProxyHandler` behind
  `atomic.Pointer[gatedProxyHandler]`. Implements both Handle +
  Authorise as thin delegators. Handle reads the pointer once
  at entry so in-flight Handle goroutines see their own
  snapshot — a swap mid-call doesn't tear allowlist state.
- `performReload` re-loads the allow-file (via
  `loadAllowFile`), reads the sidecar `<allow-file>.token`
  (0600 enforced via `readSidecarToken`), builds + authorises
  a new handler, and atomic-swaps it into the wrapper.
- `freshReloadOpts` clears plugin-specific allowlist fields
  while preserving immutable session bits (target, listen,
  timeouts) — necessary because `loadAllowFile` appends rather
  than overwrites for some fields.

Wire-up:
- New `--reload-allow-file` boolean flag on `proxy listen`
  (registered in the shared session-flags registrar). Requires
  --allow-file (validation in validateProxyListenOpts).
- `wrapForReload` returns a `*reloadableHandler` only when
  `--reload-allow-file` is set; otherwise pass-through (byte-
  identical to pre-v1.17 behaviour for non-reload runs).
- Extracted `runProxyServer` from runProxyListen to keep the
  latter under funlen.
- `startReloadWatcher` spawns a goroutine that selects on
  ctx.Done + SIGUSR1; on USR1 it calls performReload + logs
  failure to stderr.

Operator workflow:
1. Edit allow-file (typically bumping `token_generation:`).
2. Run `write <plugin> dry-run --token-generation N` to mint
   a fresh confirm-token.
3. Write the new token to `<allow-file>.token` (chmod 600).
4. `kill -USR1 $pid`. The proxy reloads in-place.

Co-existence with v1.15 chunk-5 SIGHUP:
- SIGHUP still exits 75 / EX_TEMPFAIL (supervisor-restart).
- SIGUSR1 reloads in-place when `--reload-allow-file` is set.
- Operators choose per-deployment which model to use.

11 new tests in `cmd_proxy_reload_offensive_test.go`.

## Chunk 5 — proxy_allowlist_reload audit event

`02fef1e feat(v1.17 chunk 5): proxy_allowlist_reload audit
event`

Every SIGUSR1 reload (success or failure) emits a dedicated
audit-chain entry. Operators can grep
`event_type=proxy_allowlist_reload` to audit reload cadence +
reject reasons across long-lived proxy sessions.

- `audit.EventProxyAllowlistReload = "proxy_allowlist_reload"`
  added to internal/audit/events.go + AllEventTypes.
- Migration 00003_audit_proxy_allowlist_reload_event_type.sql
  extends the SQL CHECK enumeration. Down path refuses if rows
  of the new type exist (matches the v1.1 chunk-6 / v1.2
  migration pattern).
- performReload emits the audit at every exit point: pre-load
  failure, sidecar-token failure, build-handler failure,
  authorise failure, swap success.
- emitReloadAudit best-effort: a failed audit-chain write
  doesn't block the swap.
- The internal/audit/events_test.go migration-sync test picks
  up the new event automatically.

Audit payload (JSONB, one row per SIGUSR1 firing):
- status: "ok" | "failed"
- plugin / target / allow_file / token_generation
- old_hash_prefix / new_hash_prefix: 8-hex-char correlation
  handles
- reason: present only on status=failed

2 new tests covering nil-writer no-op + hash-prefix stability.

## Operator surface at end of v1.17

Every offensive write-gated proxy supports:
- `proxy listen --plugin <p> --token-generation N` (shared
  session flag).
- `proxy listen --plugin <p> --reload-allow-file` (opt-in
  SIGUSR1 reload; requires --allow-file).
- `write <p> dry-run --token-generation N` for the 7 plugins.
- YAML `token_generation:` field in --allow-file for round-
  trip.

Reload workflow per-plugin (BACnet, CWMP, SIP, Modbus, IAX2,
pbxhttp, OPC UA):
1. Edit allow-file → bump `token_generation:` + change
   allowlist.
2. `write <p> dry-run --token-generation N+1 --allow-file
   <path>` → mints fresh token.
3. Write fresh token to `<path>.token` (chmod 600).
4. `kill -USR1 $pid`.
5. Audit chain rows: success or failure with full reason.
