# Snapshot — v2.25.0: Idempotency-Key on /clone + bulk

Single-chunk cycle. v2.18 wired Idempotency-Key only to
`/import`. v2.25 extends to `/clone` and the bulk
enable/disable endpoints via a reusable middleware
wrapper.

## Helper

`withIdempotencyKey(h http.Handler) http.Handler` —
generic wrapper:

  - No `Idempotency-Key` header → pass-through (no
    bookkeeping cost).
  - Buffers request body upfront so the inner handler can
    still read it.
  - Cache lookup:
    - **Hit** → emits cached response, short-circuits.
    - **Conflict** → 409.
    - **Miss** → invokes inner handler against a
      `idempotencyResponseRecorder` that captures
      status + body, caches the rendered bytes on
      2xx, lets 4xx/5xx pass through uncached.

## Wired routes

  - `POST /api/v1/schedules/{id}/clone`.
  - `POST /api/v1/schedules/bulk/enable`.
  - `POST /api/v1/schedules/bulk/disable`.

Bulk endpoints have no request body — empty-body hash
fingerprints the retry safely.

## Tests (+1)

  - TestClone_IdempotencyKey_Replays: 2 calls with same
    key + body → 1 clone in store + replay header set on
    2nd call.

## Honest scope

  - Only 2xx cached. 4xx/5xx pass through unrecorded so
    error states don't get permanently glued to a key.
  - Per-process cache (same as v2.18). v2.26 lands the
    PG-backed multi-process variant.
