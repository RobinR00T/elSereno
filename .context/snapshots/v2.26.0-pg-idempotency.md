# Snapshot — v2.26.0: PG-backed Idempotency cache

Single-chunk cycle. v2.18+v2.25 cache was per-process; load-
balanced serve deployments lost replay on retry-to-other-
worker. v2.26 lands a PG-backed cache so the entire fleet
shares replay state.

## Schema (migration 00018)

```sql
CREATE TABLE idempotency_keys (
    key            TEXT NOT NULL PRIMARY KEY,
    body_hash      TEXT NOT NULL,
    status_code    INTEGER NOT NULL,
    response_body  BYTEA NOT NULL,
    created_at     TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
CREATE INDEX idx_idempotency_keys_created_at ON idempotency_keys (created_at);
```

## Refactor

Old global `defaultIdempotencyCache` was a concrete struct.
v2.26 swaps to an interface:

```go
type IdempotencyStore interface {
  Lookup(ctx, key, body) (idempotencyLookupResult, idempotencyEntry)
  Store(ctx, key, body, status, response)
}
```

  - `idempotencyCache` (v2.18 in-memory) implements it.
  - `PGIdempotencyCache` (v2.26 new file
    idempotency_pg.go) implements it.
  - Global is a `sync.RWMutex`-guarded
    `IdempotencyStore` var; `SetDefaultIdempotencyCache`
    swaps; `idempotencyStoreNow()` reads.

Handler middleware + writeJSONAndCache + tryReplayIdempotency
all flow ctx through to the store.

## PG impl

  - **Lookup**: SELECT by key + created_at >= cutoff
    (TTL filter at read time so expired rows are
    miss without needing a hot prune).
  - **Store**: INSERT ON CONFLICT (key) DO NOTHING —
    first-write-wins. Concurrent retries don't clobber
    each other.
  - **PruneExpired(ctx)**: DELETE WHERE created_at <
    cutoff. Operator runs periodically (cron / future
    scheduled task).
  - Errors silently swallowed (idempotency is an
    optimisation; degrade gracefully).

## cmd_serve wiring

When `--scan-store=db` AND the pool is open:

```go
handlers.SetDefaultIdempotencyCache(handlers.NewPGIdempotencyCache(pool))
```

Operators on memory mode keep the v2.18 in-memory cache
(per-process; documented).

## Tests

Build + lint + cross-OS lint pass. Live PG integration test
for PGIdempotencyCache requires dev-db; deferred to whatever
cycle adds the first PG-integration suite.

## Honest scope

  - No automatic prune. Operator runs PruneExpired() on a
    schedule. A vNext cycle can hook it into the v1.87
    background pruner pattern.
  - INSERT-failure is silent — operator never sees a
    "cache write failed" warning. Acceptable: the next
    retry just runs the inner handler again.
  - INSTALL.md unchanged. Build sizes unchanged.

## Build sizes

Unchanged across all 3 variants.

Adds migration 00018 + ~120 LoC interface/impl + ~10 LoC
cmd_serve wiring.

## Carryover

  - **vNext**: background prune goroutine running
    PruneExpired every TTL/4.
  - **vNext**: per-endpoint TTL override (some bulk ops
    are safe to replay for longer; others should expire
    sooner).
