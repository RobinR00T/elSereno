# Snapshot — v2.36.0: MMS vendor hint + LD enumeration

Single-chunk cycle. Closes deferred roadmap item #2 (grande).
The v1.51 mms plugin established ACSE association + confirmed
the IEC 61850-8-1 OID; v2.36 extends to the actual MMS service
layer on the associated session.

## What changed

### `internal/protocols/mms/wire/mms_services.go` (NEW)

  - `ExtractMMSVendorHint(buf)` — scans AARE bytes for any of
    19 curated vendor markers (SIEMENS SIPROTEC, SCHWEITZER
    ENGINEERING/SEL-, ABB RELION, GE MULTILIN, SCHNEIDER
    ELECTRIC/MICOM, AREVA, HITACHI ENERGY, NR ELECTRIC, RTDS,
    OMICRON, BECKHOFF, KALKITECH, LIBIEC61850, etc.). Returns
    a sanitised 120-byte context window around the match;
    empty string when no marker present.
  - `BuildMMSGetServerDirectoryRequest()` — constructs a
    confirmed-service ConfirmedRequest PDU (tag 0xA0) wrapping
    a GetNameList request with objectClass=domain. ASN.1/BER
    encoded by hand (~17 bytes).
  - `ParseMMSGetServerDirectoryResponse(buf)` — best-effort
    BER scan that pulls VisibleString (0x1A) elements from
    the listOfIdentifier SEQUENCE. Drops names containing
    non-printable bytes.
  - `FormatLDList(names)` — renders the LD slice for the
    finding payload; truncates at 8 names with `+N more`.

### `internal/protocols/mms/mms.go` (extended)

  - `tryACSEAssociate` signature: now returns
    `(note, aareBytes, ok)`. Caller gets the raw AARE so it
    can extract vendor hints.
  - After ACSE associates, the plugin chains:
    1. `ExtractMMSVendorHint(aareBytes)`.
    2. `tryGetServerDirectory(conn, ioTimeout)` issuing the
       confirmed-service request.
  - Findings now carry a richer note:
    ```
    MMS ACSE associated (IEC 61850-8-1) · vendor-hint:
    ...SIEMENS SIPROTEC 5... · LDs: [CTRL, MEAS, PROT]
    ```
  - Degrades gracefully: vendor-hint or LD enumeration
    failure leaves the v1.51 ACSE-level note intact.

## Honest scope

  - GetServerDirectory uses defaults: objectScope=vmd-specific,
    no maxResultsPerRequest, no continueAfter. Real IEDs with
    >100 LDs may paginate; we don't follow continuation
    tokens yet (deferred).
  - Vendor marker list is curated, not exhaustive. We add
    new vendors as we encounter them. False-positive risk
    from incidental byte sequences is very low (markers are
    longer + more structured than common BER tag-length
    pairs).
  - Defensive only. No SetDataValues, no control blocks, no
    file-services. Read-only service (`getNameList`) by spec.
  - INSTALL.md unchanged. Build sizes unchanged.

## Build sizes
