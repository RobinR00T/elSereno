# Snapshot — v2.3.0: `elsereno schedule` CLI verbs

Single-chunk cycle. Operators scripting against the
dashboard had to write curl loops + parse JSON. v2.3 adds
a native CLI tree.

## Verbs

```
elsereno schedule list
elsereno schedule get <id>
elsereno schedule delete <id> --yes
elsereno schedule stats <id> [--days N]
elsereno schedule export [--format ndjson|csv|json]
```

Persistent flags on `schedule`:
  - `--url` (default `$ELSERENO_URL` or `http://127.0.0.1:8787`).
  - `--token` (default `$ELSERENO_TOKEN` or
    `~/.elsereno/token` if mode-0600).

Token file has hardened mode-check: refuses to use a
world/group-readable file (matches SECURITY.md). Prints
a stderr warning + falls back to no-auth.

## Design

- CLI talks to the local serve over HTTP rather than
  embedding the store directly. Avoids two-writer footguns
  + inherits the dashboard's auth middleware.
- Single `httpDo` helper handles req construction, Bearer
  injection, 30s timeout, body close, non-2xx → error.
- Per-verb output respects the existing root `--format`
  flag (json|ndjson|table). Default is human-readable
  table for `list`/`stats`, pretty-printed JSON for `get`,
  raw bytes for `export`.

## Smoke test

```
$ elsereno schedule --help
Available Commands:
  delete      Delete a schedule (with confirmation)
  export      Export schedules (csv|ndjson|json) for DR backup
  get         Get one schedule by ID
  list        List all schedules
  stats       Show aggregate run-stats for a schedule
```

## alignedWriter

Tiny tab-aligned writer for table output (avoids the
`text/tabwriter` import since we only need basic column
padding). Buffers all rows then pads each column to the
widest cell on Flush.

## Honest scope

- No `create` / `update` / `clone` / `enable` / `disable`
  / `import` / `pause-all` verbs yet. Operators with those
  use cases still write curl or use the dashboard. The
  read-only subset covers the dominant CLI use case
  (sanity + reporting + backup); mutating verbs land in
  a future cycle.
- `--token` from file refuses lax modes (>0o600) — a
  hardening choice that may surprise operators with looser
  defaults. The fallback is no-auth, which the dashboard
  accepts in dev mode.
- No tests yet — the verbs are thin wrappers around
  httpDo + JSON marshalling. A future cycle could add
  httptest-server-based tests.
- INSTALL.md unchanged.
