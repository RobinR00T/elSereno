# Snapshot — v1.4.0: offensive PBX write-gates + BACnet UDP relay + TR-069 fingerprint

Closed **2026-04-23**. Builds on v1.3.0's PBX-discovery cycle
with offensive write-gated proxies for all three PBX plugins
(SIP, IAX2, pbxhttp), plus a BACnet UDP relay that closes the
v1.2 carry-over, plus a brand-new TR-069/CWMP ACS fingerprint.

Default-build plugin count: **16 → 17** (+cwmp).
Offensive write-gated proxies: **+4** (sip, iax2, pbxhttp, bacnet
— bacnet was session-primitive-only before this cycle).

## Chunks

### Chunk 1 — SIP offensive write-gate (`9038e4b`)
- `offensive/write/sip/gatedproxy.go`: per-request method
  allowlist via net/textproto request-line parser. Always-safe
  set: OPTIONS / ACK / BYE / CANCEL / PRACK. Gated: INVITE,
  REGISTER, MESSAGE, SUBSCRIBE, NOTIFY, REFER, PUBLISH, UPDATE,
  INFO.
- Refusal: canonical `SIP/2.0 405 Method Not Allowed` with an
  `Allow:` header listing always-safe + allowlisted methods.
- 13 tests: AllowlistHash order/case insensitivity,
  Authorise happy + denied + unauthorised Handle, every always-
  safe method passes, allowed + blocked INVITE, blocked
  REGISTER with different-method allowlist, case-folded method
  canonicalisation, multiple requests in one stream, Allow
  header composition correctness (via allowLineFromResponse
  helper to avoid matching the echoed CSeq header — subtle
  pitfall caught by a failing test).
- race-clean; gocyclo split Handle + handleOne for
  maintainability.

### Chunk 2 — pbxhttp offensive write-gate (`482263f`)
- `offensive/write/pbxhttp/gatedproxy.go`: (method, path) pair
  allowlist using net/http's server-side parser (ReadRequest +
  req.Write). Read-only methods (GET / HEAD / OPTIONS) always
  pass. CONNECT is always refused (the gate can't inspect
  tunnelled traffic).
- Two-stage refusal: 405 Method Not Allowed if no allowlist
  entry uses the method; 403 Forbidden if the method matches
  but the path doesn't.
- 14 tests: AllowlistHash order/method-case/path-case
  sensitivity; Authorise happy + denied + unauthorised; GET /
  HEAD / OPTIONS always pass; POST allowed on matching path;
  POST blocked on non-matching path (403); POST blocked when
  method not allowlisted (405 + correct Allow header); CONNECT
  always refused; multiple requests in one stream (first GET
  forwards, second DELETE refused on same keep-alive); malformed
  request handling.
- bodyclose linter caught a helper pattern issue; fixed by
  returning a `statusResp{Code, Header}` snapshot instead of
  the live *http.Response.

### Chunk 3 — iax2 offensive write-gate (`b0d3ea7`)
- `offensive/write/iax2/gatedproxy.go`: per-datagram IAX2
  subclass allowlist. UDP, so each `client.Read` returns one
  complete frame (net.Pipe preserves this in tests). Mini-
  frames (audio) + non-IAX full frames (Voice / DTMF / Video)
  ALWAYS pass — media unconditional. Always-safe IAX
  subclasses: HANGUP, ACK, PING, PONG, LAGRQ, LAGRP, INVAL,
  REGAUTH, REGACK, REGREJ, REGREL, REJECT. Gated: NEW, REGREQ,
  AUTHREP, ACCEPT.
- Refusal: HANGUP full-frame addressed to the client's
  SrcCallNum. IAX2 has no "permission denied" message; HANGUP
  is the universal call-teardown signal and is interpreted
  correctly by real IAX2 stacks.
- 14 tests: mini-frame always passes, non-IAX Voice frame
  always passes, HANGUP + ACK always pass, NEW allowed with
  allowlist, NEW blocked → HANGUP back with correct Dst,
  REGREQ allowed / blocked, mixed stream with mini + blocked
  NEW.

### Chunk 4 — CLI dry-run for all three gated proxies (`26ca8df`)
- `cmd/elsereno/cmd_write_gates_offensive.go`: three new
  subcommands under the existing offensive `write` tree.
  `elsereno write sip dry-run --target h:p --method INVITE`
  builds the SessionMutation, prints the canonicalised
  allowlist + PayloadHash. When `--vault-passphrase-file` is
  supplied, also mints + prints the expected confirm-token via
  `confirm.ExpectedToken`.
- Three command shapes match the three gates:
    `write sip dry-run --method METHOD [--method ...]`
    `write iax2 dry-run --subclass {NEW|REGREQ|AUTHREP|ACCEPT}`
    `write pbxhttp dry-run --allow METHOD:/path [--allow ...]`
- 11 tests: parseAllowEntry valid/invalid (colon splitting,
  path-must-start-with-/); iaxSubclassByName case-folded +
  rejects always-safe subclasses; canonMethods sort+dedupe+
  upper; shape tests for all three dry-runs using cobra's
  SetArgs + SetOut; --target required.

### Chunk 5 — TR-069 / CWMP ACS fingerprint (`02e705f`)
- `internal/protocols/cwmp/`: 17th plugin in the default build.
  HTTP(S) probe on 7547/tcp. Identifies 15 ACS implementations:
  GenieACS, FreeACS, Axiros (AXACS / AX-MDM), Nokia Altiplano,
  Huawei FusionHome / FusionInsight, Broadcom BroadWorks, Cisco
  Prime, ADB, Friendly TR-069 Simulator, interaCMS, Netopia,
  create-net, generic open-ACS, generic TR-069 marker.
- VendorRisk tiers: 90 open-source (GenieACS / FreeACS) — lab /
  default-install heavy; 85 enterprise carrier (Axiros / Nokia
  / Huawei / Broadcom / Cisco); 75 lab / test; 80 default
  (exposed ACS is a finding regardless of brand).
- isCWMPLikely heuristic: even without vendor match, a 401 with
  realm containing "acs" / "cwmp" / "tr-069" — OR a SOAP body
  mentioning "soap" + "cwmp" — counts as CWMP confirmation.
- DenyAll proxy emits HTTP 403; offensive write-gated variant
  (allowlist-SOAP-RPCs) is slated for v1.5.
- 15 tests across vendor matcher table, risk tiers, probe paths
  (GenieACS / Nokia / 401-cwmp-realm / SOAP-fault / non-CWMP
  nginx), metadata, REPL stub, DenyAll-proxy 403.

### Chunk 6 — BACnet UDP write-gate (`e4dc2a6`)
Closes the v1.2 carry-over. BACnet/IP is UDP-only; the TCP
proxy framework never applied, so Handle-loop integration was
deferred. This chunk adds:
- `internal/protocols/bacnet/wire/service.go`: ASHRAE 135 APDU
  classification helpers. APDUType enum (ConfirmedRequest,
  UnconfirmedRequest, SimpleAck, ComplexAck, SegmentAck, Error,
  Reject, Abort). ConfirmedService enum with 20+ values from
  Table 20-7. IsMutatingConfirmedService declares the gated
  service set. ParseAPDUHeader + BuildAbortPDU.
- `offensive/write/bacnet/gatedproxy.go` (rewritten from the
  session-primitive stub): full UDP relay. Always-pass:
  non-BACnet bytes, unconfirmed-requests (Who-Is / I-Am / Who-
  Has / I-Have / TimeSync / presence), server-side acks /
  errors / rejects / aborts, confirmed-reads. Gated: confirmed-
  writes — WriteProperty, WritePropertyMultiple,
  AtomicWriteFile, AddListElement, RemoveListElement,
  CreateObject, DeleteObject, ReinitializeDevice (coldstart /
  warmstart), DeviceCommunicationControl (silence a device),
  LifeSafetyOperation (silence / unsilence alarms).
- Refusal: BVLC-wrapped Abort-PDU with ASHRAE 135 §20.1.9
  reason 5 (security-error) addressed to the client.
- extractAPDU walks past BVLC + NPDU with optional routing.
- 14 tests: AllowlistHash ordering; Authorise happy + denied +
  unauthorised; Who-Is always passes; ReadProperty always
  passes; non-BACnet bytes pass through; WriteProperty allowed
  with allowlist (service choice forwarded); WriteProperty
  blocked → Abort with correct invoke-id + security reason;
  ReinitializeDevice blocked when only WriteProperty allowed
  (per-service discrimination); IsMutatingConfirmedService
  table.

## Design patterns established / reinforced across v1.4

The ADR-040 write-gate template now has **five concrete
implementations** (modbus, opcua, sip, iax2, pbxhttp, bacnet):

- `AllowedX{Field Y}` — the scope of what the session
  authorises. Case-folded on method strings, numerically
  sorted on byte codes. SHA-256 of the NUL-separated sorted
  list + target is the session PayloadHash.
- `SessionMutation(target, allowed)` — the canonical
  confirm.Mutation.
- `WriteGatedHandler` — holds Target, Allowed, Deriver,
  Auditor, SessionConfirm. `Authorise()` runs
  confirm.Authorize once per session; subsequent calls are
  no-ops. `Handle()` is the ProxyHandler implementation, two
  goroutines: client→upstream parses + gates per
  request/datagram; upstream→client is io.Copy.
- Always-pass set: anything safe (discovery, reads, acks,
  error responses, teardown).
- Refusal: protocol-native ("405 Method Not Allowed" for SIP
  + HTTP; HANGUP full-frame for IAX2; Abort-PDU for BACnet;
  ServiceFault for OPC UA; ExceptionCode-reply for Modbus).
- Carefully documented `//nolint:gosec` waivers for
  InsecureSkipVerify in the fingerprint plugins (v1.3 and
  cwmp).

## Retrospective

### What went well
- **Template velocity**: chunks 1-3 took roughly equal time
  (~45 min each) because the SIP template (chunk 1)
  established the shape cleanly. IAX2's UDP variant was the
  only one that needed architectural thinking (per-datagram
  reads preserve-boundaries via net.Pipe).
- **Test doubles are reusable**: fakeDeriver + fakeAuditor +
  mintToken + newHandler — same 60-line preamble every gate
  test file starts with. Factored once in opcua, copied three
  more times without much friction.
- **Linter-first refactors**: every package had gofmt,
  golangci-lint, race-detector, and cross-build-tag checks
  before landing. Made chunk-to-chunk transitions smooth — no
  accumulated tech debt.
- **BACnet carry-over**: the v1.2 close explicitly flagged
  BACnet UDP relay as v1.3+ work. Landing it in v1.4 closes
  the ledger cleanly.

### Pitfalls hit + fixed
- **`strings.Contains(resp, "MESSAGE")` matching echoed CSeq
  header**: SIP responses echo the request's CSeq which
  contains the method name. Checking the full response for
  "MESSAGE" when validating the Allow header falsely matched
  on the CSeq. Fixed via an `allowLineFromResponse` helper
  that narrows to the Allow: line only.
- **bodyclose linter + helpers**: static analysis can't see
  through helper functions. `readHTTPResponse` returning a
  live *http.Response tripped the linter even when the helper
  closed the body internally. Fixed by returning a snapshot
  struct (`statusResp{Code, Header}`) so callers never touch
  the body.
- **Race detector + recorder goroutines**: upstreamRecorder
  accessed .data from two goroutines (writer + assertions)
  without a mutex. Race detector caught it. Fixed with a
  sync.Mutex + `snapshot()` helper.
- **RFC-canonical "Unauthorized"**: the misspell linter keeps
  flagging it. We use British English in prose and allow the
  US spelling only where required by HTTP / SIP RFCs, with
  `//nolint:misspell` rationale comments.
- **gosec G115 integer narrowing**: the BACnet BVLC length
  byte is a small int→byte cast. gosec flagged the narrowing.
  Justified with //nolint:gosec + rationale (bounded by
  construction).
- **IAX2 per-datagram UDP semantics on net.Pipe**: first
  driveSession helper tried to split client IO across two
  net.Pipes and got tangled. Simplified to one pair where the
  test holds both write + read on the same connection.

### Carry-overs to v1.5
- **Offensive `proxy listen` CLI verb**: today the gates have
  dry-run; running them inline requires promoting the existing
  `proxy` stub command. The listener setup + signal handling +
  MaxConns bookkeeping isn't hard, it's just deferred.
- **OPC UA per-NodeId allowlist**: ADR-040 authorises at
  service-TypeID granularity today; the UA binary-encoding
  parser that would pick apart WriteValue / CallMethodRequest
  bodies to enforce per-NodeId is v1.5.
- **BACnet per-object allowlist**: same shape — gate is
  service-choice only today; per-Object_Identifier needs an
  ASN.1 BER parser.
- **SIP To-URI allowlist for INVITE**: toll-destination
  blocking by E.164 prefix. Field hours on the method-level
  gate first.
- **REGISTER AOR allowlist**: same principle.
- **CWMP offensive proxy**: allowlist specific SOAP RPCs —
  GetParameterValues always; explicit SetParameterValues per
  allowlist.
- **HTTP paths beyond `/` for pbxhttp fingerprint**: vendor-
  specific probe paths like `/admin/config.php`, `/webclient/`,
  `/ccmadmin/` for recall improvement.
- **VoIP-SIP dial backend subprocess**: same as v1.3
  carry-over; needs seccomp-pair split.
- **`dial batch --backend` CLI wiring**: call Backend.Deliver
  on allowed numbers.
- **Audit daemon for cross-process JSONL**: same as v1.3
  carry-over.
- **seccomp arg-level filtering**: same as v1.3 carry-over.
