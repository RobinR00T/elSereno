# Snapshot — v1.31.0: TUI input parity with batch scan

Single-chunk cycle. Closes the v1.30-chunk-3 carryover that
shipped TUI's `--input list:FILE` only and explicitly deferred
the 7 other kinds (nmap, stdin, shodan, censys, fofa, zoomeye,
onyphe, internetdb).

## Why a tiny cycle

A tight cycle on a single, high-value plumbing change beats
batching it with unrelated work. The change touches three
files (cmd_scan, cmd_tui, new cmd_input_parse) and is
mechanical refactoring + symmetric wire-up. Easier to review,
easier to roll back if a regression slips, and gives operators
the missing surface immediately rather than waiting for the
next "real" cycle.

## Chunks

### Chunk 1 — Shared input dispatcher + TUI wire-up (`689def2`)

The batch `scan` verb's `readTargets` had grown into a 50-line
switch with one case per input kind. Copying that into cmd_tui
would have duplicated a maintenance dependency: every future
input kind would need to land in two places. Extracting the
dispatcher fixes this once.

Adds:

  cmd/elsereno/cmd_input_parse.go (97 lines)
    parseInput(ctx, opts inputParseOpts) → ([]core.Target, error)
    + parseStdinInput / parseListInput / parseNmapInput sub-helpers
    + providerPrefixes []string slice (shodan, censys, fofa,
      zoomeye, onyphe, internetdb) walked in a single loop so
      adding the next provider is a 1-line change.
    Stdin field on inputParseOpts is io.Reader-injectable
    (defaults to os.Stdin when nil); needed for unit-testing
    the stdin path without touching the process's real stdin.

Refactors:

  cmd/elsereno/cmd_scan.go
    readTargets becomes a 5-line shim that builds
    inputParseOpts from scanOpts + delegates to parseInput.
    Drops 3 imports (list, nmapxml, stdin) — they live in
    cmd_input_parse.go now.

  cmd/elsereno/cmd_tui.go
    buildInteractiveFeed delegates to parseInput with the
    inputKind passed straight through. Adds the
    --api-creds-file flag mirroring cmd_scan. Long help text
    rewritten to enumerate all 8 kinds.
    pickFeedArgs gains apiCredsFile field.
    newTUICmd refactored to use registerTUIFlags() helper
    (extracted to keep newTUICmd under the funlen 60-line
    ceiling as the flag count grew).

Operator-visible change:

  # v1.30 — only this worked:
  elsereno tui --input list:targets.txt

  # v1.31 — all 8 kinds work directly:
  elsereno tui --input nmap:scan.xml
  elsereno tui --input shodan:cisco --api-creds-file creds.yaml
  elsereno tui --input internetdb:1.2.3.4
  elsereno tui --input stdin                 # reads from os.Stdin
  # ... + censys: / fofa: / zoomeye: / onyphe:

Stdin caveat: `--input stdin` reads from `os.Stdin`, but the
TUI also takes its keyboard input from `os.Stdin` once the
alt screen mounts. Mixing the two on a real TTY won't behave
sensibly; operators should pipe via `--feed -` (which also
reads stdin but consumes NDJSON live), or use
`list:/dev/stdin` if they're really determined. The kind is
kept in the dispatcher for symmetry with `scan` rather than
removed.
