# Snapshot — v1.3.0: PBX discovery across SIP + IAX2 + HTTP admin UI

Closed **2026-04-22**. Three new protocol plugins land in the
default build, collectively identifying **15 PBX brands** across
the three canonical PBX attack surfaces (signalling UDP/TCP,
Asterisk binary UDP, management web). Plugin count rises from
13 (at v1.2.0 tag) to 16.

## Chunks

### Chunk 1a — SIP OPTIONS probe + 15-vendor matcher (`e8278e5`)
- `internal/protocols/sip/wire/` — minimal RFC 3261 encoder +
  parser. `BuildOPTIONS(host, branch)` crafts a single OPTIONS
  request with `Via`, `Max-Forwards: 70`, `From`, `To`,
  `Call-ID`, `CSeq: 1 OPTIONS`, `Content-Length: 0`.
  `ParseResponse(io.Reader)` uses `net/textproto` to read the
  status line + headers, extracts Server + User-Agent + Allow
  verbs for fingerprinting. `IsSIPStatus` guards against
  non-SIP responders.
- `internal/protocols/sip/vendor.go` — 15 known PBX brands with
  priority-ordered substring matchers: Asterisk, FreePBX, 3CX,
  Cisco UCM, Cisco SIP Gateway, Mitel (incl. ShoreTel), Avaya
  (incl. IP Office), Yeastar, Grandstream, Fanvil, Yealink,
  Kamailio, OpenSIPS, FreeSWITCH, SER. `VendorRisk` tiers:
  90 attack-ripe (Asterisk / FreePBX / 3CX); 85 enterprise
  (Cisco UCM / Avaya / Mitel); 80 SOHO (Yeastar / Grandstream /
  Fanvil / Yealink); 75 proxy/gateway (Kamailio / OpenSIPS /
  FreeSWITCH / SER / Cisco SIP GW); 70 unknown-vendor SIP.
- `internal/protocols/sip/sip.go` — plugin with UDP (default) +
  TCP transport paths. DenyAll proxy replies `SIP/2.0 403
  Forbidden` with `Server: ElSereno proxy (read-only)`.
  Offensive write-gated variant deferred to v1.4.
- 7 tests covering every vendor path, non-SIP responders, 401
  challenge auth_state drop, metadata, and a 15-brand
  `IdentifyVendor` table.

### Chunk 1b — IAX2 NEW/HANGUP probe (`ca68a3a`)
- `internal/protocols/iax2/wire/` — RFC 5456 full-frame parser.
  12-byte header with SrcCallNum / DstCallNum (15-bit) /
  Timestamp / OSeqno / ISeqno / FrameType / Subclass.
  `FrameTypeIAX = 0x06` with subclass enum covering NEW, PING,
  PONG, ACK, HANGUP, REJECT, ACCEPT, AUTHREQ, AUTHREP, INVAL,
  LAGRQ, LAGRP, REGREQ, REGAUTH, REGACK, REGREJ, REGREL.
  `BuildNEW(srcCall)` + `BuildHANGUP(srcCall, dstCall, oseqno,
  iseqno)`. `ErrMiniFrame` returned when byte[0] high bit is 0
  (mini-frame = audio payload, not control).
- `internal/protocols/iax2/iax2.go` — UDP/4569 probe. Sends a
  random-call-number NEW, reads the reply. Any FrameTypeIAX
  reply confirms IAX2 → `protocol_risk=90`. Subclass note:
  `iax2-accept`, `iax2-authreq`, `iax2-reject`, `iax2-hangup`,
  `iax2-ping`, `iax2-pong`, `iax2-regauth`, `iax2-regreject`.
  ACCEPT triggers a polite HANGUP so the remote's dialogue
  table doesn't grow.
- **Bug + fix**: initial draft treated any `ErrMiniFrame` as
  IAX2-confirmed. HTTP-shaped junk starting with 'H' (0x48 has
  high bit 0) would look like a mini-frame and falsely confirm
  IAX2. Fixed by requiring `n == 4` (real IAX2 mini-frames are
  exactly 4 bytes). Caught by `TestProbe_NonIAX2Bytes`.
- 5 tests covering AUTHREQ (auth_state drops to 50), ACCEPT
  (stays at 60), non-IAX2 bytes (protocol_risk=70 default),
  REJECT (still confirms IAX2), metadata.

### Chunk 1c — pbxhttp HTTP(S) admin-page fingerprint (`abd296d`)
- `internal/protocols/pbxhttp/vendor.go` — 15 PBX platforms
  fingerprinted via HTTP Server / title / body: FreePBX, PBXact
  (Sangoma), 3CX, Yeastar (+ NeoGate + Linkus + MyPBX), Cisco
  UCM, Avaya (IP Office + Aura + Communication Manager), Mitel
  (+ ShoreTel + MiCollab), Grandstream (+ UCM6 + GXP + GXW),
  Fanvil, Yealink (+ SIP-T), Asterisk HTTP Manager, Switchvox
  (Digium), Elastix, FreeSWITCH. Priority-ordered substrings;
  case-insensitive internally. `VendorRisk` tiers parallel the
  SIP plugin's.
- `internal/protocols/pbxhttp/pbxhttp.go` — `Default()` wires
  HTTPS on 443 with `InsecureSkipVerify=true` (PBX default
  installs universally ship self-signed certs; we're
  fingerprinting, not transmitting credentials, so the
  operational value outweighs the MITM risk in this narrow
  context — rationale documented in the `//nolint:gosec` comment).
  Single GET to `/` with a browser-like User-Agent + HTML Accept
  header. `flattenHeaders` concats Server / X-Powered-By /
  Www-Authenticate / Set-Cookie. `titleRE` extracts the HTML
  `<title>`. **PBX-likely heuristic**: when no vendor matches
  but the body contains "pbx", "phone system", "sip server",
  "voip admin", or "extension", protocol_risk bumps to 70 so
  the finding still surfaces. DenyAll proxy replies `HTTP/1.1
  403 Forbidden` with `Connection: close`.
- 21 tests across httptest.NewServer (5 vendor-specific probes
  + non-PBX nginx + PBX-likely heuristic + 401 auth_state drop),
  httptest.NewTLSServer (self-signed-cert path), metadata, REPL
  stub, DenyAll-proxy 403 reply, 15-vendor `IdentifyVendor`
  table, 15-tier `VendorRisk` table.

## Design patterns established / reusable for v1.4+

- **PBX brand matcher** (priority-ordered `{needle, vendor}`
  list) is the canonical template for future vendor tables.
  `sip/vendor.go` + `pbxhttp/vendor.go` share the same shape;
  new brand additions are a single line.
- **Scoring tiers**: 90 attack-ripe / 85 enterprise / 80 SOHO /
  75 gateway / 70 unknown. `VendorRisk` in every plugin that
  does vendor classification.
- **Protocol-native refusals** on the default proxy:
    SIP → `SIP/2.0 403 Forbidden` + Server header.
    IAX2 → UDP, so no reply (the Handle returns an error).
    HTTP → `HTTP/1.1 403 Forbidden` + Server + Connection: close.
- **Mini-frame / short-frame sanity**: for any binary UDP
  protocol where a single byte distinguishes the frame class,
  ALSO validate `len(frame) == expectedSize`. Protects against
  HTTP-like junk tripping classifiers.
- **Self-signed cert tolerance for fingerprint probes**: when
  the operational goal is identify-the-platform rather than
  transmit-credentials, `InsecureSkipVerify=true` is correct.
  Document the rationale in a `//nolint:gosec` comment.

## Retrospective

### What went well
- **Three chunks in one cycle** — the `vendor.go` matcher
  pattern established in chunk 1a meant 1b and 1c each took
  under an hour once the wire parsers were stable.
- **Test-driven vendor lists**: writing `TestIdentifyVendor_Table`
  as 15-case table tests forced the matcher to be both complete
  and deterministic (priority-ordered, no ambiguous needles).
- **Mini-frame sanity caught by a non-IAX2 test** — would have
  shipped as a false-positive finding generator without it.

### Pitfalls hit
- **IAX2 HTTP false-positive**: described above. Fix is the
  `n == 4` guard. Added to `pitfalls.md` as a new anti-pattern:
  *when distinguishing frame classes by a single bit, ALSO
  validate byte length expectations.*
- **SIP "Unauthorized" linter flag**: the RFC 3261 §21.4
  spelling is canonical; suppressed with `//nolint:misspell` +
  rationale comment. The same applies to `net/http`'s
  `StatusUnauthorized` constant in pbxhttp.
- **`IdentifyVendor` API ambiguity**: first draft required the
  caller to lowercase the three arguments; table tests that
  passed mixed case silently failed. Fix: lowercase internally
  and document "case-insensitive" in the docstring. Consistent
  with `sip.IdentifyVendor`.

### Carry-overs to v1.4 (or later v1.3 chunks)
- **Offensive write-gated proxy variants** for SIP, IAX2,
  and pbxhttp — the default deny handlers are operator-ready;
  the write-gated variants (INVITE allowlist, HANGUP allowlist,
  admin-API method/path allowlist) arrive with the standard
  ADR-040 ±`--accept-writes` pattern.
- **TR-069/CWMP fingerprint on 7547/tcp** — original v1.3
  chunk 2 plan; deferred so v1.3.0 ships a coherent
  PBX-discovery cycle.
- **VoIP-SIP dial-backend subprocess** (`elsereno-dial-voip-sip`)
  — the Backend interface + the dial gate already ship from
  v1.2; the subprocess binary + its seccomp-pair stays v1.4+.
- **HTTP paths beyond `/`**: the current probe is `GET /` only.
  Vendor-specific hardening (e.g. `/admin/config.php` on
  FreePBX, `/webclient/` on 3CX, `/ccmadmin/` on Cisco UCM)
  would improve recall on PBXes that serve a neutral landing
  page on the root path. v1.4 candidate.
