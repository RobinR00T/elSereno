# Snapshot — v1.57.0: DLMS/COSEM offensive write-gated proxy

Single-chunk cycle. Closes the v1.32+ D3 carryover.
Adds the offensive TCP-aware write-gated proxy for
DLMS/COSEM over TCP/4059. DLMS/COSEM (IEC 62056) is
the dominant European AMI/AMR smart-meter application
protocol; utilities speak it from back-offices to
electricity/gas/water/heat meters.

This is the final D-series carryover (D1=KNX in v1.55,
D2=M-Bus in v1.56, D3=DLMS now). The legacy-ICS trio's
offensive write paths are now complete.

## Why

SET-Request and ACTION-Request APDUs can: re-tariff
(write tariff registers), cut/restore service via the
Disconnect Control object (OBIS 0-0:96.50.0*255),
reconfigure security setup (OBIS 0-0:43.0.0*255),
trigger firmware updates via the Image Transfer
object, change clock + DST rules, modify scheduled
disconnect events. Per-COSEM-object gating lets the
operator authorise the safe ones without exposing the
nuclear ones.

## Three-tier gate

```
APDU tag → COSEM target (class+OBIS+member) → match strictness
   |              |                                |
   |              |                                └── MatchExact /
   |              |                                    MatchClassOBIS /
   |              |                                    MatchClassOnly
   |              |
   |              └── refuse if no AllowedCosem entry matches.
   |                  OBIS bytes = 255 in entry act as wildcards
   |                  (canonical OBIS convention).
   |
   └── always-safe: AARQ/AARE/RLRQ/RLRE/GET/responses/EXCEPTION.
       SET/ACTION require allowance + COSEM target match.
```

Refusal mode: **silent drop**. The synthesisable
EXCEPTION-Response APDU would deceive the operator
about wire-level reality (real meters never emit
EXCEPTION as a permission-denied signal). Silent drop
with the connection open lets the client time out
cleanly.

## OBIS wildcard semantics — operator's lever

The OBIS canonical wildcard byte is 255 (0xFF). The
gate treats it consistently with the OBIS convention:

  - Allowlist entry `OBIS: {0,0,96,255,255,255}` —
    matches any 0-0:96.*.*.* OBIS code.
  - Allowlist entry `OBIS: {1,0,94,7,0,255}` — matches
    1-0:94.7.0*255 (the canonical "F=255" used
    everywhere) BUT NOT 1-0:94.7.0*0 (different F
    byte).
  - Allowlist entry MatchType=MatchClassOBIS makes
    the MemberID wildcarded but OBIS is still
    byte-by-byte matched (with 255 wildcards).

Operators picking match strictness:

  - **MatchExact** — tightest. Use for "the
    Disconnect Control object's reconnect method
    only" (`{ClassID:70, OBIS:disconnect, MemberID:1,
    MatchType:MatchExact}`).
  - **MatchClassOBIS** — wildcards member. Use for
    "any attribute on the tariff register"
    (`{ClassID:3, OBIS:tariff, MatchType:MatchClassOBIS}`).
  - **MatchClassOnly** — loosest. Use for "any
    Register class write" (`{ClassID:3,
    MatchType:MatchClassOnly}`).

The three modes compose: an allowlist with mixed
strictnesses works fine; the gate evaluates each
entry independently.

## Honest scope

  - **Datablock + with-list SET variants** NOT
    parsed. Only the normal CHOICE (0x01) is
    supported; other CHOICEs refuse. Operators using
    multi-target SETs see them refused; workaround
    is to issue normal-CHOICE SETs.
  - **GLO_/DED_ ciphered APDUs** (0xC8..0xCF range)
    refused at the APDU-tag level. Inline gating of
    ciphered SET requires the operator's master
    key. Workaround: run the proxy ahead of the
    cipher layer (e.g., on the management
    workstation).
  - **CLI flag plumbing** NOT in this chunk. Wire +
    gate primitives are the foundation; flags in
    cmd_write_offensive.go are a future cycle's add
    (same pattern as v1.52/v1.53/v1.55/v1.56).
  - **Lab validation against real DLMS smart meter**
    pending. Confidence ~0.85 until validated
    against an Itron / Landis+Gyr / Iskraemeco IED
    in a lab harness.
