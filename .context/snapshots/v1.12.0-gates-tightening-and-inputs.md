# Snapshot — v1.12.0: gates tightening + input pagination

Closed **2026-04-25**. Ten-chunk cycle that closes every per-
object / per-path / pagination carry-over accumulated from
v1.6→v1.11. Theme: each existing gate gets a finer dimension;
each existing input provider gets pagination; one new no-key
provider; one new firmware-URL gate.

## Why this cycle matters

The v1.4 → v1.11 cycles built **breadth** — 7 offensive write-
gated proxies (modbus / opcua / sip / iax2 / pbxhttp / bacnet /
cwmp) + 5 attack-surface input providers (shodan / censys /
fofa / zoomeye / onyphe). v1.12 builds **depth**: every gate
now scopes to a specific identity, every input client paginates,
and one previously-coarse RPC (CWMP `Download`) gets per-image
allowlisting.

The hash ladder convention (each new dimension degrades to the
previous-version hash when the new field is empty) preserves
operator tokens across the upgrade; nobody who skips the new
fields needs to re-mint a confirm-token.

## Chunks

### Chunk 1 — CWMP per-parameter-path allowlist (`1a6cec3`)

`AllowedParameterPath{Prefix}` + `AllowlistHashWithParameterPaths`
(0xFE separator) + `extractParameterNames` (streaming
`xml.Decoder` walking `<ParameterValueStruct>` /
`<SetParameterAttributesStruct>`). Refusal is SOAP Fault 9005
"Invalid parameter name" + `X-Elsereno-Gate-Reason`. CLI
`--param-prefix`, YAML `param_prefixes:`. 12 tests.

### Chunk 2 — OPC UA multi-node WriteRequest walk (`0c34382`)

Closes the v1.6 chunk-2 carry-over: gate now walks every
`WriteValue` instead of only the first. New
`wire.WriteRequestAllNodes` walker; fail-closed on any
unparseable WriteValue. Numeric encodings only at this point.
8 tests.

### Chunk 3 — OPC UA String / GUID / ByteString NodeIDs (`b4bd4dd`)

Rich wire parser `WriteRequestAllNodesRich` returns
`NodeIDValue{Namespace, Kind, Numeric, String, GUID, Bytes}`
with `.Canonical()` printing `ns=N;{i,s,g,b}=…`. Operator
allowlist via `AllowedCanonicalNodeID` (string), 0xFD hash
separator, length-prefixed entries. CLI `--node-id` extended
with `s=` / `g=HEX` (32 chars, dashes tolerated, normalised
uppercase) / `b=HEX`. YAML `node_ids:` extended with optional
`canonical:` field. 17 tests.

### Chunk 4 — Modbus structured `writes:` YAML (`c3cafd1`)

Closes the v1.9 chunk-2 carry-over: Modbus allow-file used to
persist only `functions:`, dropping unit + address-range on
round-trip. New `proxyModbusWrite{Unit, FC, Start, End}` YAML
struct + `--write unit=N;fc=M;start=A;end=B` CLI (repeatable).
Emit guard lifted: legacy `--unit`/`--address-*` now
materialise as a `writes:` entry per FC. Library hash
unchanged — the existing `AllowlistHash` already sorted by
(unit, FC, start, end). 7 tests.

### Chunk 5 — SIP From-domain allowlist (`0325999`)

The identity-spoof complement to v1.10's REGISTER AOR gate +
v1.9's INVITE prefix gate. `AllowedFromDomain{Domain}` with
0xFD hash separator. Applies to every gated method (INVITE /
REGISTER / MESSAGE / SUBSCRIBE / NOTIFY / REFER / PUBLISH /
UPDATE / INFO); always-safe methods (OPTIONS / ACK / BYE /
CANCEL / PRACK) bypass. Refusal is SIP/2.0 403 +
`X-Elsereno-Gate-Reason: From domain not in session allowlist
(identity-spoof guard)`. CLI `--from-domain`, YAML
`from_domains:`. 11 tests.

### Chunk 6 — OPC UA CallRequest per-(object,method) (`9b2c4f5`)

Twin to chunk 3 on the other mutating-service surface. New
`CallRequestAllMethods` wire walker (shared
`walkWriteRequestArrayPrefix` helper with the chunk 2/3
walkers). `AllowedCallMethod{ObjectID, MethodID}` with both
fields in canonical-string form. 0xFC hash separator. CLI
`--call-method object=…;method=…`, YAML `call_methods:`.
9 tests.

### Chunk 7 — BACnet per-object WriteProperty (`196e647`)

Closes the v1.4 chunk-6 service-only carry-over for the most
common BACnet write surface. New ASN.1 BER walker
`ParseWriteProperty` (context tag 0 ObjectId + tag 1
PropertyId). `AllowedObject{ObjectType, ObjectInstance,
PropertyID}` with 0xFF hash separator. Other mutating
services (CreateObject / DeleteObject / WritePropertyMultiple
/ DeviceCommControl / ReinitializeDevice / LifeSafetyOperation
/ AtomicWriteFile / Add/RemoveListElement) keep service-only
gating — their request shapes differ; v1.13+ extends. CLI
`--object type=N;instance=M;property=P`, YAML `objects:`.
13 tests.

### Chunk 8 — Input pagination across 5 providers (`8c61ff5`)

Closes the v1.10 "page 1 only" carry-over noted in 4 of the 5
client comments. New `SearchPaged(ctx, query, totalLimit)` on
each client. Per-provider:

- Shodan: `?page=N`, partial-page detection.
- Censys: cursor follow via `links.next`.
- FOFA: `?page=N`, per-page=100.
- ZoomEye: `?page=N`, ~20/page.
- ONYPHE: `?page=N`.

CLI `readShodanTargets` etc. switched to `SearchPaged` with
`providerTotalLimit = 1000` cap (free-tier sane default). 9
tests.

### Chunk 9 — Shodan InternetDB (no-key provider) (`afb4eb3`)

6th attack-surface provider; the first that needs no API key.
Lookup-by-IP rather than search-by-query: `--input
internetdb:8.8.8.8` issues GET `/<ip>` → one `core.Target`
per open port. 404 from upstream maps to `(nil, nil)` (clean
"no data" UX). Single-IP for v1.12; bulk lookup is v1.13+.
5 tests.

### Chunk 10 — CWMP firmware-URL allowlist for Download (`9761eba`)

Closes the firmware-push attack-surface gap left by v1.11. A
misconfigured ACS can use `Download` to ship firmware to the
whole CPE fleet. `AllowedFirmware{URL, SHA256}` exact-match
(case-insensitive scheme+host, default-port stripped, path
verbatim). 0xFD hash separator with length-prefixed URL +
SHA256. SHA256 is operator metadata — TR-069 doesn't carry
it in `Download` (CPE reports it later via TransferComplete);
the gate enforces URL only. Refusal is SOAP Fault 9001 +
`X-Elsereno-Gate-Reason: CWMP firmware URL not in session
allowlist`. CLI `--firmware url=…;sha256=…` (sha256 optional,
64 lowercase hex), YAML `firmware:` array. 9 tests.

## Hash ladder summary

Each chunk's hash factors degrade to the previous version's
hash when the new field is empty. Concrete ladders:

- **OPC UA** (chunks 2/3/6):
  `AllowlistHashWithCallMethods` (chunk 6) → empty calls →
  `AllowlistHashWithRichNodeIDs` (chunk 3) → empty canonical →
  `AllowlistHashWithNodeIDs` (v1.6) → empty numeric →
  `AllowlistHash` (v1.2). Separators: `0xFC` calls, `0xFD`
  canonical, `0xFF` numeric.
- **CWMP** (chunks 1/10):
  `AllowlistHashWithFirmware` (chunk 10) → empty firmware →
  `AllowlistHashWithParameterPaths` (chunk 1) → empty paths →
  `AllowlistHash` (v1.11). Separators: `0xFD` firmware, `0xFE`
  paths.
- **SIP** (chunk 5):
  `AllowlistHashWithFromDomains` → empty domains →
  `AllowlistHashWithAORs` (v1.10) → empty aors →
  `AllowlistHashWithPrefixes` (v1.9) → empty prefixes →
  `AllowlistHash` (v1.4). Separators: `0xFD` from-domains,
  `0xFE` aors, `0xFF` prefixes.
- **Modbus** (chunk 4): library hash unchanged — `writes:` YAML
  feeds into the existing `AllowlistHash([]AllowedWrite)`.
- **BACnet** (chunk 7):
  `AllowlistHashWithObjects` → empty → `AllowlistHash` (v1.4).
  Separator: `0xFF` objects.
