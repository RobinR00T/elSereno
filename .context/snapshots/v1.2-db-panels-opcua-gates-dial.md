# Snapshot — v1.2: DB panels + OPC UA write gate + protocol Handle loops + dial backends + SLSA-via-Attestations-API

Closed **2026-04-22**. Ships five feature chunks, one housekeeping
change (sqlite retirement), one extra (SyncFromFile helper), and
a substantial documentation pack (operator manual + cheatsheet +
.docx + AUTHORS + dev-db helper + vNext TODO).

## Chunks

### Chunk 1 — DB-backed audit Writer + findings/triage/runs panels (`8370b18`)
- `internal/audit.DBWriter` persists audit entries to Postgres
  with the same chain invariant as FileWriter. Reserves IDs via
  `SELECT nextval('audit_log_id_seq')` BEFORE the INSERT so the
  JCS hash is computed with the final ID once. `DBConn` is a
  narrow two-method interface (QueryRow + Exec) — tests run
  against a 30-line fake, no live Postgres required.
- `MultiWriter` + `FileMirror` + `DBMirror` — one primary owns
  the chain, N mirrors persist verbatim. Primary error halts
  fan-out; mirror error surfaces without reverting the primary.
- `internal/repo` read-side: `ListFindings` (severity/protocol/
  min_score/created_after filters, limit clamped [1,500]),
  `ListRuns` (correlated finding counts), `Triage` (per-severity
  bucket counts, critical → info order, `[]` not `nil` when
  empty).
- `handlers.APIV1Deps{Broadcaster, Querier}` replaces the old
  single-arg signature. Missing Querier → 503 on the DB-backed
  endpoints; the rest of the router still serves.
- Dashboard panels: triage chips, findings table (top 50), runs
  table (top 20). Fetch on page load + re-fetch on SSE events
  (`finding`/`run_start`/`run_end`) debounced 500 ms.
- `serve` opens the pool when `DATABASE_URL` is set and passes
  it through.

### Chunk 2 — OPC UA write gating (`caa5b41`)
- `internal/protocols/opcua/wire` extended with OPN/MSG/CLO
  header recognition, `ServiceTypeID` decoder (TwoByte + FourByte
  NodeId encodings in namespace 0), `IsMutatingService`
  (WriteRequest 673 + CallRequest 704).
- `offensive/write/opcua` with `AllowedService{TypeID}` +
  `AllowlistHash` (sorted + SHA-256) + `SessionMutation` +
  `WriteGatedHandler`. Authorise once on the session mutation;
  per-frame filter at wire-parse time. HEL/OPN/CLO pass; MSG
  with non-mutating TypeID passes; mutating MSG outside
  allowlist → UA ServiceFault with StatusCode
  BadUserAccessDenied (0x80100000). 9 tests with real
  net.Pipe-driven session.

### Chunk 3 — Full Handle loops for dnp3/iec104/hartip/atg/fox (`caa5b41`-next)
Five TCP protocols that previously had only session primitives
now have full wire-level forwarding loops with protocol-native
refusal:
- **DNP3**: link-layer PRIMARY FC gate + app-layer FC gate
  (from transport+AC+FC offset inside user data). Refusal =
  user-data response with IIN2 bit 2 FUNC_NOT_SUPP.
- **IEC-104**: APCI I/U/S split. U/S pass; I-frames check ASDU
  Type ID allowlist; refusal = I-format mirror with COT=47 +
  P/N bit (negative confirm).
- **HART-IP**: session-lifecycle messages pass; TokenPassPDU
  extracts HART command byte (long vs short frame via the HIGH
  bit 0x80 of the delimiter — a fix to the initial misread of
  the spec). Refusal = HART response with response-code 0x40
  "command not implemented".
- **ATG Veeder-Root**: line-oriented ASCII SOH+CR; 'I' info
  commands pass; others consult allowlist; refusal = NAK
  `<SOH>9999FF1B<CR><ETX>`.
- **Fox (Niagara)**: line-oriented; hello/get/list/a verbs
  always pass; refusal = `fox a 0 -1 fox denied\n`.
BACnet remains at session-primitives only — UDP needs a
dedicated relay (v1.3).

### Chunk 4 — Dial backend interface + mock + atmodem (`8495a4a`-range)
- `offensive/dial/backend` package with `Backend` interface
  (Name / Deliver / Close) + Disposition enum (preview /
  delivered / no-answer / busy / hangup / failed) + Result.
- `Mock`: longest-prefix scripted dispositions; records every
  call; safe for CI.
- `ATModem`: Hayes sequence ATZ → ATE0 → ATDT<num>; → classify
  terminal result → ATH0. Shared bufio.Reader across phases
  (prevents the classic "fresh reader discards read-ahead"
  serial bug). Context-aware read via goroutine + select so
  dialTimeout is authoritative. 7 unit tests with a tri-phase
  modem sim on net.Pipe.
- VoIP-SIP backend declared as a v1.3 subprocess because the
  seccomp `dial` profile blocks socket() in the ElSereno
  parent.

### Extra — audit.SyncFromFile helper (`0d1b8d8`)
Bootstrap a fresh DB from an operator's existing
`~/.elsereno/audit.jsonl`. Walks the file, re-validates every
prev_hash + entry_hash, skips IDs already in target, calls
MirrorWriter.Mirror. Idempotent + tamper-detecting. Ships
`DBExistingID(conn)` as the pgx-backed default helper.

## Retrospective

### What went well
- **Dashboard panels** came together fast because the SSE
  broadcaster from v1.1 + the `APIV1Deps` dependency bundle
  kept the HTTP surface clean. Tests drive in-memory fakes;
  the dev-db is only needed for real-end-to-end checks.
- **GitHub Attestations API** replaced the slsa-generator
  reusable workflow in a single commit without losing any
  SLSA L3 property. Should have done this in v1.1 — the
  fix was half a day's work.
- **Per-protocol Handle loops** converged on a single
  pattern (parse wire → extract identifier → allowlist match
  → forward-or-protocol-native-refusal). That meant once
  DNP3 was debugged, IEC-104 / HART-IP / ATG / Fox landed
  back-to-back with only per-protocol refusal-frame
  differences.

### Pitfalls hit
- **HART long-vs-short frame delimiter**: I initially coded
  `delim & 0x07 == 0x02` thinking the low 3 bits selected
  frame format. The HART-FSK spec actually uses the HIGH bit
  (0x80). Caught by a test; fixed in both the handler and
  the refusal builder.
- **Serial-port bufio discard**: created a fresh
  bufio.Reader inside each ATModem read op. Read-ahead bytes
  got dropped → the next read blocked waiting for data
  already consumed. Fix: cache one bufio.Reader on the
  struct.
- **net.Pipe + unbounded Read**: bufio.ReadString is not
  context-aware. On a net.Pipe with nothing to read, the
  call blocks forever despite ctx.Deadline. Fix: run the
  read in a goroutine + select on `ctx.Done()`. Costs one
  goroutine per read but guarantees the dialTimeout fires.
- **`dockers_v2` schema vs `dockers` schema**: the v2 variant
  uses `labels:` map + `flags:` for buildx extras, not the
  `build_flag_templates:` list. Took one `goreleaser schema
  | jq .defs.DockerV2` to converge.

### Carry-overs to v1.3
- VoIP-SIP dial backend as a subprocess binary
  (`elsereno-dial-voip-sip`). Interface ready; implementation
  + seccomp-pair split is the task.
- BACnet full UDP relay — session primitives only today.
- OPC UA per-NodeId allowlist (currently the gate is at
  service-TypeID granularity; WriteValue / CallMethodRequest
  body parsing is v1.3).
- Address-space mutations for OPC UA (AddNodes /
  DeleteReferences / etc) are not in the mutating-service set.
- `dial batch` CLI plumbing to call Backend.Deliver on
  allowed numbers (interface shipped; CLI wire-up pending).
- `seccomp` arg-level filtering (openat O_WRONLY, socket
  AF_PACKET).
- Audit daemon so multiple processes can safely share the
  JSONL chain without flock racing.
