# Snapshot — v1.37.0: elsereno fingerprint validate verb

Single-chunk cycle. Closes the v1.28-chunk-1 +
v1.28-chunk-2 carryover that flagged the ProConOS + GE-SRTP
fingerprints as "confidence ~0.7 pending real-PLC
validation".

## Why a CLI verb instead of a fixture file

Real-PLC responses vary across vendors (Berghof / Lenze /
Phoenix Contact ILC for ProConOS; GE Mark VIe / RX3i /
PACSystems for GE-SRTP) and operators are the ones with
hardware access. A CLI verb gives them a self-service path
to validate without needing to push fixtures back into our
test suite (though they absolutely should — and the verb
output gives them a clear capture format to follow).

A pure fixture-based test file would have required us to
either ship sample-captures we don't own (legal / IP risk)
or fabricate plausible bytes (defeats the validation goal).
The verb sidesteps both: we ship the harness, operators
bring the bytes.

## Chunks

### Chunk 1 — fingerprint validate verb (`cfe268c`)

`cmd/elsereno/cmd_fingerprint.go` (new file, default build):

  elsereno fingerprint validate
      --plugin <name>
      (--file <path> | --hex <bytes>)
      [--timeout 5s]
      [--json]

Mechanism:

  1. Parse --file or --hex (whitespace-stripped) → []byte
     reply.
  2. Look up the plugin in `core.RegisteredPlugins()`
     (case-folded, "registered: <list>" hint on miss).
  3. `lc.Listen(ctx, "tcp", "127.0.0.1:0")` — random port
     localhost listener; goroutine drains whatever the
     probe sends, then writes the reply once and closes.
  4. Construct a Target pointing at the listener; call
     `plugin.Factory().Probe(ctx, target)`.
  5. Print the resulting Finding in human-readable text or
     JSON.

Helpers split out (each unit-testable + keeps the parent
function under funlen):

  - **loadFingerprintInput** — --file (os.ReadFile, gosec
    G304-tagged) vs --hex (hex.DecodeString after
    whitespace strip). Returns a 0-byte error so an
    operator pasting only whitespace gets a clean
    rejection.
  - **lookupPlugin** — case-folded EqualFold match against
    `core.RegisteredPlugins()`. Miss returns "unknown
    (registered: a, b, c)" so operators see what's
    available.
  - **driveProbeAgainstBytes** — listener + Accept-and-
    reply goroutine. The drain-goroutine inside Accept
    consumes the probe's request bytes so the server-side
    read doesn't half-block; the reply is written once
    then conn closes.
  - **emitFingerprintFinding** — text or JSON output.
    Factor keys sorted (manual sort, no extra import) for
    diff-friendly terminal copies into bug reports.

## Operator UX

```
$ elsereno fingerprint validate --plugin proconos \
    --file ~/captures/berghof-hello.bin

protocol: proconos
severity: high
score:    69
factors:
  auth_state:     90
  capability:     60       ← lifted: parser recognised KW-Software
  cve_exposure:   7
  exposure:       75
  impact_class:   75
  protocol_risk:  75
```

JSON form (for scripting + regression-pinning):

```
$ elsereno fingerprint validate --plugin gesrtp \
    --hex "020201000000010100000000..." --json | jq
{
  "created_at": "2026-05-04T11:39:36.520Z",
  "factors": { "auth_state": 90, "capability": 60, ... },
  "protocol": "gesrtp",
  "score": 64,
  "severity": "high"
}
```

The verb works for EVERY registered plugin, not just
proconos + gesrtp — it's a generic validation harness.
