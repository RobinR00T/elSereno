# Snapshot — v2.35.0: OPC UA HTTPS fingerprint plugin

Single-chunk cycle. Closes deferred roadmap item #3 (medio-grande)
from the v1.50 backlog. Adds the OPC UA HTTPS binding (Part 6)
as a separate plugin alongside the existing `opcua` TCP plugin.

## Why a separate plugin

Different default port (4843 vs 4840), different framing (HTTP
over TLS vs raw UA-TCP frames), different failure modes (TLS
handshake errors vs UA-protocol errors). Multiplexing inside
the existing TCP plugin would have made the state machine
unclear in findings.

## Probe surface (defensive only)

  1. TLS dial on (host, port) with `InsecureSkipVerify=true`
     (SCADA gateways overwhelmingly ship self-signed certs;
     spec-blessed for in-network use).
  2. HTTP/1.1 POST `/discovery` with Content-Type
     `application/opcua+uabinary` + 1-byte body.
  3. Inspect response headers in the first 8 KB.
  4. Classify into one of:
     - `uabinary-discovery` (Content-Type matches binary): strong
       UA hit, capability=75.
     - `uajson-discovery` (Content-Type matches JSON): strong UA
       hit, capability=75.
     - `ua-server-header` (Server header matches curated list of
       UA stack names: Unified Automation, Prosys, Matrikon,
       Kepware, open62541, node-opcua, etc.): weak hit,
       capability=50.
     - Plain HTTPS (no UA hints): `(nil, nil)` — no finding,
       avoids noise.

The probe NEVER establishes a SecureChannel, session, or any
mutating service request. Defensive default-build only.

## Plugin counts

  - **default build plugin count climbs 25 → 26**.
  - cve_exposure: 12 (modest — UA stacks have a few CVEs but
    HTTPS binding adds TLS-layer attack surface).
  - score formula identical weights to opcua plugin so findings
    score consistently.

## Tests (+5 unit)

  - TestProbe_UABinaryContentType (httptest TLS server with
    UA binary Content-Type) → strong finding.
  - TestProbe_UAServerHeaderOnly (404 + UA Server header) →
    weak finding.
  - TestProbe_PlainHTTPS (vanilla nginx-like response) →
    nil finding, no noise.
  - TestProbe_TLSHandshakeFailure (plain-TCP listener, no TLS)
    → TLS error propagated.
  - TestMetadata_Sane + TestProxyHandler_DenyAll.

httptest plumbing: self-signed RSA-2048 cert generated per
test with crypto/rand + x509.

## Honest scope

  - No actual GetEndpointsResponse parsing — we only consume
    response headers. A future cycle could parse the UA binary
    response and surface server-software-version /
    user-token-policies / security-modes.
  - opc.wss:// (WebSockets+TLS binding) NOT added in this
    cycle — separate plugin if/when it becomes operationally
    relevant. WebSocket framing is meaningfully different.
  - INSTALL.md unchanged. Build sizes unchanged.

## Build sizes

Unchanged across all 3 variants (TLS, http/1.1 minimal, all in
stdlib — no new dependencies).

Adds ~280 LoC plugin + 200 LoC tests.

## Carryover

  - **vNext**: opc.wss:// WebSocket binding plugin if SCADA
    deployments use it (uncommon today).
  - **vNext**: full GetEndpointsResponse UA-binary parser to
    surface server-software-version, user-token-policies,
    security-modes in the finding payload.
  - **vNext**: cve_exposure refinement once UA HTTPS-specific
    CVE history hardens.
