# Snapshot — v1.27.0: seccomp wire-up + pcworx/mms gates + record-replay

Four-chunk cycle that closes a cluster of v1.26 follow-ups +
shaves the v1.25 offensive-proxy gap from 5 (sip / iax2 /
pbxhttp / opcua / bacnet via ADR-040) + 1 (modbus from v1.4)
+ 1 (cwmp from v1.11) to 9 (adding pcworx + mms session-level).

No new fingerprint plugins; default-build plugin count stays at
**27**. Offensive write-gate count: **7 → 9** (modbus + opcua +
sip + iax2 + pbxhttp + bacnet + cwmp + pcworx + mms).

## Chunks

### Chunk 1 — seccomp arg-filter wire-up (`5533856`)

Closes the v1.26 chunk-2 follow-up: the arg-filter primitives
shipped in v1.26 are now wired into the per-profile
`sandbox.Load()` call so operators get preset enforcement
without any code change at the call site.

Per-profile mapping:

  - **ProfileHarvest** → openat-no-write. Harvest helpers emit
    findings to stdout and never legitimately write files in
    the harvest sandbox. Prevents a compromised harvest helper
    from dropping a file (e.g. offline exfil).
  - **ProfileDial** → socket-deny-AF_PACKET-AF_NETLINK. Dial
    subprocesses need AF_INET / AF_INET6 sockets but never
    legitimately need AF_PACKET (raw L2) or AF_NETLINK (kernel-
    config). Both ruled out as escape vectors.
  - **ProfileExploit** → no preset (CVE exploits sometimes
    legitimately need openat with O_CREAT for state files /
    output captures / dropped artefacts).

Implementation: new `argRulesFor(profile, nums)` function +
`sandbox.Load()` now calls `CompileFilterWithArgs(profile,
argRules)`. The Availability.Reason string gains an
`arg-rules=N` counter so audit-printout consumers can see
whether the preset fired.

### Chunk 2 — pcworx offensive write-gate (`d4489f2`)

Replaces the v1.25-chunk-2 fail-closed pcworx proxy with a
**triple-confirm session-level gate**. Honest scope: PCWorx's
command vocabulary post the IBETH01 hello (variable read /
write / runtime control IDs) is not authoritatively documented
in any public reference I could validate. Inventing a wire-
parser based on conflicting nmap-NSE / Metasploit / ICS-CERT-
advisory excerpts would risk shipping a gate that
misclassifies operator traffic. v1.27 chunk 2 ships the fence +
audit row + relay; per-frame command gating waits for a future
cycle once test vectors against real ILC firmware are available.

Public API:

  - `AllowedIntent` — operator's free-text rationale (incorporated
    in PayloadHash so two distinct sessions with different
    rationale produce different confirm-tokens).
  - `AllowlistHash` (case-folded + sorted-by-description).
  - `SessionMutation` (Category=Write / Operation=proxy_session).
  - `WriteGatedHandler` with Authorise() + Handle() — refuses
    pre-Authorise; relays bytes verbatim once authorised.

9 tests covering hash invariants + Authorise contract + relay
correctness.

### Chunk 3 — mms offensive write-gate (`cec5ab9`)

Same shape as chunk 2 for IEC 61850 MMS. Honest scope: a true
wire-level MMS gate would need an ASN.1 BER parser walking the
OSI session SPDU + ACSE A-ASSOCIATE + MMS PDUs
(ConfirmedRequestPDU / Read / Write / Status / Identify / etc.)
to extract Object references + service codes. Significant
parser surface — shipping it without test vectors against real
substation devices risks misclassifying operator traffic. v1.27
chunk 3 ships the fence + audit row + relay; full ASN.1-walking
MMS PDU gate stays on the v1.35 list.

9 tests mirroring chunk 2's coverage adapted to relay-domain
idioms.

### Chunk 4 — record & replay primitive (`986f3c2`)

Closes the v1.26 carry-over (record & replay de sesiones de
proxy). Forensic post-mortem + lab-training use case. NDJSON
file format (one event per line, jq-friendly):

  - Line 1: `HeaderEvent { schema, dir=header, started_at,
    protocol, target }`.
  - Line 2..N: `ChunkEvent { schema, ts, dir
    (client_to_upstream | upstream_to_client), len,
    hex }`.

Public API:

  - `Open(path, protocol, target)` creates a 0600 file +
    writes the header.
  - `Wrap` / `WrapClient` / `WrapUpstream` — three direction-
    aware io.ReadWriter wrappers. The proxy code Wraps each
    side of its conn pair; reads + writes flow through
    transparently while being tagged + recorded.
  - `Replay(ctx, path, cb)` iterates the file in order;
    ctx-cancel returns ctx.Canceled.
  - `SeekHeader(path)` returns just the header for "what is
    this file?" queries.
  - `DecodeBytes()` on a ChunkEvent inverts the hex encoding.

8 tests including a real net.Pipe round-trip + bytes-fidelity
+ ctx-cancel + schema-mismatch rejection.

**Wire-into-proxy integration** (where each gated proxy's
WriteGatedHandler accepts an optional Recorder + Wrap*'s its
conns) is a v1.28+ candidate. Primitive ships now so dashboard
live-feed + supervisor recordings can land independently.
