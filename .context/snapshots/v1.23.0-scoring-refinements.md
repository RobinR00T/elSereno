# Snapshot — v1.23.0: scoring refinements

Two chunks: a CVE-exposure factor expansion across 7 existing
plugins plus a banner-dictionary expansion adding 21 new vendor
patterns. Both refine the score signal without changing
default-build behaviour or wire shape. Default-build plugin
count unchanged at **25**; banner-dictionary vendor count
climbs from **9 to 30**; cve_exposure non-zero plugin count
climbs from **2 (codesys, redlion) to 9**.

## Chunk 1 — CVE-exposure factor expansion

`(commit e262b8d)` Set non-zero `cve_exposure` values on 7
plugins with well-documented CVE families. Each value is
documented in code with specific CVE references so future
maintainers can validate / extend.

| Plugin | New cve_exposure | Anchor CVEs |
|---|---:|---|
| cwmp | 15 | CVE-2014-9222 (Misfortune Cookie / RomPager), TR-064 NewNTPServer family |
| dnp3 | 12 | CVE-2013-2825 (CRC bypass), CVE-2013-2829 (frame stack overflow), CVE-2014-5410 |
| iec104 | 10 | CVE-2015-7906, CVE-2017-12089, CVE-2019-13548 (all SIPROTEC / SICAM PAS) |
| bacnet | 8 | CVE-2018-10628, CVE-2019-12480 (Wago), CVE-2020-12511 (Schneider U.motion) |
| opcua | 8 | CVE-2017-12069, CVE-2019-10936 (open62541), CVE-2022-29862 |
| hartip | 7 | CVE-2014-7494 (Honeywell XYR), CVE-2015-7905 (Yokogawa STARDOM), CVE-2019-9869 |
| atg | 6 | CVE-2017-14432/14433 (Veeder-Root TLS-450), CVE-2018-5443 |

cve_exposure remains weighted at 0.10 in `scoreFor` (per
ADR-006). The factor's intent is "is there a known CVE
attackers can chain after the fingerprint?" — values reflect
breadth of CVE family + ease-of-exploitation, not raw CVSS
scores. Combined with the v1.22 codesys=10 + redlion=5
values, **9 of 25 plugins** now publish a non-zero
cve_exposure score (was 2).

## Chunk 2 — Banner dictionary expansion

`(commit 20f7d1e)` 21 new vendor patterns added to the banner
plugin's `vendorRules`. Curated against Internet-exposed
device populations actually seen in public ICS scans (Shodan,
Censys), not exhaustive of every automation vendor.

Industrial controller / HMI / RTU (14 new vendors): Siemens
SIMATIC + RUGGEDCOM, Rockwell Automation / Allen-Bradley
(CompactLogix / ControlLogix / MicroLogix), Schneider
Modicon (M340/M580/M251), ABB (AC500 + Robotics), WAGO PFC,
Beckhoff CX/BX, Phoenix Contact (AXC + ME-PLC), Hirschmann /
Belden, Westermo, Advantech, Sealevel, Honeywell Experion +
XYR, Johnson Controls Metasys, Tridium Niagara.

Network gear adjacent to ICS (6 new vendors): Cisco IOS /
IOS-XE, MikroTik RouterOS, Ubiquiti EdgeOS / UniFi, pfSense /
OPNsense, Dropbear SSH, RomPager.

Order-sensitivity guarded by `TestDetectVendorOrderingSiemensVsCisco`
asserting that a banner referencing both SIMATIC and Cisco IOS
matches Siemens first (most-specific rule wins). 24 new test
cases bring the banner-dictionary test count from 8 to 32.

## Operator surface at end of v1.23

- 9 of 25 plugins publish a non-zero `cve_exposure` score so
  the dashboard ranks plugins with known CVE families higher.
- Banner plugin recognises 30 vendor families across ICS
  controllers, HMIs, RTUs, and network gear — operators get
  a vendor label on far more banner-style finds.

  CoDeSys / RLN write services).

- TUI (bubbletea), Windows support, OIDC + roles,
