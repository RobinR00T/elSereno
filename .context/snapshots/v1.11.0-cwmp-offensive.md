# Snapshot ‚Äî v1.11.0: CWMP offensive proxy (SOAP RPC allowlist)

Closed **2026-04-24**. Single-chunk cycle that completes the
TR-069 story v1.4 started. 7 offensive write-gated proxies in
the default build (up from 6 ‚Äî cwmp joins modbus/opcua/sip/
iax2/pbxhttp/bacnet).

## Why this chunk matters

TR-069 is the ISP-standard remote-management protocol between
an ACS (Auto-Configuration Server) and a fleet of CPEs (routers,
ONTs, STBs, phones, IP phones). The ACS-CPE channel is one of
the most privileged links in an ISP network:

- `SetParameterValues` can push config silently to every CPE.
- `Download` can push firmware.
- `FactoryReset` can wipe a device.
- `Reboot` can cycle a fleet.

A compromised or misconfigured ACS is a fleet-wide risk. The
v1.4 fingerprint plugin detects exposed ACSs; this chunk lets
operators put a gate between ACS and CPE during change windows
with an explicit allowlist of the RPCs they're authorising.

## Chunk

### Chunk 1 ‚Äî CWMP offensive proxy (`9793569`)

Library (`offensive/write/cwmp/gatedproxy.go`):

- `AllowedRPC{Name string}` opt-in type.
- `WriteGatedHandler.Allowed []AllowedRPC` field.
- `canonicaliseRPC` ‚Äî strips namespace prefix (`cwmp:` /
  `cwmp-1-0:` / `cwmp-1-2:` / any XML-name-like prefix) and
  trims whitespace. Case preserved because TR-069 ¬ßA.4 RPC
  names ARE case-sensitive.
- `AllowlistHash(target, allowed)` ‚Äî sorted,
  deterministic: `target || 0x00 || RPC<NUL> √ó sorted_rpcs`.
- `SessionMutation(target, allowed)` factory.
- `alwaysSafeRPCs` set (14 entries): `GetParameterNames`,
  `GetParameterValues`, `GetParameterAttributes`, +
  `Response` variants; `GetRPCMethods`; `Inform`/
  `InformResponse`; `TransferComplete`/`Response`/
  `AutonomousTransferComplete`; `Kicked`/`Response`;
  `Fault`. These always pass ‚Äî blocking would break the
  CPE registration cycle.
- `Handle` / `handleOne`:
    * Non-POST (GET/HEAD/OPTIONS) bypasses SOAP parser
      entirely. TR-069 is POST-only; non-POST is ACS
      status / health-probe territory.
    * POST: reads body, extracts RPC name via streaming
      `xml.Decoder` (finds first `StartElement` under
      `<*:Body>`). Handles `soap:`/`soap-env:`/`soapenv:`
      prefix variants transparently because the decoder
      compares local names.
    * If RPC is in `alwaysSafeRPCs` ‚Üí forward.
    * If RPC is in operator's `Allowed` ‚Üí forward.
    * Else ‚Üí SOAP Fault refusal.
- `writeSOAPFault` ‚Äî emits HTTP 200 OK + CWMP SOAP Fault
  body with TR-069 Annex A FaultCode 9001 "Request denied".
  `X-Elsereno-Gate-Reason` header for operator trace. TR-069
  convention: app-level errors ride on HTTP 200 OK; HTTP
  errors are for transport problems. The gate follows that.
- `forwardBuffered` / `forwardRequest` helpers ‚Äî replay the
  captured body when we allow the RPC; relay upstream
  response back to client.

## Design notes

### Why HTTP 200 OK + SOAP Fault, not HTTP 403

TR-069 treats application-level errors as SOAP Faults; HTTP
status codes are for transport-layer problems. Returning HTTP
403 would make ACS code classify the rejection as "can't reach
the CPE" (retry, alert, etc.) rather than "CPE refused this
operation" (surface to operator GUI). The SOAP Fault 9001 is
the spec-conformant way to say "RPC denied by policy".

### Why case-sensitive RPC names

TR-069 ¬ßA.4 says RPC names are case-sensitive.
`SetParameterValues` is the standard name; `setparametervalues`
is NOT a valid TR-069 RPC. The canonicaliser therefore only
strips whitespace + namespace prefix; it does NOT lowercase.

This matters for the allowlist: an operator who lists
`setparametervalues` (lowercase) would not match an actual
`<cwmp:SetParameterValues>` on the wire. We could warn at
dry-run time if the operator lists a name whose case doesn't
match any known TR-069 RPC ‚Äî a candidate refinement for
v1.12.

### Why non-POST bypasses SOAP parse

Many ACS deployments expose status / health endpoints on the
same port as the TR-069 RPC handler. These are commonly GET
requests. Refusing them would make the gate a dependency
for benign operator traffic (ACS dashboards, Prometheus
exporters, etc.). TR-069 proper is POST-only, so non-POST
can pass unconditionally without affecting the RPC gate's
security model.

### Why empty-body POST passes through

Some ACS deployments send empty POSTs as keep-alives between
RPC batches. There's no RPC to check, so the gate passes them
transparently. Upstream ACS will return its own response.

## Retrospective

### What went well
- Pattern copy from v1.10 (sip AOR gate) was mostly verbatim.
  Hash ‚Üí mutation ‚Üí handler struct ‚Üí Authorise ‚Üí handleOne
  ‚Üí CLI flag ‚Üí YAML field ‚Üí emitter ‚Üí proxy listen wiring.
  The only genuinely-new code is `extractRPCName` using the
  streaming `xml.Decoder` + the SOAP Fault emitter.
- TR-069 Annex A FaultCode 9001 is the right semantic match.
  A quick review of the spec confirmed: "Request denied"
  means "the CPE refuses to perform this RPC at this time
  (policy)", which is exactly what the gate is doing.
- Streaming XML decoder finds the RPC without materialising
  the full parameter tree ‚Äî works on arbitrarily large
  `SetParameterValues` bodies.

### Pitfalls hit + fixed
- `[32]byte` return value isn't addressable in Go, so tests
  that did `AllowlistHash(t, a)[:]` didn't compile. Had seen
  this before (v1.9 carry-over). Fix: `hashEqual([32]byte,
  [32]byte) bool` helper that binds both operands.
- gofmt complained about alignment in the `alwaysSafeRPCs`
  map literal. `gofmt -w` fixed.

### Carry-overs to v1.12+
- Per-parameter-path allowlist for `SetParameterValues`
  (e.g. allow only `InternetGatewayDevice.WANDevice.*`).
  Analogue of OPC UA per-NodeId, but using the SOAP
  ParameterList children as the inner data structure.
- Firmware-URL allowlist for `Download` (pin URL + optional
  SHA-256 of the firmware blob). Would require a second-
  layer XML parse to extract the `<URL>` + `<TargetFileName>`
  fields.
- RPC-name case-warning in dry-run ‚Äî flag lowercase or
  unknown RPC names before the operator mints a token.
- CWMP-over-TLS on 7548 ‚Äî currently the gate doesn't care
  about the transport; HTTP or HTTPS both work as long as
  the proxy is terminating TLS locally.
- Batch `DownloadResponse` / `RebootResponse` routing (CWMP
  has a deferred-response protocol; chunk 1's gate handles
  single-request-per-TCP; multi-RPC batches need a response
  path test).
- SIGHUP reload (still pending ‚Äî needs PayloadHash /
  token-invalidation redesign).
